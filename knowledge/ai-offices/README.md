# AI-офисы

Методология построения AI-офисов на базе Claude Code: агенты, скиллы, память, governance, инструменты.

## Зачем эта папка

Когда офис собирает новых агентов или ревизует существующую структуру — он опирается не на «здравый смысл», а на готовые проверенные принципы. Эта папка — место где эти принципы лежат и читаются.

## Как пользоваться

### Если ты агент архитектуры (Демиург / Architect-of-Order)

Читай `distilled/<topic>.md` ПЕРЕД задачей. Каждый дистиллят — короткая выжимка по одной теме (200-500 строк, можно за 2 минуты прочитать).

См. таблицу маршрутизации в `INDEX.md` — какой файл читать под какую задачу.

### Если ищешь эталоны вовне

`references/` — каталог публичных репо, людей и инструментов, у которых учиться:
- `repos.md` — топ-12 публичных репо с офисами Claude Code
- `people.md` — 15 практиков индустрии, за кем следить
- `tools.md` — каталог MCP-серверов, hooks, сервисов (что подключать)
- `methodologies.md` — публичные методологии (Compound Engineering, Florian rubric, AGENTS.md и др.)

### Если хочешь добавить методологию

1. Если тема **уже есть** в `distilled/` — обнови существующий файл, добавь дату в frontmatter
2. Если тема **новая** — создай новый файл в `distilled/<topic>.md`, добавь ссылку в `INDEX.md`
