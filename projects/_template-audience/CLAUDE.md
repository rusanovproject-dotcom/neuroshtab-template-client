# Stage 1 — Распаковка ЦА (Audience)

> Минимальная папка под ОДНУ задачу: глубокая распаковка целевой аудитории. Без продукта, воронки, бренда. Когда ЦА готова — наслаиваем следующие стадии отдельно.

## Зачем именно так

Клиент пришёл с базовой ЦА — где-то одно общее «эксперт 28-45», в diagnostic-card. Нужен **фундамент**: кто реально покупает, какие сегменты, какие боли дословно, какой awareness, кто конкуренты. Без этого фундамента продукт-оффер-воронка — стрельба наугад.

Папка живёт **рядом** с основным проектом клиента (`projects/main/`), не заменяет его. После распаковки ЦА её данные используются основным проектом или мигрируют туда.

## Метафора

Это не полный организм — это **диагностика**. Только что нужно для понимания «кто наш клиент»:

| Слой | Где | Роль |
|------|-----|------|
| 🍴 Желудок | `inbox/` | приём всех входящих про ЦА: цитаты, скрины комментов, голосовые с инсайтами, ссылки на конкурентов, транскрипты интервью |
| 🩸 Кровь | `audience/research/transcripts/` | сырые материалы — кастдев, опросы, выступления |
| ❤️ Сердце | `hypotheses.md` | гипотезы только про ЦА: `[ICP]`, `[SEGMENT]`, `[MESSAGE]` |
| 🧠 Мозг | `audience/` | главный орган — кому продаём (NORTH-STAR + сегменты + voice-of-customer) |
| 👁 Глаза | `metrics.md` | размер сегментов, потенциальный CAC, источники лидов |
| 📚 Гиппокамп | `learnings.md` | уроки про ЦА (которые потом перенесутся в основной проект) |
| 💾 Точка сохранения | `_state/pipeline-progress.md` | resume-pattern Stage 1 — какая сессия идёт, что закончили, next action. Алекс обновляет после каждого Шага. Между сессиями `/clear` + `/audience-resume` |

Нет 🧠 «чем отличаемся / что продаём / как доставляем» — это следующие стадии.

## Главные артефакты на выходе

После работы Алекса в этой папке появятся:

```
audience/
├── segments/
│   ├── NORTH-STAR.md                 ⭐ ТОП-3 сегмента — главный артефакт
│   ├── segments-map.md               backlog отвергнутых гипотез + 7-осевой scoring
│   └── {slug}/                       по папке на каждый ТОП-сегмент
│       ├── segment-portrait.md       БПСВ + 7 блоков интервью + awareness + цвета 4
│       ├── segment-core.md           ядро сегмента + lingvo (4 правила) + Hero-формула
│       ├── voice-of-segment.md       30-50 цитат с источниками (Phase B)
│       └── segment-observations.md   живые наблюдения (пятничная ревизия)
├── voice-of-customer.md              голос ЦА — цитаты с источниками и таймкодами
└── research/transcripts/             сырые транскрипты кастдев/интервью

intel/                                (если запущен /competitors-research)
└── competitors-{slug}/               конкурентный анализ RU/CIS + US/EU
```

## Поток работы

```
1. У клиента есть базовая ЦА (от Стратега) → читаем как starter
2. Алекс запускает /audience-stage (Internet-First, 4 фазы)
   Phase A — quick-capture (10-15 мин клиент)
   Phase B — internet-research (30-90 мин субагенты)
   Phase C — validation (15-25 мин клиент через мультиселекты)
   Phase D — awareness-lite (углубление hot-сегмента)
   → ТОП-3 сегмента в NORTH-STAR.md + voice-of-segment × N
3. Stop+wait после каждой Phase
4. Финал: NORTH-STAR + N segment-portrait + N segment-core + voice-of-customer + competitors + audience-report.html
```

## Закон голоса

Алекс наполняет:
- `audience/voice-of-customer.md` (append only — голос ЦА с источниками)
- `audience/segments/{slug}/segment-portrait.md`, `segment-core.md`, `voice-of-segment.md`, `segment-observations.md`
- `hypotheses.md` (тест-процесс)
- `intel/competitors-*/`

Владелец сам пишет (если хочет):
- `audience/segments/NORTH-STAR.md` (можно править после ревизии)
- `learnings.md` — уроки

Алекс не перезаписывает — предлагает diff в `hypotheses.md → Log`.

## Что дальше после Stage 1

После того как ЦА распакована, можно **наслаивать** следующие стадии. Каждая — отдельный пак-надстройка (когда созреют):

- **Stage 2 Product** → `/install-agent alex-product` → создаст `product/` папку, активирует распаковку оффера через 6 диалоговых протоколов Хормози
- **Stage 3 Funnel** → `/install-agent alex-funnel` → `funnel/` + per-segment маршруты
- **Sales Meetings** → `/install-agent sales-meetings` → `customers/` + парсинг транскриптов
- **Brand** → когда подключим Брендмейкера

Текущая папка остаётся живой — пятничная ревизия сегментов работает постоянно (`/revise-segment`).

## Связь с основным проектом

Эта папка живёт рядом с `projects/main/` (или как ты её назвал). Связь — через ссылки в `README.md` (см. ниже).

После распаковки данные могут:
- Использоваться напрямую отсюда (мета-скиллы Алекса знают где искать)
- Скопироваться в `projects/main/audience/` если хочешь объединить
