# office/strategy/ — мета-стратегия клиента

Эта папка заполняется автоматически после того как клиент проходит онбординг со **Стратегом** (скиллы `/strategist-intake` → `/strategist-unpack` → `/strategist-discovery` → `/strategist-roadmap`).

## Что сюда ложится

- `strategy.md` — главный блокер, 3 рычага роста, топ-3 AI-задачи, ресурсы, ограничения
- `progress-index.md` — живой чеклист по 9 блокам артефактов (profile, audience, products, funnel, brand, knowledge, clients, financial, ai-office)
- `roadmap.md` — план на 6 недель (копия из `projects/<main>/strategy/roadmap.md`)

## Кто читает

- **Director** — при каждом старте сессии, чтобы роутить задачи в контексте стратегии
- Другие агенты — когда Director даёт им задачу, они могут свериться со стратегией

## Обновления

- `/strategist-roadmap` — инициализирует все три файла
- `/strategist-checkin` — еженедельное обновление progress-index + roadmap

**До первого онбординга папка пустая.** Это нормально — Director работает и без неё, просто без контекста программы.
