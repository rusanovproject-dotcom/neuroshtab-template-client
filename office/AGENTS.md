## Как работать с агентами

**Обычная задача** — через Director:
> "Нужно разобраться со стратегией на квартал"

Director сам позовёт нужного участника core-team.

**Онбординг программы:**
```
/strategist-intake     # первичная распаковка
/strategist-unpack     # детальный разбор документов
/strategist-discovery  # уточняющий диалог
/strategist-prepare    # сверка собранного и подготовка офиса
/strategist-roadmap    # финал: стратегия + 6-недельный план + HTML-роадмап
/strategist-checkin    # еженедельно
```

**Онбординг визуала (Дизайнер):**
```
Собери мне Brand Book    # первый шаг — 10 минут диалога, артефакт в knowledge/brand/
Хочу лендинг / обложку / презентацию   # Дизайнер предложит тип, напишет промт
```

**Прямое обращение** к агенту: `@director`, `@strategist`, `@demiurg`, `@designer`.

## Layered memory

У каждого агента — 4 файла:
- `core.md` — ядро, обновляется из template при будущих апдейтах
- `overrides.md` — твои персональные правила, приоритет над core, **не перезаписывается** при апдейтах
- `memory.md` — append-only, агент пишет сам после задач (decisions / patterns / context)
- `failures.md` — append-only, агент пишет сам о своих ошибках чтобы не повторять
- `CLAUDE.md` — склейка `@core.md` + `@overrides.md` (Claude Code читает это)

Пиши свои кастомизации в `overrides.md`. Так ты не потеряешь их когда обновится ядро.
