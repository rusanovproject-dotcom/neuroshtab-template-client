# Autopilot — Полный пайплайн задачи

Director берёт сырую задачу и проводит через 4 фазы:
**THINK → PLAN → REVIEW → EXECUTE**

Между THINK и PLAN — обязательное подтверждение Никиты.

---

## Фаза 1: THINK (анализ и видение)

**Цель:** понять задачу глубже чем {{CLIENT_NAME}} её сформулировал, предложить как реализовать.

**Процесс:**

1. **Прочитай контекст:**
   - `../../CLAUDE.md` — Algorithm, Revenue Tiers, роутинг
   - `../../context.md` — текущие приоритеты
   - `../../AGENTS.md` — кто из команды доступен
   - Notion Tasks — что уже в работе, нет ли дублей

2. **Прогони через Algorithm:**
   ```
   QUESTION  — Зачем? Кому? Какую проблему решает? Первый принцип или аналогия?
   DELETE    — Что можно НЕ делать? Что лишнее в формулировке?
   SIMPLIFY  — Как упростить до ядра? MVP vs полный вариант?
   ACCELERATE — Что ускорит? Есть готовое? Можно переиспользовать?
   AUTOMATE  — Нужна ли автоматизация или разовое действие?
   ```

3. **Определи Revenue Tier** (1-4) и оцени масштаб (< 30 мин / 30-120 мин / > 2 часов)

4. **Исследуй кодовую базу** (если задача техническая):
   - Найди связанные файлы
   - Пойми текущую архитектуру в зоне задачи
   - Определи что менять, что создавать

5. **Сформулируй видение реализации:**
   - Что конкретно делаем (scope)
   - Как это работает (подход)
   - Кто из агентов нужен
   - Какой результат получим
   - Что НЕ делаем (out of scope)

**Output THINK:**
```
## Задача: {переформулированное название}

**Revenue Tier:** {1-4} | **Масштаб:** {оценка} | **Агенты:** {кто нужен}

### Что я понял
{1-3 предложения — суть задачи своими словами}

### Как вижу реализацию
{3-7 пунктов — конкретные шаги на высоком уровне}

### Что НЕ делаем
{явно обозначить границы}

### Риски / вопросы
{если есть неясности — спросить}

---
Подтверждаешь? Или скорректируем?
```

**ВАЖНО:** После THINK — СТОП. Жди подтверждения.
- "да", "погнали", "ок", "го" → переходи к PLAN
- Коррекция от Никиты → обнови видение, покажи снова
- "нет", "отмена" → останови пайплайн

---

## Фаза 2: PLAN (детальный план)

**Цель:** превратить подтверждённое видение в пошаговый план выполнения.

**Процесс:**

1. **Декомпозируй** задачу на конкретные шаги (actions, не абстракции):
   - Каждый шаг = одно действие с проверяемым результатом
   - Шаги пронумерованы
   - Зависимости между шагами явно указаны

2. **Определи оркестрацию:**
   - Один агент или несколько?
   - Параллельно или последовательно?
   - Какие handoff'ы между шагами?

3. **Для каждого шага укажи:**
   - Что делать (конкретно)
   - Кто делает (агент или Director сам)
   - Какие файлы затрагиваются
   - Как проверить что шаг выполнен

4. **Определи критерии готовности** всей задачи

**Output PLAN:**
```
## План: {название задачи}

### Оркестрация
{одиночный / параллель: X + Y / цепочка: X → Y → Z}

### Шаги

| # | Что делать | Кто | Файлы | Проверка |
|---|-----------|-----|-------|----------|
| 1 | {действие} | {агент} | {пути} | {как проверить} |
| 2 | {действие} | {агент} | {пути} | {как проверить} |
| ... | | | | |

### Зависимости
- Шаг N зависит от шага M (почему)

### Готово когда
- [ ] {критерий 1}
- [ ] {критерий 2}
```

→ Автоматически переходи к REVIEW (без паузы).

---

## Фаза 3: REVIEW (проверка плана)

**Цель:** найти дыры, неэффективности, риски в плане. Докрутить.

**Чеклист ревью:**

1. **Полнота:** все ли шаги ведут к результату? Нет пропущенных?
2. **Порядок:** зависимости правильные? Можно что-то распараллелить?
3. **Эффективность:** нет ли лишних шагов? Можно проще?
4. **Риски:** что может пойти не так? Есть fallback?
5. **Файлы:** все нужные файлы указаны? Нет ли конфликтов?
6. **Агенты:** правильно назначены? Не перегружен ли кто?
7. **Out of scope:** не полез ли план за границы задачи?
8. **Критерии готовности:** конкретные, проверяемые?

**Если найдены проблемы:**
- Автоматически исправь план
- Покажи что изменил и почему:
```
### Ревью плана

**Найдено {N} улучшений:**
1. {что} — {почему изменил}
2. {что} — {почему изменил}

**План обновлён. Запускаю выполнение.**
```

**Если план чистый:**
```
### Ревью плана
План проверен, дыр не найдено. Запускаю выполнение.
```

→ Автоматически переходи к EXECUTE.

---

## Фаза 4: EXECUTE (выполнение)

**Цель:** выполнить план, контролируя качество каждого шага.

**Процесс:**

1. **Запусти шаги** согласно плану оркестрации:
   - Независимые шаги → параллельно через Agent tool
   - Зависимые шаги → последовательно
   - Шаги Director'а → сам

2. **Dashboard** (при 2+ параллельных агентах):
   ```
   | # | Задача | Агент | Статус |
   |---|--------|-------|--------|
   | 1 | ... | ... | работает... |
   | 2 | ... | ... | работает... |
   ```

3. **После каждого шага:**
   - Проверь результат по критерию из плана
   - Если ок → следующий шаг
   - Если ошибка → попробуй исправить (1 retry), если не получилось → эскалируй Никите

4. **После всех шагов:**
   - Проверь критерии готовности всей задачи
   - Покажи итог

**Output EXECUTE:**
```
## Результат: {название задачи}

### Что сделано
{краткий список выполненных шагов с результатами}

### Файлы изменены
- {путь} — {что изменилось}

### Проверка
- [x] {критерий 1} — ок
- [x] {критерий 2} — ок

### Notion
Задача «{название}» → статус обновлён.
```

## Граничные случаи

### Задача < 15 мин (мелкая)
Сокращённый пайплайн: THINK (1-2 строки) → сразу спроси "Делаю?" → EXECUTE.
Без формального PLAN и REVIEW.

### Задача требует только одного агента
Director делает THINK + PLAN, затем handoff агенту с полным контекстом.
Агент выполняет, Director проверяет результат.

### Задача чисто для Director'а (координация, Notion, статусы)
Director выполняет всё сам, без делегирования.

### {{CLIENT_NAME}} скорректировал план
Обнови план, покажи diff, продолжи с REVIEW.

## Примеры

### Пример 1: Техническая задача
```
{{CLIENT_NAME}}: "Сделай автоматизацию Директора чтобы он мог брать сырую задачу..."

THINK:
  Задача: Autopilot — пайплайн полного цикла задачи для Director
  Tier: 3 (продукт) | Масштаб: 1-2 часа | Агенты: Director (сам)
  
  Как вижу: создать новый скилл Director'а с 4 фазами...
  Подтверждаешь?

→ "да"

PLAN:
  1. Создать SKILL.md в director/skills/autopilot/
  2. Создать пользовательский скилл в .claude/skills/
  3. Обновить CLAUDE.md Director'а
  4. Обновить корневой office/CLAUDE.md

REVIEW:
  Найдено 1 улучшение: добавить обработку мелких задач (< 15 мин)

EXECUTE:
  [выполнение шагов]
  Результат: скилл создан, протестирован, документирован.
```

### Пример 2: Контентная задача
```
{{CLIENT_NAME}}: "Напиши пост про то как я строю AI-офис"

THINK:
  Задача: Пост в ТГ-канал про строительство AI-офиса
  Tier: 2 (контент) | Масштаб: 30-60 мин | Агенты: Copywriter
  
  Как вижу: Copywriter пишет пост через hot-trail пайплайн...
  Подтверждаешь?

→ "да, но добавь про ошибки"

PLAN:
  1. Собрать контекст (lessons-learned, history)
  2. Copywriter → hot-trail с акцентом на ошибки
  3. Ревью на стиль Никиты

EXECUTE:
  → Copywriter (Agent tool) с полным handoff
```

## Интеграции

- **Notion:** обновляй статус задачи на каждой фазе (В процессе → Done)
- **day-state.json:** обновляй current_task при старте EXECUTE
- **session-log:** запиши итог после EXECUTE
- **lessons-learned:** если были ошибки — зафиксируй
