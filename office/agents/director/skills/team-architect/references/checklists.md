# Checklists — Чеклисты

## Чеклист: Создание нового агента

Перед созданием:
- [ ] Есть активные задачи этого типа (не "на будущее")
- [ ] Существующий агент не может взять (проверено)
- [ ] Определён output contract (что именно производит)
- [ ] Определены boundaries (что НЕ делает)
- [ ] Span of control после добавления в норме (<= 9 у Director)

Передача Demiurg:
- [ ] Brief: роль, задачи, связи с командой
- [ ] Смежные агенты определены (от кого получает, кому передаёт)
- [ ] Expected output первой задачи описан

## Чеклист: Интеграция в команду

После создания агента Demiurg:
- [ ] Агент добавлен в AGENTS.md
- [ ] Роутинг Director обновлён (таблица в CLAUDE.md)
- [ ] Handoffs прописаны (structured handoff template)
- [ ] Смежные агенты знают о новом (обновлены их "НЕ отвечает за" секции)
- [ ] context.md обновлён (если агент влияет на приоритеты)
- [ ] Первая задача = простая, с явным expected output
- [ ] ops/log.md — запись о добавлении агента

Проверка через 10 задач:
- [ ] Агент автономен в своей зоне
- [ ] Нет конфликтов с существующими агентами
- [ ] Output соответствует контракту
- [ ] Handoffs работают без потерь контекста

## Чеклист: Реструктуризация

Подготовка:
- [ ] Текущая структура задокументирована (AGENTS.md актуален)
- [ ] Проблемы идентифицированы (чеклист диагностики пройден)
- [ ] 2 варианта структуры с trade-offs подготовлены
- [ ] {{CLIENT_NAME}} informed / approved

Выполнение:
- [ ] AGENTS.md обновлён
- [ ] CLAUDE.md Director обновлён (роутинг)
- [ ] CLAUDE.md затронутых агентов обновлены
- [ ] context.md обновлён
- [ ] Handoffs пересмотрены и обновлены
- [ ] Новые Cluster Leads (если есть) имеют CLAUDE.md

Проверка:
- [ ] Каждый агент знает кому подчиняется (1 руководитель)
- [ ] Нет пересечений зон ответственности
- [ ] Span of control в норме на каждом уровне
- [ ] ops/log.md — запись о реструктуризации

## Чеклист: Диагностика проблем

Bottleneck:
- [ ] У кого самая длинная очередь задач?
- [ ] Кого чаще ждут другие агенты?
- [ ] Кто является зависимостью в большинстве handoff?

Нагрузка:
- [ ] Есть агенты без задач > 2 дней? (недогруз)
- [ ] Есть агенты с > 70% capacity на 1 тип задач? (перегруз)
- [ ] Director тратит > 50% на координацию? (bottleneck Director)

Структура:
- [ ] Задачи падают между агентами? (unclear ownership)
- [ ] Конфликты приоритетов между агентами? (matrix problem)
- [ ] Один агент не знает что делает другой? (silos)
- [ ] overrides.md обновляются? (knowledge transfer)

Waste (Lean):
- [ ] Overproduction: агенты делают больше запрошенного?
- [ ] Waiting: агенты ждут context/handoff?
- [ ] Motion: информация разбросана по местам?
- [ ] Defects: output не соответствует ожиданиям?
- [ ] Over-processing: лишние шаги в процессах?
- [ ] Inventory: незавершённые задачи накапливаются?
- [ ] Transportation: лишние передачи между агентами?

Итог:
- [ ] 1 главный bottleneck определён
- [ ] Рекомендация: 1-3 конкретных действия
