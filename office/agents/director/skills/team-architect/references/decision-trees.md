# Decision Trees — Деревья решений

## 1. Нужен ли новый агент?

```
Есть регулярные задачи нового типа?
├── НЕТ → Не нужен. Стоп.
└── ДА → Есть агент, который может взять?
    ├── ДА → Добавь скилл существующему агенту.
    │        Агент перегружен (>70% capacity)?
    │        ├── НЕТ → Скилл достаточно. Стоп.
    │        └── ДА → Создавать нового агента.
    └── НЕТ → Задачи занимают >20% времени Director?
        ├── НЕТ → Мониторь 2 недели. Если рост → создавай.
        └── ДА → Создавать нового агента.
```

**Красные флаги (НЕ создавать):**
- Агент "на будущее" без активных задач
- Дублирует функцию существующего агента
- Нет определённого output contract

## 2. Нужен ли новый отдел / кластер?

```
Более 2-3 агентов с похожей специализацией?
├── НЕТ → Не нужен. Стоп.
└── ДА → Задачи одного типа >30% общей нагрузки?
    ├── НЕТ → Мониторь. Пока не нужен.
    └── ДА → Coordination overhead между ними растёт?
        ├── НЕТ → Группируй логически, но без Cluster Lead.
        └── ДА → Создавай кластер с Lead.
```

**Типовые кластеры для AI-офиса при 10+ агентах:**

| Кластер | Агенты | Lead |
|---------|--------|------|
| Production | Copywriter, Designer, Equalizer | Production Lead |
| Intelligence | Intelligence, Producer | Strategy Lead |
| Engineering | Tech Lead, Frontend Designer, Vibe Coder | Tech Lead (как Lead) |

## 3. Agent Team или Subagent?

```
Задача параллелизуема?
├── НЕТ → Один агент + subagents (sequential).
└── ДА → Агентам нужно общаться между собой?
    ├── НЕТ → Subagents (дешевле по токенам).
    └── ДА → Агентов нужно 3-5?
        ├── ДА → Agent Team.
        └── НЕТ (>5) → Декомпозируй задачу. Потом Agent Team.
```

**Agent Team = дорого.** Используй только когда:
- Research с несколькими гипотезами
- Независимые модули без shared files
- Cross-layer изменения (frontend + backend + tests)

## 4. Плоская vs Иерархическая структура?

```
Количество агентов у Director?
├── <= 9 → Плоская. Без изменений.
├── 10-12 → Зона выбора:
│   Director = bottleneck?
│   ├── НЕТ → Плоская, но готовь кластеры.
│   └── ДА → Вводи Cluster Leads.
└── > 12 → Иерархическая. Обязательно.
```

**Гибридная модель (рекомендация при 10+):**
```
Level 0: Director — стратегия, роутинг
Level 1: Cluster Leads (2-3) — координация внутри кластера
Level 2: Specialists (3-5 на кластер) — исполнение
```

## 5. Нужна ли реструктуризация?

```
Симптомы:
- Задачи падают между агентами?
- Director тратит >50% на координацию?
- Конфликты приоритетов между агентами?
- Один агент не знает что делает другой?

Любой = ДА?
├── НЕТ → Без изменений. Стоп.
└── ДА → Сколько симптомов?
    ├── 1 → Точечный фикс (чеклист диагностики).
    ├── 2-3 → Частичная реструктуризация.
    └── 4 → Полная реструктуризация (все паттерны пересмотреть).
```
