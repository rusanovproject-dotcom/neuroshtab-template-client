# Team Patterns — Паттерны организации команды

## 1. Supervisor Pattern (текущий AI-офис)

```
Director (Orchestrator)
├── Agent A
├── Agent B
├── Agent C
└── ...
```

**Как работает:** Director получает задачу, декомпозирует, делегирует агенту, проверяет output.

| Плюсы | Минусы |
|-------|--------|
| Единая точка контроля | Director = потенциальный bottleneck |
| Простая координация | Не масштабируется > 9-12 |
| Чёткая ответственность | Зависимость от 1 координатора |

**Когда:** 1-9 агентов. Стандарт для малых AI-офисов.
**Текущий AI-офис (7 агентов):** этот паттерн. Работает. Не менять.

## 2. Hierarchical Pattern (при 12+ агентах)

```
Director
├── Production Lead
│   ├── Copywriter
│   ├── Designer
│   └── Equalizer
├── Strategy Lead
│   ├── Intelligence
│   └── Producer
└── Tech Lead (как Cluster Lead)
    ├── Frontend Designer
    └── Vibe Coder
```

**Как работает:** Director роутит в кластер, Cluster Lead распределяет внутри.

| Плюсы | Минусы |
|-------|--------|
| Масштабируется до 20-50 | Медленнее (2 уровня) |
| Director не bottleneck | Больше overhead на координацию |
| Кластеры автономны | Риск silos между кластерами |

**Когда:** 10-20 агентов. Вводить при span Director > 9 и Director = bottleneck.

**Пороги:**
- 10 агентов: готовить кластеры
- 12 агентов: вводить Cluster Leads
- 20 агентов: обязательные отделы

## 3. Sequential Workflow (конвейер)

```
Task → Agent A → Agent B → Agent C → Result
```

**Как работает:** каждый агент обрабатывает и передаёт дальше. Конвейер.

| Плюсы | Минусы |
|-------|--------|
| Предсказуемый процесс | Медленно (sequential) |
| Каждый шаг проверяем | Один сбой = остановка |
| Чёткие handoffs | Не параллелизуется |

**Когда:** задачи с фиксированной последовательностью.
**Пример в офисе:** Demiurg: Knowledge Mining -> Build -> Validate -> Iterate.
**Пример для контента:** Producer (тема) -> Copywriter (текст) -> Designer (визуал).

## 4. Agent Teams (параллельные)

```
Task → [Agent A + Agent B + Agent C] → Synthesis
```

**Как работает:** несколько агентов работают параллельно, результаты объединяются.

| Плюсы | Минусы |
|-------|--------|
| Быстро (параллельно) | Дорого по токенам |
| Разные перспективы | Merge conflicts |
| Хорошо для research | Макс 3-5 teammates |

**Когда (Claude Code docs):**
- Research с несколькими гипотезами
- Независимые модули без shared files
- Cross-layer изменения
- Debugging с конкурирующими теориями

**Когда НЕ:**
- Sequential задачи с зависимостями
- Одни и те же файлы
- Простые задачи (overhead > польза)

**Лимиты:** 3-5 teammates, 5-6 задач на teammate, nested teams НЕ поддерживаются.

## 5. Кросс-функциональные цепочки

```
Producer → Copywriter → Designer
     ↕            ↕
Intelligence   Tech Lead
```

**Как работает:** агенты из разных кластеров формируют временную цепочку для задачи.

| Плюсы | Минусы |
|-------|--------|
| Лучший результат (multi-skill) | Сложная координация |
| Гибкость | Нужен project owner |
| Нет silos | Конфликты приоритетов |

**Когда:** комплексные задачи (лонч продукта, ребрендинг, новая воронка).
**Правило:** временная цепочка, не постоянная. Director = project owner.

## Сравнительная таблица

| Паттерн | Агентов | Скорость | Cost | Контроль | Сложность |
|---------|---------|----------|------|----------|-----------|
| Supervisor | 1-9 | Средняя | Низкий | Высокий | Низкая |
| Hierarchical | 10-50 | Средняя | Средний | Высокий | Средняя |
| Sequential | Любое | Низкая | Низкий | Высокий | Низкая |
| Agent Teams | 3-5 | Высокая | Высокий | Средний | Средняя |
| Кросс-функц. | 3-5 | Средняя | Средний | Средний | Высокая |

## Рекомендация для AI-офиса

**Сейчас (7 агентов):** Supervisor. Без изменений.
**При 10:** Supervisor + готовить кластеры (группировать логически).
**При 12:** Hierarchical (Cluster Leads).
**Для сложных задач:** кросс-функциональные цепочки под управлением Director.
**Для research/параллельных:** Agent Teams (редко, по необходимости).
