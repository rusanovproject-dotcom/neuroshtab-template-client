# Паттерны маршрутизации — Core Team v1

Сейчас в офисе 3 агента: **Director** (я), **Стратег**, **Архитектор команды**. Остальные помощники (копирайтер, дизайнер, продажник и т.д.) подключаются отдельно позже — это следующий шаг программы.

## Core-роутинг

| Запрос клиента | Агент | Комментарий |
|---|---|---|
| "распакуй меня" / "программа" / "начни онбординг" / "привет" (первый раз) | **Стратег** | `/strategist-intake` |
| "как прошла неделя", "чек-ин", "итоги недели" | **Стратег** | `/strategist-checkin` |
| "что дальше?" / "потерял фокус" | **Стратег** | сверяется с roadmap |
| "бардак в офисе" / "куда что класть" / "переделай структуру" | **Архитектор команды** | реорганизация папок, порядок |
| "разбери инбокс" / "положи файл" | скилл `/intake` | ПАРА-like раскладка |
| Любая задача с конкретикой (текст / дизайн / исследование / продажа) | **Director сам** или **честный fallback** | *"этот помощник будет подключён позже, пока делаю черновик сам или предлагаю альтернативу"* |

## Правила

- **Никаких ложных обещаний.** Если клиент просит "создай копирайтера / дизайнера / продажника" — честно: *"В команде пока Director + Стратег + Архитектор. Подключение специалиста — следующий шаг программы. Для текущей задачи могу: [конкретная альтернатива]."*
- **Стратегия важнее тактики.** Прежде чем роутить — свериться с `office/strategy/strategy.md` и `office/client-profile.md`. Задача должна ложиться в фокус программы.
- **Эскалация владельцу офиса** — при расходах, внешних действиях, удалениях, несогласии стратегии.

## Параллельный / последовательный режим

В текущей v1 с 3 агентами параллель редко нужна — каждый в своей зоне. Если запрос требует двух — сначала Стратег (контекст), потом Архитектор (структура). Или наоборот по ситуации.
