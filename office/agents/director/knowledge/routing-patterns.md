# Паттерны маршрутизации

## Простые (1 агент)
| Паттерн | Агент | Пример |
|---------|-------|--------|
| "напиши пост" | Copywriter | Текстовый контент |
| "исследуй рынок" | Intelligence | Аналитика |
| "сделай оффер" | Producer | Продуктовая упаковка |
| "задеплой" | Tech Lead | Технические задачи |
| "нарисуй / визуал / обложка / лендинг / презентация / дашборд / Brand Book" | **Дизайнер** | см. секцию "Дизайн-триггер" в `core.md` — сначала предложить 3-5 типов из каталога |
| "собери стиль / брендбук / визуальный язык" | **Дизайнер** → `brand-onboarding` | первый шаг для любого проекта, создаёт `knowledge/brand/{project}/brand-book.md` |
| "диагностируй" | отложено | другие помощники подключаются позже отдельным шагом программы |
| "проверь артефакт" | отложено | ревьюер (Брахма) подключается позже |

## Составные (2+ агента, последовательно)
| Паттерн | Цепочка | Пример |
|---------|---------|--------|
| "пост с данными" | Intelligence → Copywriter | Сначала данные, потом текст |
| "запуск продукта" | Producer → Copywriter + Designer | Стратегия → контент + визуал |
| "проверь оффер" | Producer → Equalizer | Создание → валидация |

## Параллельные (независимые)
| Паттерн | Параллель | Условие |
|---------|-----------|---------|
| "пост + исследование" | Copywriter || Intelligence | Не трогают одни файлы |
| "визуал + текст" | Designer || Copywriter | Независимые результаты |

## Антипаттерны (НЕ делать)
- Copywriter зависит от свежих данных Intelligence → НЕ параллелить
- Equalizer проверяет оффер Producer → НЕ параллелить
- 3+ агента на одну задачу → разбить на подзадачи

## Сигналы эскалации
- "бюджет", "деньги", ">$20" → спросить Никиту
- "удалить сервис", "деплой в прод" → с подтверждения
- Не подходит ни один триггер → разбить или спросить

## Правила

- **Brand Book — первый шаг дизайн-работы.** Если клиент просит дизайн, а `knowledge/brand/{project}/brand-book.md` отсутствует — Дизайнер сначала запускает `brand-onboarding`, потом делает первый промт.

## Связки через фазы

- **Стратег → Дизайнер** — после распаковки стратегии можно сразу собрать Brand Book под проект
- **Дизайнер → Архитектор** — если для нового проекта нужна свежая папка `knowledge/brand/{project}/`
