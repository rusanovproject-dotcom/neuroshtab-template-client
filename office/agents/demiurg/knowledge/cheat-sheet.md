# Cheat Sheet — все правила на одной странице

> Загружай этот файл вместо 5 отдельных. Детали — по ссылкам.

## Агент

- Identity <= 300 токенов, уникальный принцип (не "делай хорошо")
- CLAUDE.md = 60-150 строк. Больше → вынеси в knowledge/
- Ссылки на файлы (handle), не inline контент
- "НЕ отвечает за" с redirect путями обязательна
- Секция "Как работать" = пронумерованные шаги (структурность!)
- MEMORY.md для самообучения

## Скилл

- Description: цепляющий + MANDATORY TRIGGERS + "DO NOT use when"
- >= 2 примера Input → Output (конкретные, не абстрактные)
- Фазы с quality gates и exit criteria
- Error recovery: что делать если сломалось
- Один скилл = одна задача (узкий > универсальный)
- Max 500 строк

## Knowledge

- INDEX.md обязателен если > 3 файлов, с колонкой "Когда загружать"
- Один факт = одно место (grep перед созданием)
- Общий (workspace/) vs частный (agent/) — см. knowledge-ownership.md
- Файлы <= 300 строк
- evolving/ для самообучения (learnings, failures)

## Связи

- reports_to обязателен
- Все redirect пути валидны (проверить!)
- Нет пересечений зон с другими агентами
- Прописан в AGENTS.md
- Handoff через Task Brief (задача + контекст + результат)

## Проверка

- validate.sh после каждой сборки
- validate-office.sh для проверки всей системы
- DRY-RUN: 2-3 сценария за <= 3 шага
- Конвейер: Demiurg → Brahma → Cherry

## Запрещено

- Клонировать шаблон без уникальности
- Копировать файлы вместо ссылок
- Пропускать валидацию
- Data pack score < 5 → сборка заблокирована
