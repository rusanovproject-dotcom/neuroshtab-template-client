# `/marketer-revision` — Первичная ревизия маркетинга

Первый заход Маркетолога. Цель — **понять правду бизнеса**, найти **где бабки** сейчас и **куда можно двигаться**, выбрать **лучший сегмент ЦА** из 3, распаковать его в компактные артефакты, согласовать с клиентом.

Поведение и тон — из `office/agents/marketer/soul.md` + `core.md`. Читай их перед стартом.

---

## Шаг 0. Чтение проекта (первые 5 минут)

Читай что уже есть. **Не задавай вопросы клиенту пока не прочитал.**

**Обязательно:**
- `office/client-profile.md` — центральная карточка
- `projects/<main>/CLAUDE.md` или `README.md` — навигатор проекта
- `projects/<main>/diagnostic-card.md` — если Стратег распаковывал
- `projects/<main>/audience/` или `audience.md` — текущая ЦА
- `projects/<main>/offer.md` или `products/core-offer.md` — оффер
- `projects/<main>/positioning.md` — позиционирование
- `projects/<main>/strategy.md` — стратегия 6 недель

**По наличию:**
- `projects/<main>/journey.md`
- `projects/<main>/voice.md` или `brand/voiceprint.md`
- `projects/<main>/knowledge/cases.md`
- `inbox/` — транскрипты созвонов, касдевы, прислан материал

**Внутренняя библия агента:**
- `office/agents/marketer/knowledge/kadyrov-3w-bpsv.md`
- `office/agents/marketer/knowledge/hormozi-offer-market.md`
- `office/agents/marketer/knowledge/schwartz-awareness.md`

## Шаг 1. Краткий диагноз «что я вижу»

Напиши клиенту **живой дружеский заход** (по `soul.md`) + первые наблюдения в 3-5 пунктов. Формат не шаблонный — перефразируй под проект.

Пример:
> *«Короче. Прочитал что есть. Вот что вижу:*
>
> *1. Оффер продаёт [X], а клиент судя по касдевам покупает [Y]. Это разрыв — именно он и не даёт конверсии.*
> *2. ЦА описана как "эксперты 28-45". Это не ЦА, это перепись. Реальные клиенты — четверо, все из одной ниши [Z]. Это твой настоящий сегмент, остальное — гипотеза в вакууме.*
> *3. Бабки сейчас идут из [канал X]. Но на лендинге про этот канал — ни слова. Ты продаёшь не то, откуда реально приходят.*
>
> *Мне нужно 1-2 часа чтобы пройти по настоящему — задать вопросы, разобрать рынок если понадобится, выбрать сегмент и распаковать его. Погнали?»*

**Если клиент согласен** → Шаг 2. **Если не согласен / хочет обсудить** → обсуди, потом Шаг 2.

---

## Шаг 2. Вопросы клиенту (если что-то непонятно)

Максимум **7 вопросов**. По одному за раз. Живым тоном, не анкетой.

Классический набор (адаптируй под проект):

1. Что ты продаёшь сейчас — конкретный продукт, не сфера
2. Кто уже купил — 3 последних клиента: имя, профессия, почему купил
3. Какой результат ты обещаешь? В цифрах? За какой срок?
4. Откуда приходят клиенты сейчас (каналы трафика)?
5. Кого считаешь конкурентами? Минимум 3
6. Что клиенты говорят в отзывах — 3-5 **дословных** цитат (не «им нравится», а **их словами**)
7. Какая самая частая причина «нет» от потенциального клиента?

**Правило остановки:** если клиент на 3+ вопроса отвечает «не знаю» / «пальцем в небо» → **Шаг 3 (рынок)**. Иначе — Шаг 4.

## Шаг 3. Анализ рынка (если агент не уверен)

Запускается если:
- У клиента мало данных
- Есть противоречия в том что он говорит
- Гипотеза звучит сомнительно
- Клиент отвечает «не знаю» на критичные вопросы

**Три инструмента рынка:**

### 3.1 Таблица конкурентов (5 × 7)

| Параметр | К1 | К2 | К3 | К4 | К5 |
|----------|----|----|----|----|----|
| Название / URL | | | | | |
| Главный оффер (1 строка) | | | | | |
| Заголовок лендинга (дословно) | | | | | |
| Главное обещание | | | | | |
| Гарантия | | | | | |
| Цена / модель | | | | | |
| Главная слабость | | | | | |

Источники: WebFetch на их сайты, поиск YouTube-видео, Telegram-каналы конкурентов.

### 3.2 Voice of Customer — сбор 20+ дословных цитат

Источники:
- Reddit / Quora / профильные форумы
- YouTube-комментарии под видео конкурентов
- Отзывы на Wildberries / Ozon / Amazon (книги в нише)
- Telegram-группы по теме
- Отзывы клиента (если есть)
- Касдевы которые клиент прислал в `inbox/`

**Цель:** собрать **настоящий язык** ЦА — боли, желания, страхи, возражения **их словами**.

### 3.3 Gap Map

| Что говорят ВСЕ конкуренты (банальность) | Что НЕ говорит НИКТО (возможность) |
|------------------------------------------|-----------------------------------|
| ... | ... |

Минимум 5 строк. **Пустая правая колонка = нет дифференциатора**, возвращайся рыть глубже.

Сохрани результат в `projects/<main>/archive/market-reconnaissance-<YYYY-MM-DD>.md` для истории.

## Шаг 4. Генерация 10-15 сегментов по 3W

Используй методологию из `knowledge/kadyrov-3w-bpsv.md`.

**Процедура:**

1. Выписать 10-15 болей ЦА (много, не только очевидные)
2. К каждой боли → потребность
3. К паре (боль, потребность) → **What** (что покупает/делает)
4. К каждому What → **Why** (зачем)
5. К каждому Why → **Who** (кто этот человек)

На выходе — таблица из 10-20 потенциальных сегментов.

**Сохрани её в черновике:** `projects/<main>/archive/segments-all-<YYYY-MM-DD>.md`.

---

## Шаг 5. Фильтрация до 3 сегментов

Из всех сгенерированных выбери **3 разной теплоты**:

| Тип | Критерий | Пример |
|-----|----------|--------|
| **Горячий** | Точно знает проблему, ищет решение, готов платить (≥85%) | Те кто уже купил |
| **Тёплый** | Знает проблему, интересуется, не решился (≥60%) | Подписчики TG которые читают но не пишут |
| **Неочевидный/холодный** | Боль есть, не осознаёт или не приоритизирует (≥40%) | Другая профессия рядом с основной ЦА |

---

## Шаг 6. Starving Crowd тест — выбор победителя

Прогони **каждый из 3 сегментов** через тест Хормози (4 критерия, до 10 баллов каждый, максимум 40):

| Критерий | Вес | Что проверяем |
|----------|-----|---------------|
| **Massive Pain** | /10 | Боль сильная и частая? Проговаривают спонтанно? |
| **Purchasing Power** | /10 | Есть деньги платить премиум? |
| **Easy to Target** | /10 | Легко найти в 1-2 каналах? |
| **Growing Market** | /10 | Рынок растёт, не умирает? |

**Дополнительные критерии (qualitative):**
- Совпадает ли сегмент с текущей экспертизой клиента?
- Можем ли сформулировать оффер в 1 предложение под этот сегмент?
- Есть ли 10+ дословных цитат из этого сегмента?
- Готов ли клиент **отказаться** от остальных двух и сфокусироваться?

**Победитель** = сегмент с максимальным скором + YES на вопрос клиента о фокусе.

Если победитель не очевиден — вынеси развилку клиенту: *«У нас два финалиста, X и Y. Вот разница: [...]. Моя рекомендация — X, потому что [...]. Но выбор твой.»*

## Шаг 7. Распаковка выбранного сегмента в артефакты

Создаёшь **3 компактных файла** в `projects/<main>/`:

### Файл 1: `ICP.md` — one-pager идеального клиента

Максимум 300-500 слов. Вот шаблон:

```markdown
# ICP — [Название сегмента]

## Одна фраза
[Кто он — в 1-2 предложениях]

## Соцдем
- Пол / возраст / доход
- Профессия / роль
- География

## Awareness Level: [1-5]
## Market Sophistication: [1-5]

## JTBD (Jobs To Be Done)
Когда **[ситуация]**, я хочу **[мотивация]**, чтобы **[результат]**.

## Main Pain (дословно из Voice of Customer)
> *«[цитата]»*

## Dream Outcome
[Конкретный результат с метрикой и сроком]

## Триггер покупки
[Что происходит в жизни когда решает купить]

## Anti-Avatar (кого НЕ берём)
- ...
```

### Файл 2: `audience.md` — полная распаковка

Расширенный файл с БПСВ по Кадырову. Структура — в `knowledge/kadyrov-3w-bpsv.md` секция БПСВ.

- Соцдем
- Три слоя боли (если применимо)
- Потребности
- Страхи (Ты как продукт / Я как потребитель)
- Возражения (ценовые / ценностные / доверительные)
- 4 уровня осознанности + распределение тезисов по уровням
- Voice of Customer — 15-20 дословных цитат
- Пять проявлений боли в повседневности

### Файл 3: `hypotheses.md` — список гипотез для проверки

```markdown
# Гипотезы по сегменту [название]

## H1: [название] — уверенность [высокая/средняя/низкая]
- Что утверждаем
- На основании чего (цитаты, данные)
- Как проверим (конкретный тест)
- Критерий успеха/провала

## H2, H3, H4...

## Red Flags (что НЕ подтверждено — требует проверки перед решениями на деньги)
- ...
```

**Минимум 3 гипотезы.** Каждая с критерием проверки.

## Шаг 9. Отчёт клиенту + передача

Отчёт по схеме (адаптируй под проект, не копируй шаблоны):

1. **Что увидел** — 3-5 главных тезисов с конкретикой
2. **Gap Map** — что делают конкуренты, чего не делает никто (возможность)
3. **Рекомендованный сегмент** — с Main Pain дословно + Dream Outcome
4. **Starving Crowd скор** — почему этот сегмент, а не другие
5. **3 гипотезы** — что проверить до решений на миллион
6. **Следующий шаг** — что логично делать дальше

**Следующие шаги (выбор одного под ситуацию):**
- `/marketer-audience` — если нужна ещё глубже распаковка (например, есть ресурс на 5 интервью)
- `/marketer-ladder` — если ЦА ясна, пора собирать продуктовую лестницу
- `/marketer-funnel` — если лестница есть, нужна воронка
- Пауза, клиент идёт проверять гипотезы в рынке

## Guardrails

1. **Не пиши оффер / лендинг / КП в этом скилле.** Это только ревизия ЦА. Дальше — другие скиллы.
2. **Минимум 15 дословных цитат** перед финализацией сегмента. Меньше — возвращайся в рынок.
3. **Awareness Level обязателен.** Без диагноза уровня все дальнейшие артефакты — наугад.
4. **Не выбирай сегмент сам без согласования.** Рекомендация — да. Выбор — за клиентом.
5. **Не раздувай файлы.** `ICP.md` — 300-500 слов, не 3000. Если хочется больше — значит ещё не понял суть.
6. **Помечай гипотезы.** Всё что не проверено — `[ГИПОТЕЗА, уверенность X]`.
