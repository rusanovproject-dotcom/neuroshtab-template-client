# Рита — Хранительница офиса

Помощник по имени **Рита**. Хранительница AI-офиса. Следит чтобы твой офис не превратился в свалку через 3 месяца. Не пишет контент. Не делает офферы и стратегию. У неё одна работа — **порядок**: память агентов, структура папок, роутинг, качество скиллов, дубли в knowledge, повторяющиеся ошибки.

Думай о ней как о **кураторе музея**. Не громит, не выкидывает — спокойно ходит между экспонатами, видит где пыль, где экспонат сместился, где табличка устарела. Предлагает реставрацию, ничего не делает без твоего «да».

Зовёшь по имени — откликается: «рита, проверь офис», «рита, наведи порядок», «рита, полный аудит».

---

## Что внутри

- **6 архетипов уборщиков** — каждый ловит свой класс проблем: память (AutoDream), скиллы (frontmatter / triggers / size), роутинг (коллизии триггеров), структура (PARA / INDEX.md / symlinks), дубли в knowledge (Jaccard), паттерны ошибок (повторяющиеся фейлы → правила).
- **3 режима** — быстрый осмотр (5 мин read-only), лёгкая чистка (15 мин под approval), полный аудит (60 мин с оценкой 0–100).
- **Защита от over-cleaning** — никогда не удаляет файлы (только `_archive/<YYYY-MM-DD>/`), не трогает `.env*` / `*.pem` / секреты, не лезет в `overrides.md`. 10 железных правил перед каждым действием.
- **Citation rule** — каждое замечание со ссылкой `path/file.ext:LINE`. Без citation — это не finding, это сплетня.
- **«Looks bad but is actually fine»** — обязательная секция в каждом отчёте: что выглядит плохо, но трогать **нельзя** (load-bearing patterns). Защищает от тупой чистки.
- **История аудитов** — все отчёты копятся в `office/ops/audits/YYYY-MM-DD-<mode>.md`. Через месяц видишь как офис стал лучше.
- **Не отвлекает от денег** — если работаешь над Tier 1, аудит откладывает.

## Установка — 2 команды

В терминале (внутри своего AI-офиса):

```bash
git clone https://github.com/<your-org>/office-cleaner-pack.git _agent-packs/office-cleaner
```

В Claude Code:

```
/install-agent office-cleaner
```

Что произойдёт автоматически:
1. Файлы скопируются в `office/agents/office-cleaner/`
2. Скилл `/office-cleaner` появится в `.claude/skills/`
3. В `office/AGENTS.md` добавится строка про Риты
4. В корневой `CLAUDE.md` появится `@office/agents/office-cleaner/core.md`
5. Director получит роутинг — будет звать Риты на «проверь офис» / «наведи порядок» / «аудит»
6. Старый скилл `/audit-project` (если был) переедет в архив — его триггеры теперь у Риты
7. В `.claude/settings.json` появятся `deny`-правила на `.env*`, `*.pem`, опасные `Bash(rm:*)` — базовая защита для всего офиса
8. Создастся папка `office/ops/audits/` под отчёты

Руками править ничего не нужно.

## Как пользоваться

После установки — зови Риту по имени или пиши Director'у живым языком:

| Скажи | Что произойдёт | Сколько займёт |
|-------|---------------|----------------|
| **«рита, проверь офис»** / **«рита, наведи порядок»** | Быстрый осмотр — Рита посмотрит и скажет что хорошо, что просело. **Ничего не трогает.** | ≤5 мин |
| **«рита, почистим офис»** / **«лёгкая чистка»** | Осмотр + мелкие правки. Перед **каждым** изменением спросит твоё «да». | ≤15 мин |
| **«рита, полный аудит»** / **«что не так с офисом»** | Всё разложит по полочкам с оценкой 0–100 (Starter / Growing / Established / Optimized) и списком что улучшить. | ≤60 мин |

После отчёта — в режимах «лёгкая чистка» и «полный аудит» Рита спросит **что применять**:

| Ответ | Что сделает |
|-------|-------------|
| **«да»** | Применит все предложенные правки |
| **«только важное»** | Применит только P0 (критичное) и P1 (серьёзное) |
| **«1, 3»** | Применит только finding'и под номерами 1 и 3 |
| **«нет»** | Ничего не трогает, отчёт остаётся в `office/ops/audits/` |

**Без явного ответа — Рита не трогает ничего.** Это закон.

## Когда звать

- **Прошёл первый месяц** использования офиса — пора посмотреть как наполнено.
- **Установили несколько паков подряд** — проверить не пересеклись ли триггеры.
- **Память агентов растёт**, ощущение что «много мусора» — Memory Cleaner соберёт redundancy.
- **Чувствуешь что роутинг сбоит** — Routing Validator покажет где Director не знает кому передать.
- **Перед важной фазой работы** — убедиться что фундамент не дрейфит.

Рита **не нужна каждый день**. Раз в 1–4 недели — нормально.

```bash

git pull
```

В Claude Code:

```
/install-agent office-cleaner
```

**Твои данные не затрутся:**
- `overrides.md` (твои кастомизации) — остаётся
- `memory.md` / `failures.md` — остаются (накопленный опыт)
- `office/ops/audits/` — все прошлые отчёты сохраняются

Обновится только ядро: логика 6 архетипов, Florian-rubric, защиты, скилл `office-cleaner`.

## Если что-то не работает

- **«не могу установить»** — проверь что в офисе есть скилл `/install-agent` (приходит вместе с `client-office-template`). Если его нет — `git clone` всё равно сработал, ставь руками по `install.md`.
- **«Рита не откликается на имя»** — проверь что в `office/AGENTS.md` есть строка про Риту, а в `office/agents/director/core.md` есть роутинг на `/office-cleaner` (с триггерами «рита» / «позови риту»). Если нет — переустанови (`/install-agent office-cleaner`).
- **«Рита отвечает «офис тормозит, попроси git add сначала»»** — у тебя uncommitted changes. Аудит на грязной копии = ложные finding'и про дрейф. Закоммить всё (`git add . && git commit`) → запускай заново.
- **«Рита предложила снести что-то нужное»** — скажи «это load-bearing, не трогать». Она зафиксирует правило в `failures.md` и больше не предложит. Так она учится твоему офису.
- **«Третий аудит подряд показывает одно и то же»** — значит fix'ы не применяются. Рита сама спросит: «либо ты не одобряешь, либо мы в разном понимании что чинить — расскажи?». Расскажи где именно затык.
- **«Хочу запускать только определённые архетипы»** — пропиши в `overrides.md`: «не запускать M5 (Knowledge Dedup) — у меня специально дублируется». Рита увидит при следующем прогоне.

## Источники

Пак собран по 6-документному исследованию архитектуры AI-офисов (2025–2026):

- **Florian audit-prompt v5.0** — 4-фазная структура, 8-мерная rubric, tier'ы
- **ksimback Tech Debt Audit** — citation rule, «Looks bad but is actually fine», Phase 1 «Orient»
- **Compound Document Review** — 3-классовая классификация findings (P0/P1/P2)
- **Anthropic Memory Tool API** — immutability, append-only, audit trail
- **AutoDream** — правила memory cleanup (Redundancy / Contradictions / Stale Timestamps)
- **Microsoft Agent Governance Toolkit** — 10 OWASP Agentic Risks
- **Arize / MindStudio** — failure-pattern recognition

Это не «индустриальный канон» (он ещё не устаканился), а **сводка лучших практик** на момент сборки пака.
