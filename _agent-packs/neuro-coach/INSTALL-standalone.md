# Установка НейроКоуча

Коуч — это набор файлов, которые читает и исполняет **Claude Code**. Ничего не компилируется.

## 1. Положить папку

Склонируй репозиторий (или скопируй папку) туда, где работает твоя сессия Claude Code:

```bash
git clone <repo-url> neuro-coach
cd neuro-coach
bash scripts/bootstrap.sh   # создаёт рабочие файлы инстанса из шаблонов (идемпотентно)
```

`CLAUDE.md` в корне — точка входа: он объясняет коучу, кто он, как читается память и где роутер.

> **Движок и данные разделены.** В git живут только шаблоны (`*.template.md`). Твои личные файлы (`core/profile.md`, `goals.md`, `state.md`, метрики) создаются из них при первом запуске и лежат вне git — поэтому обновление движка (`git pull`) никогда не затрёт твои данные. `bootstrap.sh` можно не запускать вручную — онбординг сделает это сам первым действием.

## 2. Первая сессия — онбординг (живая сессия, ~30-40 минут)

Запусти Claude Code в папке коуча и скажи что-нибудь вроде «давай знакомиться».
Коуч (его зовут **Атлас**) войдёт в навык `skills/first-session.md` — это не анкета, а живой разговор:
- представится, по желанию прочитает, что о тебе уже есть в офисе, склеит картину;
- через образ «джинна» вытащит, что тебя цепляет: что надоело → чего по-настоящему хочешь → что мешает;
- договорится, как вы работаете, спросит, что тебе важнее от него, и зафиксирует контракт на 3 месяца;
- в конце назовёт три команды (вечерний/утренний/недельный чек) и предложит потом отдельно разложить цели;
- сильные стороны, как ломаешься, близкий круг **дозревают сами** по дням, не допросом.

Выдохся на середине — коуч свернёт по-доброму, остальное доберёт по ходу. После онбординга заполнятся `core/profile.md`, `core/drives.md`, `state.md` (контракт, таймзона, флаг `cold_start: true` на первые ~2 недели).

## 3. Ежедневный ритм

- **Вечером** — чек: наговори одним сообщением, как прошёл день. Коуч свернёт его в `tracking/daily/` + строку в `tracking/metrics-YYYY.jsonl`, задаст один живой вопрос, поможет выбрать 3 ключевые на завтра.
- **Утром** (если вечером не закрыл) — коуч догонит чек и поможет настроить день.
- **Стрик** виден каждый день — он растёт за сам факт чека.

## 4. Задачи: где они живут

По умолчанию `state.md` → `task_source: internal` — задачи живут **внутри коуча** (вчерашние «три на завтра» + `goals.md`). Ничего подключать не нужно.

Если у тебя уже есть своя база задач — поставь `task_source: notion` / `github` / `file`, и коуч будет брать оттуда кандидатов при утреннем планировании. Источник недоступен — тихо вернётся к `internal`.

## 5. Проактивный контур (опционально, требует планировщика)

Чтобы коуч **сам** окликал утром/вечером и ловил тишину, нужен планировщик на стороне транспорта:

- **Вход:** `claude --channels` — сессия слушает Telegram и отвечает.
- **Выход:** `openclaw cron` — по расписанию запускает Claude Code с навыком `proactive`, тот читает `state.md` + `tracking/metrics-*.jsonl` и решает: написать или промолчать (`HEARTBEAT_OK`).

Минимум — два cron-якоря (утро ~08:30, вечер ~21:30, в таймзоне человека) + триггер тишины. Времена и тихие часы — поля в `state.md` (`morning_ping_at`, `evening_ping_at`, `quiet_hours`, `vacation_until`).

Без планировщика коуч полностью рабочий, но **реактивный** — отвечает, когда ты пишешь сам.

## 6. Обновление коуча

Движок (навыки, роутер, душа, метод) развивается — подтянуть свежую версию, не трогая свои данные:

```bash
bash scripts/update.sh
```

Скрипт прячет локальные правки движка (если были), делает `git pull --rebase`, возвращает их и дозапускает `bootstrap.sh` (вдруг появились новые шаблоны). Твои профиль, цели, дневники и метрики — вне git, обновление их не касается, merge-конфликтов по ним не бывает.

> **Миграция для тех, кто ставил коуча до разделения движок/данные** (одноразово): после первого `git pull` твои `core/profile.md`, `goals.md`, `state.md` и т.п. станут локальными (untracked) — это нормально и безопасно, данные на месте. Дальше `update.sh` работает штатно.

## 7. Приватность

Память на твоём диске. Личные файлы (`core/profile.md`, `goals.md`, `state.md`, дневники, метрики) — в `.gitignore`, git их не видит: в репозиторий уезжают только пустые шаблоны (`*.template.*`). Дневники `tracking/daily/`, отчёты `weekly/`/`monthly/` и `_archive/` тоже исключены. Реальный коучинг веди в **отдельном клоне** от публичного шаблона.
