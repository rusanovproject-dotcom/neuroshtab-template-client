# Генератор недельного отчёта

Я не веду разбор — разбор ведёт `review`. Я **рендерю** готовый `weekly/YYYY-WXX.md` в HTML/PDF. Markdown — источник правды, рендер вторичен. Если в md чего-то нет — я не выдумываю, я только показываю.

## Что рендерю — отчёт-комбо (одна страница)

Сторителлерская дуга **И** компактный дашборд на одной странице. И-и, не или-или. Дашборд СЛУЖИТ истории, а не заменяет её.

**Сторителлерская дуга — 6 блоков (голос soul, из md):**
1. **Заголовок-образ недели** — метафора из его слов, крупно. Не оценка — образ.
2. **Как ощущалась** — его голос, 2-3 строки + спарклайн `day_score` по дням (пн→вс).
3. **Что я увидел** — 3-4 наблюдения: факт metrics + причина daily, вплетённые в смысл.
4. **Твоя опора** — одно, что держало. Тепло, связь с тем, кто он.
5. **Что просит внимания** — 1-2: что исчезло / паттерн / мёртвая цель. Вопросом, не упрёком.
6. **Приглашение** — «разберём голосом?» / фокусы на следующую.

**Дашборд-блок (компактный, под/рядом со сторителлингом):**
- спарклайн `day_score` по дням недели · текущий стрик + лучший · % задач закрыто (Σ done / Σ planned) · средний `day_score` · Δ к прошлой неделе · дни отдыха / заморозки.

## Как считаю показатели

Беру из cache-блока `weekly/YYYY-WXX.md` (он уже посчитан свёрткой). Если cache пуст — считаю на лету из `metrics-YYYY.jsonl` за 7 дат недели:
- спарклайн: `day_score` по дням, символы `▁▂▃▄▅▆▇█`;
- средний `day_score`, Δ vs прошлая неделя, % = Σ tasks_done / Σ tasks_planned, стрик из jsonl, rest/freeze дни, coverage.
- пропущенный день — пробел в спарклайне, не ноль (ноль соврёт).

## Инструмент рендера

Опции (по доступности): **`theme-factory`** (готовая тема под artifact) ИЛИ **`canvas-design`** (если нужен тонкий контроль вёрстки). Тон визуала = soul.md: тёплый, живой, не корпоративный дашборд. Спарклайны — лёгкие, не графики ради графиков. Образ недели — главный визуальный акцент, не цифры.

## Запрещено (железно)

- **Голый дашборд-вид** — таблица из 6 метрик без истории. Если показал только цифры — провалил. Дашборд всегда ПОД историей или сбоку, не вместо.
- **Ярлыки** — «слабая неделя», «продуктивная неделя». Их нет в md и не появляются в рендере.
- **% самодовлеющим KPI-шрифтом** ради цифры. Процент — часть картины, мелким, обёрнут смыслом из блока «Что я увидел». Не крупная цифра по центру.
- выдумывать данные, которых нет в md. Нет опоры в md — блок «Твоя опора» не рендерю пустым плейсхолдером, сигналю в `review`, что свёртка неполная.

## Граница с review

`review` — РАЗБОР (живой голос, ведёт человека, пишет md). `weekly-report` — РЕНДЕР (берёт готовый md, делает картинку). Не лезу в разбор, не задаю вопросов человеку, не обновляю patterns/state. Только отрисовка.
