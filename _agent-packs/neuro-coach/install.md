# НейроКоуч — install manifest

Машиночитаемые метаданные для установки пака через скилл `/install-agent neuro-coach`.
Скилл читает этот файл, копирует файлы коуча в офис, запускает bootstrap и вписывает строки-осведомлённости в конфиги офиса.

> ⚠️ **ВАЖНО — модель интеграции «указатель», не «слияние».** НейроКоуч — самодостаточная система со СВОИМ входом (`CLAUDE.md` коуча = его точка старта, 14 навыков, свой роутер, своё состояние и метрики). Он НЕ становится инлайн-агентом офиса. Поэтому:
> - **НЕ добавлять `@office/agents/neuro-coach/core.md` в layered include корневого `CLAUDE.md`** (иначе весь коуч грузится в КАЖДУЮ сессию офиса и дерётся с офисом за роутинг и память).
> - Director **НЕ роутит на коуча инлайн** — он только ЗНАЕТ, что коуч есть, и подсказывает: «для коучинга открой папку `office/agents/neuro-coach/` отдельной сессией».
> - Коуч живёт в офисе папкой, в которую заходишь сессией Claude Code, когда нужен коучинг. Личные данные коуча гитигнорятся его вложенным `.gitignore` — в офис-репо не утекают.

## Files to copy

Источник — `_agent-packs/neuro-coach/`, цель — `office/agents/neuro-coach/`. Копируется рекурсивно **всё содержимое КРОМЕ `install.md`** (этот манифест в агентскую папку не переносится). Включая `.gitignore` коуча (он нужен — вложенно игнорит личные данные относительно своей папки) и `INSTALL-standalone.md` (родная инструкция коуча на случай отдельной установки).

```yaml
files:
  - src: ./           # всё содержимое пака
    dest: office/agents/neuro-coach/
    recursive: true
    exclude: [install.md]
    preserve_if_exists:        # личные файлы ученика не затирать при переустановке
      - core/profile.md
      - core/drives.md
      - core/history.md
      - core/people.md
      - core/what-works.md
      - goals.md
      - state.md
      - tracking/metrics-2026.jsonl
      - tracking/daily/
      - weekly/
      - monthly/
```

## Post-copy step (обязательно)

После копирования — развернуть рабочие файлы инстанса из шаблонов (идемпотентно, существующие не трогает):

```bash
bash office/agents/neuro-coach/scripts/bootstrap.sh
```

Создаёт из `*.template.*` личные файлы коуча (`core/profile.md`, `goals.md`, `state.md`, метрики) — они лягут в `office/agents/neuro-coach/` и будут проигнорены вложенным `.gitignore` коуча. Без этого шага коуч тоже заработает (онбординг сам запустит bootstrap первым действием), но лучше сразу.

## Message to client (after install)

Живым языком, без IT-терминов. Скилл показывает это после успешной установки.

```
Готово. В офис встал личный коуч — его зовут Атлас.

Что он делает:
— каждый вечер сворачивает твой день за ~20 секунд (наговорил как прошло — он разобрал, задал один точный вопрос, помог выбрать 3 задачи на завтра)
— утром помогает настроиться, показывает «заряд» и стрик
— раз в неделю и раз в месяц — разбор «со стороны»: что сдвинулось, куда идёшь
— ставит цели из того, что реально болит, и держит их с тобой
— ловит спад и поддерживает; если совсем тяжело — честно направит к живому специалисту

Важное: коуч — отдельная штука со своим «мозгом», поэтому живёт в своей папке. Чтобы поговорить с ним — открой Claude Code в папке office/agents/neuro-coach/ и напиши «давай знакомиться». Он сам всё проведёт.

Твои записи (состояние, цели, дневник) остаются только у тебя на диске — наружу не уходят.
```
