**Файлы accountability (goals/commitments/decisions/open-loops) лежат в корне `coach/`**, не в подпапке — так в дереве §7.

## 2. Шесть режимов загрузки (роутинг)

> Фикс-контекст (§5) грузится ВСЕГДА. Таблица показывает ТОЛЬКО ДОБАВКУ сверх фикса. Фикс не дублируется.

| Режим | Когда | Добавка сверх фикса | Потолок добавки (ток.) |
|-------|-------|---------------------|------------------------|
| **NUDGE** | короткий пинок, эмоции нет, energy вчера ≥6 | metrics вчера + эмоция вчера | ~250 |
| **MORNING** | `/morning` ИЛИ 05:00–11:00 | goals(неделя) + open-loops + metrics 3д + итог вчера | ~700 |
| **EVENING** | `/evening` ИЛИ 19:00–24:00 с итогами | goals(неделя) + open-loops → пишет день + строку metrics | ~500 |
| **RECOVERY** | energy/meaning низкие 2-3 дня подряд ИЛИ резкая дельта вниз (НЕ по одной строке) ИЛИ тихий-спад в тексте | классификация (§4) → под тип: history + open-loops + people(если отношения) + ОДИН patterns-файл | ~900 |
| **CRISIS** | острый эмо-маркер «здесь и сейчас» | method-фреймворки + history + decisions + open-loops + people(если имя) | ~1100 |
| **DEEP** | «давай разберём», стратегия | весь core + history + people + patterns + goals + open-loops + method-фреймворки + окно daily | ~2200 |
| **WEEKLY** | вс вечер ИЛИ `state.last_weekly < current_week` | metrics 7д + итоги daily недели + прошлый weekly + goals | потолок 10K, усечение |

- Фикс уже содержит soul + method-ядро + profile + drives + what-works + commitments + state — поэтому в добавке EVENING не повторяется commitments/profile, в MORNING не повторяется profile/state.
- **Захват (CAPTURE)**, **first-session** и **GOALS (goal-setting)** — НЕ режимы, а процедуры (см. §8). В роутере помечены как процедуры. GOALS — opt-in по явному намерению поставить цели.
- **cold_start** — флаг, модифицирует ЛЮБОЙ режим («данных мало, не выдумываю паттерны»; первые ~2 нед активно раскапывает profile/drives/history).

**Приоритет при совпадении:** CRISIS > RECOVERY > WEEKLY > DEEP > MORNING/EVENING > NUDGE.

**Сквозные правила (поверх любого режима):**
1. Острая эмоция последнего дня всплывает ВСЕГДА (не только тренд).
2. open-loops по дате наступления — поднимаются проактивно.
3. Имя значимого человека (из people) ИЛИ реляционная лексика → подгрузка people, независимо от метрик.

## 3. Метрики — единая jsonl-схема с тройной градацией (главное разрешённое противоречие)

**Почему так:** стрик не должен зависеть от заполнения wellbeing-полей. Floor > ceiling — система переживает плохой день, если за 30 секунд закрыты только 5 FLOOR-полей. CORE и OPTIONAL обогащают картину, но их пропуск НЕ рвёт стрик.

| Градация | Поля | Правило |
|----------|------|---------|
| **FLOOR** (обязательно для стрика, 30 сек) | `day_score`(1-10), `check_done`(full\|min\|none), `tasks_planned`(0-3), `tasks_done`(0-3), `day_type`(normal\|rest\|freeze) | Закрыты эти 5 → чек засчитан → стрик считается. `day_score` — он же 1-я из 4 цифр Заряда. |
| **ЗАРЯД-цифры** (быстрый вход вечером, пропуск ≠ разрыв) | `energy`(0-10), `stress`(0-10), `focus`(0-10) | Спрашиваются ЯВНО четвёркой (+`day_score` из FLOOR). Топливо метрики **Заряд** (см. ниже). Сдвиг 2026-05-24 (F18): переведены из «обогащения» в опору Заряда. Floor не меняется. |
| **Из потока** (НЕ выпытывать, пропуск ≠ разрыв) | `bedtime`/`wake`("HH:MM"), `sleep_h`(число), `workout`(0/1) | `sleep_h`/`workout` извлекаются из надиктовки; `bedtime`/`wake` — вопрос утра про сон. Нет — `null`. Читатели: линза Тело, RECOVERY-тренд, корреляции свёртки. |
| **OPTIONAL** (по ситуации) | `anxiety`(1-10), `meaning`(1-10), `rested`(0-10) | `anxiety` — кризис-маркер ≥8, в Заряд НЕ входит. `rested` — выспанность (утро, 2-й вопрос про сон), в Заряд НЕ входит, калибрует личную норму сна. meaning не выпытывать каждый день. `focus_blocks` убран (→ `focus`). A+ = важнейшая из трёх (покрыта `tasks_planned/done`). |

> **Поля `mood` и `meditation` убраны (2026-05-23), `focus_blocks` — (2026-05-24, заменён на `focus` 0-10):** их никто не читал по-разному / появился лучший заменитель. Поле живёт, только если на него реагируешь по-разному (принцип §1). Сбор чисел без читателя = трекер в маске, прямо запрещён.

> **Метрика Заряд (charge), 2026-05-24 — F18.** `charge = (day_score + energy + focus + (10−stress) + sleep_score) / 5` → 0-100%. Пять равных компонентов (4 вечерних цифры + `sleep_score` из часов сна, горка с пиком = личная норма сна, дефолт 8ч). `charge` КЕШИРУЕТСЯ в строку, недельная свёртка пересчитывает (как стрик). `charge_week` — среднее по закрытым дням ISO-недели. Полная формула, контрольные точки, калибровка пика через `rested` — `tracking/metrics-schema.md` секция «Заряд» (источник правды). Наружу — «бак ~XX%», не формула, не датчик.

**Служебное:** `v`(версия схемы, нет поля = null) · `date`(ISO).

**Финальная строка jsonl (пример полного дня):**
```json
{"v":2,"date":"2026-05-23","day_score":7,"check_done":"full","tasks_planned":3,"tasks_done":2,"day_type":"normal","energy":7,"stress":4,"focus":6,"bedtime":"01:00","wake":"08:00","sleep_h":7,"rested":7,"workout":1,"meaning":8,"charge":6.9}
```

**Минимальный (floor) день:**
```json
{"v":2,"date":"2026-05-24","day_score":5,"check_done":"min","tasks_planned":3,"tasks_done":1,"day_type":"normal"}
```

**Заморозка (авто-строка):**
```json
{"v":2,"date":"2026-05-25","check_done":"none","day_type":"freeze"}
```

- Пропущенный день без freeze = нет строки → стрик рвётся.
- **Любая эмо-цифра — повод прочитать текст дня, не вывод.** meaning:8 при тексте «не понимаю зачем» → верим тексту.
- Все проценты/тренды/стрик — **производные, считаются на лету** из jsonl. Кешируются только: сводка в `weekly/`, стрик в `commitments.md`, `charge` в строке дня (истина — 5 полей, свёртка пересчитывает). Год читается целиком дёшево.
- Арифметика тренда: среднее за 7 дней + дельта vs прошлая неделя (по существующим строкам, с пометкой покрытия).

## 4. RECOVERY — классификация как гипотеза (данные ИЛИ тихий-текст)

RECOVERY включается ПО ТРЕНДУ, не по одной строке, двумя путями:
- **по данным:** `energy`/`meaning` низкие 2-3 дня подряд ИЛИ резкая дельта вниз (был 7-8 — упал в пол). Одна низкая строка после ровной недели — случайный день, НЕ повод лезть с классификацией;
- **по тихому-тексту (тир 2):** «выгорел», «ничего не хочу», «на нуле», «всё через силу», «нет сил», «вымотан» — истощение БЕЗ острого отчаяния (отчаяние → CRISIS). Текст перевешивает: прямо сказанный спад не ждёт трёх дней.

**При `cold_start: true`** (данных и тренда ещё нет) RECOVERY работает как **crisis-lite**: тёплый контакт + «вижу, тебя подсадило, что случилось?» — БЕЗ классификации по типам. Типы (ниже) включаются только при `cold_start: false`.

Коуч читает daily 3д + anxiety/meaning, различает тип как ГИПОТЕЗУ, проверяет вопросом (не клеит ярлык):
- **тело** (сон↓/энергия↓) → восстановление, ритм → `patterns/energy`
- **смысл** (meaning↓) → drives + history; отдых сделает хуже
- **тревога** (anxiety↑) → заземление + open-loops
- **отношения** (people/реляционная лексика) → про человека → `core/people`

**Водораздел anxiety (RECOVERY vs CRISIS):** RECOVERY по anxiety = **5-7** (повышенная тревога, спад); `anxiety ≥ 8` → уже **CRISIS** (острое). Совпадает с приоритетом CRISIS > RECOVERY и порогом маркеров §6.

> «Похоже, дело не в усталости, а в смысле — так?» Классификация = гипотеза, человек подтверждает.

RECOVERY vs CRISIS: **RECOVERY = я заметил спад (данные/тихий текст). CRISIS = он кричит сейчас (острый текст).**

## 6. Шесть линз + красная линия (фокус внутри режима, секция роутера, НЕ файлы)

Линза = сигнал + ОДИН patterns-файл/набор + ходы. Приоритет при двойном сигнале: тревога острее смысла (сначала заземление).

| Линза | Сигнал | Грузит | Ходы |
|-------|--------|--------|------|
| Тело/энергия | energy↓ / charge↓ / сон↓ / rested низкий при норме | patterns/energy | восстановление, ритм |
| Фокус/продуктивность | первая из 3 ключевых не закрыта (tasks_done<1) / focus↓ | patterns/productivity | приоритет, что убрать |
| **Смысл** (вкл. под-режим Идентичность) | meaning↓; «я не тот», «перерос» | drives + history | «зачем», ценности; в под-режиме — кто я, согласие жизни с собой |
| Отношения | имя из people ИЛИ реляционная лексика | people + open-loops | динамика с конкретным человеком |
| Решения | «не знаю что выбрать» | profile(как решает) + decisions | GROW, критерии, вынести выбор |
| Эмоц. регуляция | anxiety↑ | daily текст + open-loops | назвать, заземлить «здесь и сейчас» |

Разведение: Эмоции = регуляция состояния сейчас; Отношения = динамика с человеком. Деньги — НЕ линза (через эмоции/решения/смысл). Привычки — НЕ линза (commitments + тело/фокус).

**Маркеры острой эмоции → CRISIS** (единый источник; роутер подчинён этому списку): лексика отчаяния/слома — «всё бессмысленно», «не вижу смысла», «хочу всё бросить», «не могу больше», «сломался», «паника», «ненавижу», «всё зря» · резкое падение в тексте дня · `anxiety ≥ 8` ИЛИ `meaning ≤ 2` в строке metrics. Любой триггер → захват не пишет молча, поднимает CRISIS. (Тихий спад тир-2 и водораздел anxiety 5-7 vs ≥8 — в §4.)

**🔴 Этическая красная линия (выше всех линз и режимов):** при клинических маркерах — суицидальные мысли, намерение причинить вред себе, признаки тяжёлой клинической депрессии — коуч **НЕ коучит**. Переходит в escalation-to-human: тёплый контакт + прямое направление к специалисту (психотерапевт / кризисная линия). Не диагностирует, не «вытаскивает» сам. Детально — в `soul.md` + `skills/crisis.md` (этап 1).

## 8. Операционный хребет (по чему человек живёт день и неделю)

**§0 Главный закон — floor > ceiling.** Два независимых датчика, не смешивать:
- **СТРИК** = непрерывность практики ЧЕКА (контролируемо на 100%). Главное число, которое человек видит. Растёт за сделанный чек, НЕ за выполнение задач.
- **3 КЛЮЧЕВЫЕ ЗАДАЧИ** = отдельный счётчик эффективности (% закрытия, зависит от мира). Под капотом коуча, всплывают только в недельном разборе. На стрик НЕ влияют.

**§1 Ежедневный чек** (bookend: вечером закрываю, утром открываю если не закрыл):
- **Полный:** (1) балл дня 1-10 → (2) что хорошо / благодарность → (3) чего не хватило до 10 → (4) что завтра лучше + выгрузка + отметка X/3 + 3 задачи на завтра.
- **Минимальный (floor):** балл дня + 3 ключевые на сегодня.
- **Лестница вниз:** полный → минимальный → только балл (5 сек) → заморозка. Никогда не остаёмся с «ничего».
- В провальный день вопрос «чего не хватило» в лоб НЕ задаётся — floor + поддержка.

**§2 Три ключевые задачи** — MIT, самые ЭФФЕКТИВНЫЕ (не срочные). «1-3 по ситуации», три — потолок не норма. Бинарно (закрыта/нет), без подзадач/дедлайнов. Связь с `goals.md`(неделя) обязательна.

**§3 Стрик (производная из jsonl, кеш в commitments):**
```
check_done = full   → стрик +1
check_done = min    → стрик +1   (floor сработал)
day_type   = rest   → стрик +1   (отдых — выполненная задача дня)
day_type   = freeze → стрик СОХРАНЯЕТСЯ (пауза, не +1, не разрыв)
check_done = none И не freeze → стрик РВЁТСЯ → 0
```
Милстоуны 7/30/100 → триггер `celebrate`. Порвался → переобрамить, не реанимировать цифру.

**§4 Проценты/динамика** — аналитический контур КОУЧА, не человека. Человек видит одно число (стрик). 3 правила: в день максимум одна цифра · цифры только в недельном разборе, обёрнутые в смысл · не реагируешь на показатель — не показываешь. Анти-паттерн: ежедневный дашборд из 6 чисел.

**§5 Заморозка/отдых** — активный retention-инструмент:
- `rest` — осознанный отдых, стрик +1, заполняется минимально.
- `freeze` — пауза, стрик сохраняется, не заполняется, лимит 1/нед (мягкий сигнал, не стена). 2-я заморозка → мягкое «что происходит?» (вход в RECOVERY).
- Проактивная заморозка коуча: `last_capture > 1 день` + низкая energy → «два дня тихо, закрою заморозкой, стрик целый, продолжаем?».

**§6 Недельный контур + отчёт (комбо: сторителлинг + дашборд).** Контур: `current_week > last_weekly` → «не свернули прошлую»; `week_planned=false` — легитимное состояние (хаотик живёт без плана), спросил раз — ведёт по дням. Разбор из позиции **свидетеля** («смотри, что я ВИЖУ по твоей неделе»), не судьи.

⚠️ **Ежедневный контур НЕ трогаем (незыблемо):** человек видит ОДНО число — стрик. Ежедневного дашборда НЕТ. floor>ceiling на дневном уровне сохраняется. Комбо ниже — ТОЛЬКО про недельный отчёт, в ежедневный чек не протекает.

Недельный отчёт = **одна страница, где сторителлинг и дашборд комбинируются** (и-и, не или-или):
- **Сторителлерская дуга — 6 блоков, голос soul** (остаётся): 1. Заголовок-образ недели · 2. Как ощущалась (его голос + спарклайн баллов) · 3. Что я увидел (факт metrics + причина daily) · 4. Твоя опора · 5. Что просит внимания · 6. Приглашение.
- **ПЛЮС компактный дашборд-блок измеримых показателей** (под/рядом со сторителлерской частью): спарклайн `day_score` по дням недели · текущий стрик + лучший · % задач закрыто (`tasks_done/tasks_planned` за неделю) · средний `day_score` · Δ к прошлой неделе · дни отдыха/заморозки.

**Принцип комбинации:** дашборд СЛУЖИТ истории, не заменяет её. Цифры связаны с нарративом («закрыл две трети, и просадка — в недосыпные дни»), но теперь МОГУТ быть показаны и визуально (блок метрик/спарклайны), а не только вплетены текстом.

Тон = soul.md. **Дашборд-блок разрешён** как часть отчёта. Живые ограничения остаются: ярлыки («слабая неделя») запрещены; % не самодовлеющим KPI-шрифтом ради цифры — он часть картины, обёрнут в смысл.

**ТЗ генератору отчёта (этап 5):** рендерит И нарративную дугу (6 блоков), И дашборд-блок показателей — на ОДНОЙ странице. Тон = soul.md. Рендер HTML/PDF из `weekly/YYYY-WXX.md` (theme-factory / canvas-design).

**Процедуры (не режимы):**
- **CAPTURE** (захват голос/текст вне утра/вечера): транскрипт → дедуп(id) → append в daily + metrics. **Мини-плейбук эскалации:** прогнать текст по маркерам §6/острой эмоции → если матч → войти в CRISIS (НЕ append-silent). Иначе — тихо пишет день.
- **first-session** (онбординг): поток **A→M** (представление с самоиронией над именем Атлас → чтение офиса intake по согласию → 3 уточняющих → джинн→боль→дожим→желаемое→препятствие → контракт начистоту → счётчик «осталось 3» → «что хочешь от меня»+7 вариантов → опыт → большой контракт → завершение с 3 командами + передача в цели). Полноценная сессия, не «лёгкие 15 мин» (осознанный сдвиг 2026-05-24, F19). Drives (зажигает/ценности) вытягиваются из боли/желаемого, не из отдельного peak-момента. Wheel-снимок (одна фраза, если прозвучал; иначе пусто → goal-setting) + большой контракт. Объясняет методологию (стрик/чек/3 задачи/заморозка) как контракт практики. Выставляет `cold_start: true`, `onboarded_at`. **goals.md НЕ заполняет** — только роняет семя в goal-setting. Защиты: 🔴 микро-гейт красной линии на дожиме боли (E–H; красная линия ШАГ 0 сквозная и дальше), floor усталости (свернулся до J → счётчик «осталось три» не объявляется, укороченное M), голос soul (день 1 без мата, без «не X а Y»). Пост-шаг: cold-start review на ~14 день → дописать profile (как решает, рефлексия) → `cold_start: false`.
- **GOALS / goal-setting** (целеполагание, opt-in, потолок добавки ~900-1200 ток.): тот самый момент постановки каскада, которого в системе не было. Триггер — ЯВНОЕ намерение («давай поставим цели», `/goals`) или приглашение из онбординга / месячного разбора / celebrate. **МЯГКИЕ** триггеры («не знаю чего хочу», «куда я иду») — только при ровном состоянии: спад/апатия за фразой → RECOVERY/CRISIS первее (они выше в приоритете), не Колесо. Развилка ясность/туман → при тумане **Колесо разовой диагностикой словами** (баллы — опция, НЕ трекер, см. §10) → западающие сферы (предложить, выбирает человек) → проблематика (что болит, с микро-гейтом красной линии перед каждым дожимом) → цель из боли на год (его формулировка, не подсунутая) → декомпозиция год→**3 месяца**→месяц→неделя (week_focuses); минимум живого каскада год+3мес+неделя, остальное дозревает. Пишет `goals.md` целиком + `state.last_goals_review`. `big_contract` каноничен в state; goals.md «3 месяца» держит лишь его декомпозицию, не копию (без дублей). Каскад уже стоит → режим обновления (пересмотр, отпускание мёртвого → decisions). Красная линия первее: клиника на проблематике → CRISIS. НЕ в цепочке приоритета режимов (процедура, как first-session).

NB: сам `first-session` после онбординга не перезапускается — флип делает именно этот триггер, не онбординг-навык |
| profile.операционка (пики энергии) | 4+ нед | `MONTHLY`: confirmed-паттерн из patterns/energy → промоушн в profile |
| drives.защиты, drives.противоречие | 3-6 нед | сначала `patterns/`(hypothesis); навыки recovery/crisis/deep-session пост-шагом «зафиксируй обнаруженное»; подтверждено → промоушн в drives |
| history.срывы (новые) | после каждого спада | recovery/crisis пост-шагом «допиши что вытащило» |
| drives.Сильные стороны (топ-5) | ~14 день | **cold-start review** активным вопросом «на что опираешься, что вытаскивает» (единственный писатель). БЕЗ него поле пусто навсегда — а его читают celebrate и RECOVERY («на чём держался в худшую неделю»). Закрывает регрессию C3. NB: рабочие МЕХАНИЗМЫ из побед/спадов идут в `what-works` через `patterns` (celebrate/recovery) — это другое поле, не путать с характерными силами |
| drives.зажигает (новые темы) | со временем | курируемо: новое вытесняет угасшее (потолок), НЕ бесконечный append |

**Граница drives ↔ patterns:** гипотеза (защита/противоречие, не подтверждена) живёт в `patterns/`(hypothesis) → подтверждена наблюдением → промоушн в `drives`. Не два места сразу. `profile` — легитимный приёмник промоушна наравне с what-works (пики энергии).

**Пороги паттернов:** `hypothesis → confirmed` при ≥10 точек И ≥3 недели; `confirmed → core` при удержании 4+ недели; hypothesis без набора за 6 недель → `_archive` (TTL). Тип «гипотеза-о-человеке» = поведение в статусе hypothesis (не отдельная сущность).

## 10. Что НЕ делаем (анти-раздувание)

- Линзы — НЕ файлы, секция роутера. Фикс-контекст НЕ дублируется в строках роутинга.
- НЕ плодим сущности (гипотеза-о-человеке = поведение+hypothesis; coverage убран).
- Деньги/привычки — НЕ отдельные линзы.
- **Тесты-ассессменты НЕ собираем:** OCEAN/Big Five, MBTI/DISC/соционика, полное Колесо **как трекер** (повторяющаяся шкала в чеке), Ikigai (4 круга), детство/травмы (граница терапии), ценностные иерархии топ-20. **Берём:** ценности через peak/low (в ведении — даёт правду; в новом онбординге A→M день-1 ценности берутся 🔵-гипотезой из боли/желаемого E/G и подтверждаются peak/low-наблюдением позже, не выпытываются списком), VIA топ-5, Wheel снимок разово (state.wheel_baseline), Immunity to Change на «противоречие». **Уточнение (goal-setting):** Колесо по 8 сферам с баллами разрешено как **разовая диагностика прицела внутри opt-in сессии целеполагания** — это не трекер (не возвращается в ежедневный чек, не шкала-метрика). Запрет §10 — про Колесо-как-ежедневный-датчик; разовый прицел в целевой сессии ему не противоречит. floor>ceiling на дневном уровне не тронут.
- **Floor > ceiling:** стрик на чеке (контролируемо), проценты под капотом (не на лице ежедневно), заморозка — retention-инструмент. Не добавлять шкалы в чек, не делать ежедневный дашборд, не вводить подзадачи/дедлайны в тройку. Защищать минимализм даже от энтузиазма человека.
- Лимиты токенов/строк — жёстко. Превысил — режь, не оправдывай.
