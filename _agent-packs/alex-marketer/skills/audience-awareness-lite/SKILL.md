## 2. Архитектура — 5 шагов

### Шаг 1 — Subagent-классификатор: цитаты → 4 уровня (5-7 мин)

**Запускается через Task tool параллельно с подготовкой Шага 2.**

**Промпт subagent-классификатора:**

```
Ты — классификатор цитат по 4 уровням осознанности (адаптация Schwartz Awareness Levels).

## Контекст
Сегмент: {hot-slug}
Источник: voice-of-segment.md + segment-portrait.md (50-100 валидированных цитат)

## 4 уровня осознанности (русифицированная сборка от Schwartz, см. knowledge/schwartz-awareness.md)

| Уровень | Состояние ЦА | Маркеры в цитатах |
|---------|--------------|-------------------|
| **L1 Indifference** | Не знают что у них проблема. Живут «как все», списывают на усталость. | «у меня всё нормально», «бывает у всех», «это просто такой период», без proactive поиска решений |
| **L2 Awareness** | Знают проблему, не знают решений. Ищут описания «как у меня». | «упёрся в потолок», «выгораю», «не понимаю почему», но без упоминания категории решений |
| **L3 Consideration** | Знают категории решений (курс / коуч / AI / ассистент), сравнивают подходы. | «пробовал курсы — забросил», «думаю взять коуча», «нужно AI внедрить», но без конкретного выбора. |
| **L4 Choice** | Знают конкретных поставщиков, сравнивают «X vs Y». | «Like Центр vs Аяз vs Высоцкий», «нужен AI-наставник а не курс», «сравниваю Х и Y». |

⭐ Маркетинговая логика: каждый уровень требует **разной коммуникации** на лендинге:
- L1 → educational контент, диагностические тесты, симптомы (НЕ оффер)
- L2 → нарратив «было так — стало так», узнавание боли
- L3 → демонстрация **уникального механизма** vs привычные категории
- L4 → прямое сравнение, оффер, гарантия

## Задача

1. Прочитать все цитаты из:
   - `intel/{slug}/voice-of-segment.md` секции «Боли», «Желания», «Возражения»
   - `audience/segments/{slug}/segment-portrait.md` все цитаты с цветами 🟢🟡🟠

2. Классифицировать **каждую цитату** по уровню L1/L2/L3/L4 + статус «спорный» если не уверен.

3. Маркеры классификации (примеры):
   - «Топчусь на 200К» → L2 (есть боль, нет упоминания решений)
   - «Купил курс Like — не зашло» → L3 (знает категорию «курсы», пробовал)
   - «Like vs Visotsky vs наставник 1-1?» → L4 (сравнивает поставщиков)
   - «Иногда устаю но в целом норм» → L1 (списывает на «бывает»)

4. Output формат:

```markdown
# Awareness levels — {slug}

## L1 Indifference (N цитат, % от всех)
1. «{цитата}» — [источник] — маркер: {почему L1}
2. ...

## L2 Awareness (N цитат, %)
1. ...

## L3 Consideration (N цитат, %)
1. ...

## L4 Choice (N цитат, %)
1. ...

## Спорные (требуют валидации клиентом)
1. «{цитата}» — почему спорная: между L2 и L3 (нет однозначного маркера) — варианты: L2 / L3
2. ...

## Сводка распределения

⭐ Маркетинговая интерпретация:
- Доминирующий уровень: {L?} — {N% цитат}
- Это значит ЦА сейчас **массово находится** в состоянии {описание}
- Точка входа в воронку должна быть на этом уровне
- Lift до L4 (готовность купить) требует {описание перехода}

⭐ Распределение по типам цитат:
- Боли: L1 X% / L2 Y% / L3 Z% / L4 W%
- Возражения: L1 / L2 / L3 / L4
- Желания: L1 / L2 / L3 / L4

⭐ Ключевые маркеры перехода между уровнями:
- L1 → L2: «осознал что у меня плато»
- L2 → L3: «начал искать "как пробить потолок"»
- L3 → L4: «понял что курсы не работают, нужно X»
```

**Acceptance subagent-классификатора:**
- 100% цитат распределены или помечены «спорные»
- Доминирующий уровень определён
- Маркеры перехода между уровнями явно прописаны
- Спорных не больше 20% (иначе классификатор слабый, нужен другой подход)
```

### Шаг 2 — Валидация спорных цитат через AskUserQuestion (5 мин)

После возврата subagent-классификатора → клиент валидирует **только спорные** (≤20% от всех).

```yaml
question: |
  🤔 Я разложил {N} цитат по 4 уровням осознанности.
  {M} цитат спорные — нужен твой выбор. Помоги:
  
  «{спорная цитата 1}» — это L2 (понимает боль, но не решений) или L3 (думает про категорию решения)?

multiSelect: false
options:
  - id: l1
    label: "L1 Indifference (не понимает что проблема)"
  - id: l2
    label: "L2 Awareness (понимает боль, не знает решений)"
  - id: l3
    label: "L3 Consideration (знает категорию решений, сравнивает подходы)"
  - id: l4
    label: "L4 Choice (сравнивает конкретных поставщиков)"
```

**Цикл по спорным цитатам.** Если их >10 — спрашиваем «давай я сам определю по контексту, доверяешь?» — клиент либо да, либо проходим первые 5 для калибровки + остальное автоматом.

⭐ **Маркетинговая логика:** клиент не должен валидировать ВСЕ цитаты — это перегруз. Только **спорные** + первые 5 как калибровка автоматического решения по остальным.

### Шаг 3 — Lingvo-маркеры по 4 правилам (10 мин)

⚠️ **ИСТОЧНИКИ ОБЯЗАТЕЛЬНЫ** (правка по reviewer P0):

```bash
# Pre-flight Шага 3:
[ -f "intel/{slug}/competitors-map.md" ] || STOP "Phase B competitors-map.md отсутствует. Антипаттерны не из чего собрать."
ls inbox/_raw/refusals/*.md 2>/dev/null || WARN "Нет файлов отказов клиентов. Антипаттерны будут только из конкурентов."
```

⚠️ **4 правила НЕ схлопываем в 2** (правка по маркетинг-reviewer P0):
- Якорь и триггер — функционально РАЗНЫЕ:
  - Якорь = слово которое **клиент сам произносит** и узнаёт себя («упёрся в потолок» — узнавание)
  - Триггер = слово которое **запускает эмоцию** у читателя на лендинге («ты тоже бьёшься головой» — реакция)
- Метафоры и антипаттерны — тоже разные:
  - Метафоры = образы для нарратива (использовать)
  - Антипаттерны = слова которые звучат шумом / AI-слопом для этой ЦА (НЕ использовать)

**4 отдельных мультиселекта** (по 10-12 опций каждый):

**Lingvo = слова и обороты которые ЦА использует** — это материал для копирайтера. **4 правила Lingvo (см. `knowledge/audience-framework.md` секция «БПСВ»):**

| Правило | Что | Источник |
|---------|-----|----------|
| **1. Якоря** | Слова которые сразу включают узнавание («упёрся в потолок», «белка в колесе») | Из лингво-маркеров segment-portrait.md GOLD-статуса |
| **2. Триггеры** | Слова вызывающие эмоцию (страх / надежду / гнев) | Из цитат болей с самой высокой evidence_weight |
| **3. Метафоры** | Образы для нарратива («театр одного актёра», «ремесленничество vs конвейер») | Из секции «Метафоры» segment-portrait.md |
| **4. Антипаттерны** | Слова которые звучат шумом / AI-слопом для этой ЦА — НЕ использовать | Из конкурентной карты (что все говорят но никого не цепляет) |

**Алекс собирает 4 списка:**
- 10-15 якорей (с evidence_count)
- 5-7 триггеров (с эмоцией)
- 5-10 метафор (валидированных)
- 5-10 антипаттернов (что НЕ говорить)

**Валидация через AskUserQuestion:**

```yaml
question: |
  🗣 Финальная валидация lingvo-маркеров для копирайтера.
  Я собрал 4 списка из voice-of-segment.md + Phase C валидации.
  Подтверди что попадает / убери / допиши.

multiSelect: true
header: "Якоря — слова которые сразу включают узнавание"
options:
  - id: anc-01
    label: "«упёрся в потолок» (8 источников + клиент)"
  - id: anc-02
    label: "«белка в колесе» (Forbes + 4 цитаты + клиент)"
  ...
```

(И так для триггеров, метафор, антипаттернов — 4 быстрых мультиселекта подряд.)

### Шаг 4 — Dream Outcome (5-10 мин) + ANTI-SLOP GATE

⚠️ **MUST-FIX: Dream Outcome anti-slop gate** (правка по reviewer P0):

После 3-этапной сборки (кейс + голос эксперта + желания) — **обязательная проверка через subagent**:

```
Subagent prompt:
"Прочитай Dream Outcome: '{формулировка}'.
Один вопрос: эту фразу клиент-эксперт СЛЫШАЛ от своих ЦА дословно (или близко) — 
ИЛИ это интерпретация желания, его собственная формулировка?

Источники для проверки:
- voice-of-segment.md секция 'Желания' (есть ли близкие цитаты)
- inbox/_raw/transcripts/ (упоминалось ли в диалогах)
- segment-portrait.md FROM CLIENT цитаты

Ответ:
- VALIDATED → есть совпадение в источниках (привести цитату)
- HYPOTHESIS → только интерпретация (звучит правильно, но в источниках не подтверждено)"
```

**Если HYPOTHESIS** — Dream Outcome идёт в `hypotheses.md` под тегом `[OFFER]`, а **не** в финальный `segment-core.md`. Алекс честно говорит клиенту:

> *«Слушай, я сформулировал — звучит хорошо, НО в реальных цитатах ЦА такого нет. Это твоя интерпретация желания, не их голос. Пометил как гипотезу — проверим на первых лендингах. Если нужно прямо сейчас Dream — давай возьмём дословную цитату желания из voice-of-segment.md, она будет менее красивая но более точная.»*

⚠️ **Без anti-slop gate Dream Outcome уйдёт «масштабируешься без выгорания через AI-офис» — это AI-слоп, не язык ЦА.**

⭐ **Маркетинговая логика:** Dream Outcome — это **главное обещание лендинга**. Должно быть в **языке клиента** (а не Алекса) и опираться на голос **эксперта-владельца** (а не ЦА).

Источники:
- `inbox/_raw/cases/` — кейсы клиента (что **обещано** в результате конкретного клиентского кейса)
- `expert-bank.md` (если есть) — голос владельца как он сам формулирует обещание
- `voice-of-segment.md` секция «Желания» — что **просит** ЦА

**3-этапная сборка Dream Outcome:**

1. **Из кейсов:** «У {клиент Лена} было {до}, стало {после} за {срок}, плюс {бонус}» → формат твёрдого результата с цифрой
2. **Из голоса эксперта:** одна фраза которой клиент сам обозначает миссию проекта
3. **Из желаний ЦА:** что именно мечтают (не «больше денег», а «масштабироваться без выгорания»)

**Финальный Dream Outcome — 1-2 предложения:**

```markdown
## Dream Outcome — обещание hot-сегмента

> *«За 90 дней я довожу эксперта 200К/мес до **3 продающих созвонов в неделю с конверсией 40%** — без выгорания, без ассистентов, через AI-офис который работает руками за тебя. Это не курс по нейронкам. Это готовая система, которая отражает твой стиль работы и масштабируется без потери качества.»*

**Источник:**
- Цифры: кейс Лены (200К → 320К за 3 мес)
- Стиль: голос владельца из expert-bank.md
- Формулировка боли: «выгорание + масштаб» — топ-2 кластер из voice-of-segment.md
```

**Stop+wait:** клиент валидирует Dream Outcome или предлагает свою формулировку.

### Шаг 5 — Anti-avatar (5 мин)

Anti-avatar = **кого НЕ берём** в клиенты. Без него лендинг пытается продать всем → не продаёт никому.

**Источники:**
- `inbox/_raw/refusals/` — отказы клиента (кто не купил и почему)
- Phase A вопрос про «кого ты НЕ зажигаешь» (если был)
- Конкурентная карта — кого они забирают что нам не подходит

**5 типов anti-avatar** (расширено по reviewer P0 — для русского коучинг/AI-рынка):

⚠️ **Перед мультиселектом** — Алекс объясняет почему это важно:

> *«Кого отсекаем — это половина оффера. Если возьмёшь не своего, выгоришь и не получишь денег. Сейчас разметим 5 типов "не наши".»*

**5 типов:**

1. **Токсики / манипуляторы** — «всё должны мне». Сожрут энергию команды.
   Дисквалификатор: *«Если приходишь с настроем "вы мне всё должны" — не по пути»*

2. **Не наша стадия** — клиническое выгорание / тревога / депрессия → к врачу, не в программу.
   Дисквалификатор с этикой: *«Если в острой фазе — сначала к специалисту»*

3. **Без основы** — нет 5+ клиентов / нет кейсов / нет экспертизы.
   Дисквалификатор: *«Если меньше 5 проведённых клиентов в теме — рано, продукт ускоряет существующее»*

4. **Не наша скорость** — хочет результат за 7 дней, у нас 90.
   Дисквалификатор: *«Если ждёшь магии за неделю — мы про 90 дней методично»*

5. **Не своими руками** — хочет «сделайте за меня под ключ».
   Дисквалификатор: *«Программа требует твоего вовлечения — не подойдёт если ищешь "белые перчатки"»*

**Опционально 6-й тип для инфобиз/AI-ниш:**

6. **Серийный покупатель курсов** — берёт всё подряд, не внедряет, потом возвращает деньги.
   Дисквалификатор: *«Это не очередной курс. Если уже 3+ курса не внедрил — посмотри сначала почему, потом возвращайся»*

```yaml
question: |
  🚫 Кого мы НЕ берём в эту программу? Помоги собрать дисквалификаторы для лендинга.
  Я собрал 5 типов из отказов и интуиции — отметь которые точно подходят и допиши свои.

options:
  - id: anti-01
    label: "Токсик / манипулятор — сожрёт энергию команды"
  - id: anti-02
    label: "В клинической стадии выгорания / тревоги — нужен врач, не программа"
  - id: anti-03
    label: "Без 5 проведённых клиентов в теме — рано, продукт ускоряет существующее"
  ...
```

### Шаг 6 — Hero-формула (отдельно от Dream Outcome) — ДОПОЛНЕНО

⚠️ **MUST-FIX**: Hero-формула != Dream Outcome (правка по маркетинг-reviewer P0).

**Различие:**
- **Hero-формула** = первый экран лендинга = **узнавание боли** (Schwartz: Problem-Aware)
- **Dream Outcome** = главное обещание программы (Schwartz: Solution/Most-Aware)

Их **всегда путают** на лендингах — поэтому держим явно отдельно в `segment-core.md`.

**Сборка Hero-формулы (3 шага):**

1. Взять **топ-1 GOLD-VIP цитату** из `voice-of-segment.md` (single-source но >200 лайков на vc.ru / >500 на YouTube)
2. Если GOLD-VIP нет — взять **топ-3 GOLD цитаты болей** + найти общий смысловой паттерн
3. Сформулировать в формате:
   - *«[узнаваемая боль дословно] — но это не ты, это [система которой нет]»*
   - Длина: 1-2 предложения max
   - Schwartz уровень: Problem-Aware (не объяснять решение, только разрезать боль)

**Пример (для сегмента «Эксперт в потолке»):**

```
Hero-формула:
> «Год назад мне казалось что просто нужно ещё один курс. Я прошла три. 
>  Купила Notion, GetCourse, ChatGPT-плюс. Подписалась на пять бизнес-коучей. 
>  И всё равно — каждый понедельник просыпаюсь с одной мыслью: 
>  "опять не понимаю за что браться". Мой потолок 200К.
>  
>  Если ты узнаёшь себя — это не ты. Это система которой у тебя нет.»

Источник цитаты: GOLD-VIP из vc.ru комментов под статьёй о выгорании эксперта (487 лайков)
Schwartz уровень: Problem-Aware → переход в Solution-Aware на следующем экране
```

**Dream Outcome** записывается отдельно в `segment-core.md`:

```
Dream Outcome (главное обещание программы):
> «За 90 дней доведу эксперта 200К до 3 продающих созвонов в неделю с конверсией 40%»

Schwartz уровень: Solution-Aware → Most-Aware (что они получат)
```

## 5. Anti-patterns

❌ **Запустить Phase D для всех 3 сегментов** — overkill. Только hot. Warm/cold остаются на уровне Phase C (это **достаточно** для Stage 2 копирайта).

❌ **Заставить клиента валидировать все 50 цитат** — перегруз. Subagent делает 80%, клиент валидирует только спорные ≤20%.

❌ **Dream Outcome без цифры и срока** — «станешь успешнее» это AI-слоп. Только твёрдые результаты («3 продающих созвона в неделю с конверсией 40% за 90 дней»).

❌ **Anti-avatar в стиле «всех берём»** — Lендинг продаёт всем = никому. Минимум 3 чёткие дисквалификации.

❌ **Lingvo-якоря которые клиент не отметил** — это не якоря для **этой** ЦА. Используем только GOLD статус (интернет + клиент валидирует).

❌ **Произносить «awareness levels / Schwartz / classifier subagent / Dream Outcome»** в чате клиенту — жаргон. Клиенту: «уровни понимания», «главное обещание».
