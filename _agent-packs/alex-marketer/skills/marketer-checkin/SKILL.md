# `/marketer-checkin` — Периодический ревью маркетинга

Живой пульс маркетинга. Агент сам обнаруживает паттерны, зависшие гипотезы, дыры в customers.

## Когда запускается
- Раз в 7 дней (если клиент активен)
- По запросу клиента
- При 3+ новых карточках в customers.md с последнего чек-ина

## Шаги

### 1. Прочитай актуальное
- `projects/<main>/hypotheses.md` — все Active + последние Validated/Invalidated
- `projects/<main>/customers.md` — Segment Reality + отказы + агрегаты
- `memory.md` — что делал в прошлом чек-ине

### 2. Прогон Active Hypotheses
Для каждой из ≤5 активных:
- **Дедлайн прошёл?** → закрой как Validated или Invalidated по данным
- **Больше 14 дней без движения?** → спроси клиента *«Закрываем или продолжаем?»*
- **Данные есть, но частично?** → обнови прогресс + уверенность

### 3. Проверь Key Hypothesis
- Она всё ещё главная? Или появилась сильнее?
- Key = максимальный impact × confidence
- При смене Key — явно выноси клиенту: *«Было Key — H12. Сейчас H15 сильнее, потому что [X]. Переключаем?»*

### 4. Паттерны в customers.md
Проверь:
- 3+ покупки одного сегмента → новая `[PRODUCT]` гипотеза на фокус?
- Средняя actuality упала ниже 7 → **оффер требует пересборки** (выноси как проблему №1)
- 3+ отказа одной причины → новая `[OFFER]` гипотеза на закрытие дыры?
- Повторяется «что не хватает до 10» → системная дыра в оффере

### 5. Синхронизация с файлами воронки
Посмотри на валидированные за период гипотезы — обновил ли ты связанные файлы?
- `[PRODUCT]` ✅ → ICP.md, audience.md, offer.md
- `[CHANNEL]`/`[LM]`/`[FUNNEL]` ✅ → funnel.md
- `[MESSAGE]` ✅ → voice.md
- `[POSITIONING]` ✅ → positioning.md

Если что-то не обновлено — обнови.

### 6. Key Learnings
Паттерны подтвердились 3+ раз → переноси в `memory.md` секция Key Learnings (переносятся между проектами).

### 7. Отчёт клиенту

Живой формат (не список как в отчёте):

> *«За неделю: [X] новых покупок, средняя actuality [Y]/10 (тренд ↑/↓/=). Валидировал H[N], провалил H[M]. Key сейчас — H[K]. Замечу паттерн: [наблюдение]. Предлагаю [одно конкретное действие]. Что дальше?»*

### 8. План следующих 7 дней
1-2 приоритетных действия на ближайшую неделю. Не больше — чтобы не распыляться.

## Quality Gate
- [ ] Все Active hypotheses проверены на дедлайн/движение
- [ ] Key Hypothesis переподтверждена или переключена
- [ ] Паттерны customers.md проанализированы (4 триггера)
- [ ] Все валидированные за период → связанные файлы обновлены
- [ ] Key Learnings перенесены в memory (если 3+)
- [ ] План на следующие 7 дней чёткий (1-2 действия)

## Выход
Короткий отчёт + 1-2 действия на неделю + (опционально) предложение запустить `/marketer-audience` / `/marketer-ladder` / `/marketer-funnel` если данные требуют углубления.
