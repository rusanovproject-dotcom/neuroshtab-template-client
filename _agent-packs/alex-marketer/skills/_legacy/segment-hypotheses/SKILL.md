# /segment-hypotheses — Гипотезы сегментов из ответов клиента

## Зачем

Money-map (`/segment-money-map`) показывает «где деньги в нише» с конкурентной разведкой.

Этот скилл отвечает на другой вопрос: **«какие сегменты УЖЕ видны в данных самого клиента?»** — без конкурентов, без интернета. Только из его ответов и его файлов.

Это важный противовес. Money-map смотрит наружу — что в нише. Этот смотрит внутрь — что у самого клиента в реальности.

---

## Когда запускать

Внутри `/audience-stage` — параллельно с `/segment-money-map`, после двойного интервью клиента. Использует:
- Ответы клиента на 3 базовых + 2-3 уточняющих вопроса
- `projects/<main>/diagnostic-card.md` Стратега (если есть)
- `projects/<main>/audience/*.md` если есть наработки Стратега
- `projects/<main>-audience/audience/voc.md` уже собранные цитаты ЦА
- Транскрипты существующих покупателей в `inbox/_raw/transcripts/` или `audience/research/transcripts/`
- Карточки клиентов из `clients/` (если есть)

## Алгоритм

### Шаг 1 — Сбор сырых сигналов

Из всех источников выписать **сырые сигналы**:
- Дословные боли (цитата)
- Дословные желания (цитата)
- Дословные возражения
- Что куплено реально (тариф / продукт)
- Откуда пришли (канал)
- Демография где упомянуто

Без интерпретации — чистые сигналы.

### Шаг 2 — Кластеризация по паттернам

Сгруппировать сигналы по **общим паттернам**:
- Похожие боли = один кластер
- Похожие желания = один кластер
- Похожие профили (демография + канал прихода) = один кластер

Перекрытия кластеров = потенциальный сегмент.

### Шаг 3 — Формулирование гипотез сегментов

Для каждого устойчивого кластера — гипотеза сегмента:

```markdown
### Гипотеза сегмента: {имя-slug}

**Тип:** [ICP / SEGMENT]

**Описание:** 1-2 предложения кто это.

**Основная боль (цитатой):** «...»

**Главное желание (цитатой):** «...»

**JTBD (Christensen):** «Когда {ситуация-триггер}, я хочу {мотивация — что хочу почувствовать/сделать}, чтобы {прогресс — какой результат}.»

Пример: «Когда я выгораю на 3-й неделе подряд из-за объёма команды, я хочу разгрузиться без потери контроля, чтобы вернуть себе вечера и стратегические задачи.»

**Что покупает:** какой тариф / продукт

**Канал прихода:** откуда обычно приходит

**Демография гипотеза:** возраст / стадия / доход

**Подтверждено:** N сигналов из {источники}
**Уверенность:** 0-100%

**Открытый вопрос для проверки:** что нужно валидировать чтобы поднять confidence
```

### Шаг 4 — Anti-avatar

Из отказов и не-покупателей вывести **anti-avatar** — кого НЕ берём:
- Кто отказался и почему
- Кто пришёл но не подошёл

```markdown
## Anti-avatar

- Профиль X — почему не подходит
- Профиль Y — почему не подходит
```

### Шаг 5 — Структурированный вывод

```markdown
## 5-7 гипотез сегментов (из данных клиента)

### Гипотеза 1: {slug-1}
{поля выше}

### Гипотеза 2: {slug-2}
...

## Anti-avatar
{...}

## Где у клиента уже есть подтверждение
(сегменты которые УЖЕ покупают — приоритет)

## Где гипотеза «вычисленная»
(сегменты которые мы предполагаем по логике, без прямых данных)
```

## Output

```yaml
output_file: projects/<main>-audience/intel/segment-hypotheses-{date}.md
returns:
  - segment_hypotheses: [{ slug, description, main_pain, main_desire,
                          confidence, source_signals, validated_or_predicted }]
  - anti_avatar: [...]
  - top_recommendation: "сегмент-X — самый сильный по подтверждению"
```

Возвращает 5-7 гипотез + краткое резюме для Алекса (4-7 предложений) — **самый сильный сегмент по подтверждению из существующих данных**.

---

## Anti-patterns

- ❌ **Гипотезы из головы** — все должны опираться на реальные сигналы из файлов клиента или его ответов
- ❌ **Перефразированные цитаты** — только дословно, как сказал клиент или его покупатель
- ❌ **Игнор существующих покупателей** — если есть кто реально купил, эти кластеры в приоритете
- ❌ **Без open questions** — каждая гипотеза должна иметь критерий валидации
- ❌ **Лезть в интернет** — это `/segment-money-map`, разный скилл
- ❌ **Гипотеза без JTBD** — это плоский профиль (демография + боль), а не job. Без JTBD оффер собирается вслепую — нет привязки к ситуации триггера.

## Время

~3-7 мин синтез из файлов. В фоне параллельно с money-map.

Если файлов мало (только diagnostic-card, нет транскриптов) — работа выполняется быстрее, но confidence ниже.

---

## Lite-режим

Если из источников **меньше 3 уникальных сигналов** — выдать только 2-3 гипотезы вместо 5-7 + явно эскалировать: «недостаточно данных, нужны транскрипты или интервью с покупателями».
