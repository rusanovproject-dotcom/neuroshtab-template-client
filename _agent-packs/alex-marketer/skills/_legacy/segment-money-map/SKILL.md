# /segment-money-map — Money-карта рынка по сегментам

## Зачем

Старый `/competitors-research` отвечает на вопрос «кто конкуренты и что у них на лендингах». Это **продуктовый** анализ.

Этот скилл отвечает на другой вопрос: **«В каких сегментах ниши клиента максимум денег НА МАССЕ?»**. Это **денежный** анализ для холодной воронки и масштаба.

⚠️ **КРИТИЧЕСКОЕ правило — не на тёплых.** Money-map ищет **где деньги в массе рынка**, не «где деньги среди тех кто уже купил». Тёплые покупатели = ошибка выжившего (пришли по сарафану/связям, не из массы). Их боли используем как **сигналы паттерна** при формулировке гипотез сегментов, но **размер рынка / средний чек / конкуренция** считаем по большому рынку, не по 8 знакомым.

Разница принципиальная. Money-map даёт ответы:
- Какой сегмент в нише большой (есть масса)
- Какой сегмент платит много (high-ticket)
- Какой сегмент прибыльный для конкурентов (LTV × marketing efficiency)
- Где есть «голубой океан» по деньгам (мало игроков, есть платёжная способность)

Не «у конкурента X 5 фич», а «сегмент-A — рынок ~N чел, средний чек Y₽, конкуренты-лидеры делают Z₽/мес».

---

## Когда запускать

Внутри `/audience-stage` — параллельно с `/segment-hypotheses`, **после** того как Алекс провёл двойное интервью с клиентом (3 базовых + 2-3 уточняющих вопроса) и понял:
- Что у клиента за продукт
- Кого он целит (хотя бы в общих чертах)
- Главную боль ЦА (по версии клиента)

Без этого контекста скилл не запускается — у него нет рамки для поиска.

## Вход

```yaml
inputs:
  niche: "ниша клиента"  # например "AI-внедрение для предпринимателей"
  client_target: "кого клиент целит"  # из ответов клиента
  geo: "RU/CIS" | "US/EU" | "GLOBAL"  # рынок для разведки
  client_pricing: "<min — max ₽ от клиента>"  # ценовой коридор клиента (для калибровки)
  hypotheses: ["сегмент-A", "сегмент-B"]  # предварительные гипотезы из ответов
```

## Алгоритм

### Шаг 1 — Разбить нишу на 4-7 потенциальных сегментов

На основе ниши + ответов клиента — накидать гипотетические сегменты для разведки. Например для «AI-внедрение для предпринимателей»:
- Соло-эксперты в выгорании
- B2B SMB до 50 человек
- Корпорат >500 человек
- Реселлеры/партнёры (хотят сами оказывать)
- Курсы для самостоятельного внедрения
- Агентства которые хотят разгрузить клиентских эксп.

### Шаг 2 — По каждому сегменту собрать через WebSearch

| Метрика | Что искать | Пример запроса |
|---------|------------|-----------------|
| **Размер рынка** | сколько таких людей в гео | «количество соло-предпринимателей Россия 2026» |
| **Средний чек** | сколько они платят за похожие решения | «цены на внедрение AI малый бизнес Россия» |
| **LTV / повторные** | возвращаются ли | «retention rate AI-консалтинг» |
| **Конкуренты в сегменте** | кто специализируется на этом сегменте | «AI-консалтинг для соло-экспертов Россия» |
| **Выручка лидеров** | сколько зарабатывают топ-3 | через VC.ru, Forbes, годовые отчёты |
| **CAC / каналы** | где привлекают, дорого/дёшево | «реклама B2B AI-агентство стоимость лида» |
| **Market Sophistication** | На какой стадии стоит сообщение конкурентов в этой нише (Schwartz 1-5) | анализ лендингов конкурентов + объявлений |

**Schwartz 5 стадий market sophistication:**

- **Стадия 1** — рынок новый, никто не обещает. Можно говорить просто «делает X».
- **Стадия 2** — пара игроков. Делаешь то же, но «лучше, дешевле, быстрее».
- **Стадия 3** — много обещаний. Нужна **новая механика** или **новый угол**.
- **Стадия 4** — рынок насыщен механиками. Нужен **new mechanism** + позиционирование.
- **Стадия 5** — рынок устал от обещаний. Нужно **новое имя категории** / переизобретение.

### Шаг 3 — Карта денег (money-map)

Свести в таблицу — для КАЖДОГО сегмента:

```markdown
## Сегмент: {имя}

| Метрика | Значение | Источник |
|---------|----------|----------|
| Размер рынка | ~XXX K чел/компаний | {URL/расчёт} |
| Средний чек | XXX ₽ | {URL/конкурент} |
| LTV | XXX ₽ × Y мес | {URL/расчёт} |
| Лидеры сегмента | {3 имени} | {URL} |
| Выручка топ-1 | ~XXX М ₽/год | {URL} |
| CAC оценка | XXX ₽ за лид | {URL} |
| Доступность | LOW/MED/HIGH | (свой вывод) |

**Вывод по сегменту:** 1-2 предложения — массово/нет, дорого/нет, конкурентно/нет.
```

### Шаг 4 — Финальное ранжирование по «жирности»

**Жирность ЦА = размер × средний чек × LTV × твоё соответствие.** Не один параметр, а **произведение всех**.

```markdown
## Money-map итоговая таблица

| # | Сегмент | Рынок | Чек | LTV | Конкуренция | Sophistication | Соответствие клиенту | 🎯 Жирность (1-10) | Вывод |
|---|---------|-------|-----|-----|-------------|----------------|---------------------|--------------------|-------|
| 1 | ... | ... | ... | ... | ... | ... | ... | ... | ... |

**Что Sophistication меняет в стратегии входа:**

- Стадия 1-2 → быстро заходим, прямое сообщение
- Стадия 3 → нужен свой угол / механика
- Стадия 4-5 → нужен new mechanism, иначе провалишься в шум

## Где деньги (топ по money-score)

1. Сегмент-X — самый высокий потенциал по {почему}
2. Сегмент-Y — ...
3. Сегмент-Z — ...

## Где «голубой океан» по деньгам
(сегменты где есть деньги + мало конкурентов)
- ...

## Где «красный океан»
(сегменты где много конкурентов + всё забито)
- ...

## Где «болото»
(мало денег + мало конкурентов — не лезть)
- ...
```

## Output

```yaml
output_file: projects/<main>-audience/intel/money-map-{date}.md
returns:
  - segments: [{ name, market_size, avg_check, ltv, competition, money_score }]
  - blue_ocean_segments: [...]
  - red_ocean_segments: [...]
  - top_recommendation: "сегмент-X"
  - confidence: 0-100%
```

Возвращает структурированный отчёт + краткое резюме для Алекса (4-7 предложений) — **которое сегмент видит самым денежным и почему**.

---

## Anti-patterns

- ❌ **Описание продуктов конкурентов** (это `/competitors-research`, другая задача)
- ❌ **Без цифр** — «большой рынок» не считается, нужны конкретные числа из источников
- ❌ **Без источников** — каждая цифра с URL или пометкой `[ГИПОТЕЗА — расчёт без подтверждения]`
- ❌ **Только один сегмент** — нужно минимум 4 для сравнения
- ❌ **Без вывода «голубой/красный океан»** — таблица без вывода = «вот данные, разберись сам»

## Время

~5-15 мин WebSearch + анализ. В фоне пока клиент отвечает на двойное интервью.

---

## Lite-режим

Если WebSearch недоступен — fallback на анализ из памяти (knowledge cutoff январь 2026) с пометкой каждой цифры `[ГИПОТЕЗА]`. Но эскалирует владельцу: «не могу подтвердить цифры, нужны живые данные».
