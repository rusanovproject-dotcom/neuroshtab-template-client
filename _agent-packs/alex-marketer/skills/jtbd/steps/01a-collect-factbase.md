# Шаг 01a. Собрать фактуру

**⛔ ПРАВИЛО КОММУНИКАЦИИ: весь этот шаг выполняется МОЛЧА. Пользователь НЕ видит: карточки клиентов, цитаты, промежуточные выводы, сколько файлов прочитал. Пользователь видит ТОЛЬКО короткое сообщение в начале (или развилку — см. ниже), и одно — в конце.**

## Цель

Прочитать ВСЮ фактуру из проекта. Вытащить из каждого источника ключевое про клиентов: кто, ситуация, запрос, что получил (если есть). Записать в черновик.

Это подготовительный шаг. Результат — черновик с выжимкой из всех источников. На следующем шаге (01b) — группировка в кластеры.

**Связь вперёд:** черновик → шаг 01b (группировка).

---

## ⚡ Развилка 1 vs 2 — что делать когда фактуры в офисе мало

**Перед началом чтения** проведи быструю инвентаризацию того что уже есть в проекте:

**tool call: view** — посмотри `projects/<main>/audience/`, `projects/<main>/inbox/`, `projects/<main>/clients/`, `office/clients/` (если есть), любые папки с диагностиками / транскриптами / интервью / детальными карточками.

Подсчитай **источники живой фактуры** — каждый из:
- транскрипт диагностического / продающего созвона с реальным клиентом
- интервью с купившим / отказавшимся клиентом (текстом или аудио)
- детальная карточка платящего клиента (полная, не одна строка)
- дословный кейс с цитатами клиента (не пересказ автора)
- развёрнутая переписка с клиентом из мессенджеров с реальной речью клиента
- развёрнутый отзыв с дословной речью клиента

**Считаются только дословные источники.** Не считается: лендинг, оффер, пост в канале, описание программы, общие пометки маркетолога «у нас 3 типа клиентов».

### Если живой фактуры **меньше 5 источников** ИЛИ **от менее чем 3 разных клиентов** — обязательная развилка:

Покажи клиенту **одно** сообщение, не начинай чтение:

> «Прежде чем распаковывать ЦА — нужна фактура. Без неё анализ будет 🔴 AI-генерация целиком, и любой следующий шаг (оффер, лендинг) поедет на гипотезах.
>
> У меня по твоему офису [N] источников живой фактуры — [перечислить кратко: «один кейс [имя] + одно интервью [имя]» или «только лендинг и оффер»]. Этого мало для нормальной распаковки.
>
> Два варианта как двинемся:
>
> **Вариант 1 — ты даёшь данные.** Скидываешь в чат: транскрипты диагностик, интервью с клиентами, детальные кейсы платящих, дословные цитаты из отзывов или переписок. Минимум 5 источников от 3+ разных клиентов. Это реальная фактура, на ней JTBD работает по-настоящему — цитаты идут как 🟢 факт.
>
> **Вариант 2 — я собираю синтетические касдевы.** Я генерирую 10-15 правдоподобных портретов клиентов из твоей ниши и провожу с каждым глубинное интервью по основным вопросам. Получишь готовую фактуру за 30-40 минут — но вся она будет помечена 🔴 AI-генерация. Это **временная заглушка**, чтобы первая версия анализа собралась. Потом, как соберёшь 3-5 реальных интервью — заменим в первую очередь по сегменту-фокусу.
>
> **Вариант 3 — гибрид.** Если у тебя есть немного живых данных, но не 5 — скидывай что есть, а я добью синтетикой. Реальные = 🟢/🟡, синтетические = 🔴.
>
> Какой вариант?»

### Реакция на ответ клиента

| Что сказал клиент | Что делаешь |
|---|---|
| «Сейчас скину» / «У меня есть, вот» | Жди материалы. Когда пришлёт — продолжай **обычный сбор фактуры (раздел «Алгоритм» ниже)** на присланных файлах + том что уже было в офисе. `/jtbd-aicustdev` НЕ запускается. |
| «Синтетика» / «Вариант 2» / «Нет данных, давай AI» | **Запусти `/jtbd-aicustdev` Step 00.** После возврата (с готовым `synthetic-factbase-<main>.md`) переходи на Step 01b cluster-segments — группируй сегменты на синтетике. |
| «У меня есть немного, добей синтетикой» / «Гибрид» | Сначала — **обычный сбор фактуры** на том что есть в офисе и что клиент дослал. Потом **запусти `/jtbd-aicustdev` Step 00** с режимом `hybrid`. Реальные источники → 🟢/🟡, синтетика → 🔴. |
| «Дай подумать» / «Не хочу решать сейчас» | НЕ запускай чтение, не переходи дальше. Зафиксируй в `agent-state.md` `interrupted: true, interrupted_reason: "ждём решения по фактуре"` и предложи вернуться когда определится. |

После записи решения в `agent-state.md`:

```yaml
factbase_source: real | synthetic | hybrid
factbase_decided_at: YYYY-MM-DD HH:MM
```

### Если живой фактуры **5+ источников от 3+ клиентов** — развилка не нужна

Сразу переходи к разделу «Алгоритм» ниже без вопроса клиенту, **с одним сообщением:**

> «Сейчас прочитаю всю фактуру — все файлы, которые ты загрузил. Это займёт время, но мне важно ничего не пропустить. Когда закончу — покажу результат.»

## Алгоритм

### 1. Составить список всех источников

**tool call: view** — посмотреть содержимое папки проекта.

**НЕ начинай читать файлы. Сначала — список.** Запиши в черновик ВСЁ что видишь:

**tool call: create_file** `сегменты-черновик.md`:
```
=== ИСТОЧНИКИ ФАКТУРЫ ===
1. [файл или папка] — тип: [диагностика / кейс / описание / анкета / отзыв / другое]
2. [файл или папка]
3. ...
Всего источников: [N]
```

Если видишь папки — загляни внутрь (`view`), допиши содержимое в список. Если видишь архивы — распакуй (`bash`), допиши. Если что-то непонятное — всё равно запиши, разберёшься при чтении.

### 2. Читать порциями, записывать после каждой

Читай по 5-7 файлов за раз. **Если файл больше 500 строк — читай его порциями по 500 строк (view с range: [1,500], [501,1000], [1001,1500] и т.д.). НЕ пытайся прочитать весь файл за один вызов. Частичное чтение = мусор на выходе.**

После КАЖДОЙ порции — два обязательных действия:

**Действие А.** Записать в черновик что нашёл:

**tool call: str_replace** → дописать в `сегменты-черновик.md`:
```
=== ПОРЦИЯ [N]: [какие файлы] ===
- [кто]: ситуация — [кратко], желаемый результат — «[что хочет получить — дословно из карточки]», получил — «[если есть, из кейса]»
- [кто]: ...
```

**Поле «желаемый результат» ОБЯЗАТЕЛЬНО.** Без него невозможно группировать по запросу на шаге 01b. Берётся из полной карточки клиента (поле «Желаемый результат»), не выдумывается. Если в карточке нет — записать «не указано».

**Действие Б.** Перечитать черновик:

**tool call: view** `сегменты-черновик.md`

Посмотреть список источников из пункта 1. Какие ещё НЕ прочитаны? → следующая порция. Все прочитаны? → пункт 3.

**ЧИТАЙ ФАЙЛЫ ЦЕЛИКОМ.** Не «первые 30-40 строк». Весь файл.

### 3. Финальная сверка

**tool call: view** `сегменты-черновик.md`

Пройдись по списку источников (пункт 1). Каждый файл/папка должен быть упомянут хотя бы в одной порции. Если что-то пропущено — дочитай сейчас.

### 4. Сказать пользователю

> «Прочитал всю фактуру: [N] источников, [N] клиентов. Перехожу к группировке.»

**Переходи к шагу 01b.**

---

## ЗАПРЕТЫ

- **НЕ пропускай источники.** Все файлы, все папки, все архивы. Без исключений.
- **НЕ читай только «первые строки».** Весь файл.
- **НЕ решай что «хватит, паттерны повторяются».** Читай до конца списка.
- **НЕ показывай пользователю** карточки клиентов, цитаты, промежуточные выводы.
- **НЕ начинай группировку** пока не прочитал всё. Группировка — шаг 01b.
