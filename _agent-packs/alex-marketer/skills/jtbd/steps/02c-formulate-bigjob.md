# Шаг 02c. Сформулировать Big Job + шапка сегмента

**⛔ ПРАВИЛО КОММУНИКАЦИИ: пользователь видит ТОЛЬКО финальный результат (шапка + Big Job + вопрос). НЕ видит: проверочные таблицы, критерии А-Ж, версии формулировок, внутренние рассуждения, термины JTBD.**

## Где ты сейчас (контекст)

Ты определяешь **главную задачу в жизни** (Big Job) для конкретного сегмента. У тебя уже есть проверенные цели клиентов — теперь нужно из них собрать одну формулировку, проверить её и показать пользователю вместе с шапкой сегмента.

Определение Big Job состоит из трёх подшагов:
1. **02a** (сделано) — извлёк запросы, классифицировал
2. **02b** (сделано, если был нужен) — задал вопросы, получил и проверил ответы
3. **02c (ТЫ ЗДЕСЬ)** — сформулировать Big Job, проверить, собрать шапку, показать, записать

Результат этого подшага: в основном документе записана шапка сегмента (название + Big Job + критерии), подтверждённая пользователем.

**Связь назад:** проверенные цели из 02a или 02b (в `big-job-черновик.md`).
**Связь вперёд:** шапка сегмента → шаг 03 (jobStory).

## Алгоритм

### 1. Перечитать ВСЕ промежуточные результаты по этому сегменту

**tool call: view** `big-job-черновик.md`

Найти и перечитать ВСЕ блоки по этому сегменту:
- Запросы клиентов (из 02a)
- Классификация (из 02a)
- Ответы пользователя + проверки (из 02b, если были)

Это твои «стикеры» — промежуточные результаты, из которых собираешь Big Job.

### 2. Сформулировать Big Job v1

Из проверенных целей (те, что помечены ✅) — найти общий знаменатель. Люди в одном сегменте уже похожи по ситуации, поэтому обобщение должно быть конкретным, не абстрактным.

**Что такое Big Job (методология JTBD-методологии):**

Big Job — самая высокая работа, которую продукт НЕ полностью выполняет. Это то, ради чего клиент вообще пришёл — цель В ЕГО ЖИЗНИ, за пределами продукта.

Правильная Big Job:
- Конкретный результат с моментом достижения (можно сказать «сделано»)
- Снаружи продукта (продукт не может полностью выполнить)
- На языке клиента (не маркетинговый слоган)
- Клиент скажет это другу своими словами

**Тест на булшит (JTBD-методология):** если формулировка звучит широко, абстрактно и красиво — скорее всего это булшит. Два признака: (1) обобщающие слова («комфорт», «успех», «свобода», «трансформация», «развитие»), (2) невыполнимость гарантии.

**Запрещённые формулировки:**
- Процессы: «навести порядок», «контролировать», «разобраться», «управлять», «накапливать», «внедрить», «сделать так чтобы», «обеспечивать», «начать зарабатывать»
- Средства: «иметь план», «получить систему», «найти алгоритм», «иметь сумму на счёте»
- Эмоции: «чувствовать уверенность», «перестать тревожиться», «жить спокойно»

**Запрет тарифов и цен в Big Job + критериях сегментации:**
- Big Job, jobStory, критерии сегментации НЕ должны содержать конкретные тарифы продукта (60К/90К/250К) или ценовые коридоры продукта
- Это Stage 2 (Product/Offer), не Stage 1 (Audience)
- Допустимо: «выручка компании клиента 300К-2М/мес» — это критерий платёжеспособности сегмента
- Недопустимо: «бюджет на программу 60-250К», «тариф 90К», «соразмерно базовому тарифу» — это привязка к нашему продукту, не к жизни клиента

### 3. Записать формулировку и проверку в черновик

**tool call: str_replace** → дописать в `big-job-черновик.md`:
```
=== СЕГМЕНТ: [название] — Big Job v1 ===
Формулировка: «[текст]»

Проверка:
А. За пределами продукта? [ДА/НЕТ] — Тест: «Если клиент купит продукт и пройдёт идеально — формулировка УЖЕ выполнена?» Если ДА = ❌ (это Core Job).
Б. Не процесс? [ДА/НЕТ] — Есть конечная точка?
В. Не средство? [ДА/НЕТ]
Г. Не эмоция? [ДА/НЕТ]
Д. Клиенты сегмента помещаются? [проверить каждого по имени]
Е. Клиент скажет это другу? [ДА/НЕТ]
Ж. Похожа на эталоны по структуре? [ДА/НЕТ] — на какой

ИТОГ: ✅ / ❌ [по каким критериям провалена]
```

**Эталоны Big Job из JTBD-методологии (для сверки по СТРУКТУРЕ, не для копирования):**
- «Оказаться в нужном месте» (такси)
- «Пробить 1 млн/мес и продать 3000 лицензий» (AI-курс)
- «Нормально жевать и не стесняться состояния своих зубов» (стоматология)
- «Сдать экзамен на гражданство» (немецкий)
- «Обеспечить финансовую стабильность для меня и будущих детей» (девелопер)

Общее: конкретный результат, момент достижения, снаружи продукта, язык клиента.

### 4. Перечитать и решить

**tool call: view** `big-job-черновик.md` — найти строку «ИТОГ:» в последнем блоке.

- **ИТОГ = ✅** → переходи к пункту 5 (показать)
- **ИТОГ = ❌** → попробуй переформулировать (v2). Если v2 тоже ❌ → **НЕ ВЫДУМЫВАЙ. Иди в шаг 02b и задай вопросы пользователю.** Лучше спросить, чем показать кривую формулировку.

**НЕ ПОКАЗЫВАЙ результат пользователю пока ИТОГ ≠ ✅. Никогда не показывай формулировку с пометкой 🟡 если она не прошла проверку — это дерьмо, а не гипотеза.**

### 5. Собрать шапку сегмента

Три элемента:

**Название** — 3-5 слов, понятных без контекста. Тест: прочитал — представил конкретного человека? Если нет — переписывай.

**Big Job** — формулировка из пункта 2-4 (прошедшая проверку).

**Критерии сегментации** — минимум 2-3 признака, по которым можно сказать «этот человек из этой группы / не из этой». По работам, не по демографии.

### 6. Показать пользователю — ФОРМАТ «3 вопроса → 4 варианта»

**КРИТИЧНО:** Big Job — это формулировка которая определяет всю дальнейшую работу. Нельзя выдавать **одну** Big Job и спрашивать «бьётся?». Это подсовывает клиенту твоё видение.

Делишь подачу на **2 хода**:

#### Ход 1 — 3 наводящих вопроса РАЗОМ (одним сообщением)

Перед формулировкой Big Job выпиши **ровно 3 вопроса**, без ответов на которые формулировка будет угадайкой. Это вопросы про конкретику, которую ты не вытаскиваешь из фактуры самостоятельно.

**Что обычно в этих 3 вопросах для Big Job:**
1. Где **граница «получилось»** — какой конкретный результат клиент сам признаёт «сделано» (цифра / событие / момент)
2. Где **антипример** — что внешне похоже, но **НЕ** Big Job этого сегмента (проверка на широкость формулировки)
3. **Зачем** клиент это делает — конечный мотив за результатом (не повторение результата другими словами, а уровень глубже)

> *«Прежде чем сформулировать главную задачу этой группы — мне нужно докопаться до 3 вещей. Без этого формулировка будет угадайка:*
>
> *1. [конкретный вопрос про границу «получилось»]*
> *2. [конкретный вопрос про антипример — что НЕ относится]*
> *3. [конкретный вопрос про мотив — «зачем» а не «что»]*
>
> *Ответь по каждому одной фразой, дальше предложу 4 варианта формулировки на выбор.»*

#### Ход 2 — 4 варианта Big Job на выбор (после ответов клиента)

После ответов на 3 вопроса — выдаёшь **4 разные грани** Big Job. Каждый вариант — про одно и то же, но с разным фокусом. Клиент выбирает один, или комбинирует, или дописывает свой.

**Чем 4 варианта должны различаться:**
- Разный **фокус ценности** (например: выручка / время / спокойствие / контроль / репутация)
- Разная **ширина** формулировки (узко-конкретная vs широкая стратегическая)
- Разная **точка отсчёта** (от боли клиента vs от результата vs от процесса)
- Разный **горизонт** (быстрая победа vs долгосрочное преимущество)

> *«Группа клиентов:*
>
> ***[Название сегмента]***
>
> *Отличаем от других по: [критерии]*
>
> *Основано на:*
> *— [Имя]: «[его цель]»*
> *— [Имя]: «[его цель]»*
>
> *На основе твоих ответов — 4 варианта главной задачи. Каждый — про одно и то же, но с разным фокусом. Выбери один, скомбинируй, или допиши свой:*
>
> ***Вариант 1 — [угол: например, выручка]:***
> *«[формулировка]»*
>
> ***Вариант 2 — [угол: например, освобождение времени]:***
> *«[формулировка]»*
>
> ***Вариант 3 — [угол: например, контроль / спокойствие]:***
> *«[формулировка]»*
>
> ***Вариант 4 — [угол: например, рост без найма]:***
> *«[формулировка]»*
>
> *Какой ближе? Или комбинация?»*

**НЕ показывать:** термины JTBD, внутреннюю классификацию, проверочные таблицы.

**Чего НЕ делать:**
- ❌ Выдавать одну Big Job и спрашивать «бьётся?»
- ❌ Задавать 3 вопроса по одному (растягивает на 3 итерации без нужды)
- ❌ Скрывать угол варианта — клиент должен видеть **на чём фокус** в каждом
- ❌ Делать 4 одинаковые формулировки с разными словами — это халтура. Делай 4 **реально разные грани**

### 7. Записать в основной документ

После подтверждения пользователя:

**tool call: view** `big-job-черновик.md` — перечитать финальную версию.

**tool call: str_replace** → записать в `JTBD_анализ_[продукт].md`:
```
## Сегмент 1: [Название]
### Шапка сегмента
- Название: [название]
- Главная задача (Big Job): [формулировка] 🟡 экспертная гипотеза
- Критерии сегментации: [список]
```

Обновить файл прогресса. **Переходи к шагу 03** (jobStory) для этого сегмента.

## Антипаттерны

**Абстрактные названия сегментов:**
❌ «Зарабатывающие в хаосе», «Искатели свободы», «Осознанные предприниматели»
✅ Конкретная ситуация + ключевой признак

**Big Job = процесс:**
❌ «Развивать бизнес», «Управлять финансами», «Расти профессионально»
✅ Конкретный результат с моментом достижения

**Big Job = то, что делает продукт:**
❌ Если продукт может полностью выполнить — это Core Job, не Big Job

**Сегментация по демографии:**
❌ Пол, возраст, город — не основа. Демография — дополнительный признак.

## Не-сегменты

Если видишь группу людей которая НЕ подходит — не выкидывай. Запиши ПОЧЕМУ: слишком мало платит / нет частотности / не можем создать ценность / не можем привлечь.

## Разметка достоверности

- 🟢 **факт** — из реальных данных (интервью, CRM, записи)
- 🟡 **экспертная гипотеза** — на основе знания рынка, ответов владельца
- 🔴 **AI-генерация** — требует обязательной валидации
