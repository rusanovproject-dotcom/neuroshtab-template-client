# `/audience-check` — Self-check Алекса по Stage 1

Внутренний чек что Алекс идёт по плану и не сорвался в Stage 2-4. Без видимого
участия клиента — самопроверка перед следующим ответом.

---

📍 **Где это в системе:**
- **Уровень:** Internal self-check (Алекс себя, без клиента)
- **Запускается из:** авто-счётчика каждые 5 ходов внутри `/audience-stage`, после слов-триггеров из Stage Lock, перед `/marketer-revision`
- **Передаёт в:** `/audience-status` (если drift detected) → `/audience-resume`
- **Если detected срыв:** обязательная запись в `failures.md` + update `agent-state.md` `interrupted: true` до возврата

---

## Шаг 1 — Прочитать last 5 ходов диалога

Просмотри последние 5 своих ответов клиенту. На что смотри:

- Появлялись ли слова из запрещённого списка (см. core.md «🔒 Stage Lock»):
  tier, цена, чек, тариф, бизнес-модель, timeline, оффер, ladder, lead magnet,
  позиционирование, «продукт X отменяется/запускаем»?
- Решал ли я за клиента на развилке (фразы типа «моя рекомендация — делаем так»
  без выноса 3 вариантов)?
- Использовал ли формат «10 вариантов + цифры + допиши» при БПСВ — или спросил
  «дай 2-3 формулировки»?
- Был ли активный скилл во время этих 5 ходов — или работа шла в свободном
  диалоге без `/audience-stage`?

---

## Шаг 2 — Сверить с agent-state.md

Прочитай `office/agents/alex-marketer/agent-state.md`:

- Какой `active_skill`? Если null — это сигнал срыва (без скилла на тему ЦА работать нельзя).
- Какой `active_step`? Совпадает ли он с тем что я делаю по факту?
- `interrupted: false`? Если true — должен был быть `/audience-resume`, а не
  свободный продолжающий диалог.

---

## Шаг 3 — Решить (без участия клиента)

| Состояние | Действие |
|-----------|----------|
| Всё чисто, идём по плану | Update `agent-state.md` → `last_check: <date> ok`. Продолжай. |
| Найдено ≥ 1 запрещённое слово в Stage 1 | **STOP**. Перепиши последний ответ. Запиши в `failures.md` срыв. Update `agent-state.md` → `interrupted: true, reason: scope-drift`. |
| Решал за клиента на развилке | Перепиши ответ — «3 варианта, рекомендую X, выбирай». |
| БПСВ-вопрос без 10 вариантов | Перепиши вопрос в формат «вот 10 пронумерованных, отметь цифры + допиши 2-3». |
| Работа без активного скилла > 5 ходов | **STOP**. Запусти `/audience-stage` или `/audience-resume`. |
| `interrupted: true` но идёт свободный диалог | **STOP**. Запусти `/audience-resume`. |

---

## Шаг 4 — Если был срыв — записать и вернуться

При обнаружении срыва (любой триггер из таблицы кроме «всё чисто»):

1. Append в `failures.md`:
   ```
   ### YYYY-MM-DD HH:MM — auto-detected scope drift via /audience-check
   **Что произошло:** [конкретные слова из запрещённого списка / решение за клиента / пр.]
   **Контекст:** [последний ход диалога]
   **Возврат:** запустил [`/audience-resume` / переписал ответ / др.]
   ```

2. Update `agent-state.md`:
   ```
   interrupted: true
   interrupted_reason: scope drift detected at <step> via audience-check
   resume_hint: вернуться к <active_step> Шага <N>
   ```

3. В чате клиенту краткое признание + действие (без долгих объяснений):
   > *«Услышал — я съехал. Возвращаюсь в [текущий Шаг].»*
