# Шаг 04. Кластеризация цитат и запись в synthetic-factbase

**⛔ ПРАВИЛО КОММУНИКАЦИИ: весь шаг МОЛЧА. Клиенту в конце шага одно сообщение с краткой выжимкой (топ-кластеры) — не длинный отчёт, а 5-7 строк.**

## Цель

После Step 03 в `synthetic-factbase-<main>.md` лежат полные интервью. Этот шаг — **кластеризовать повторяющиеся цитаты** по 6 категориям JTBD-фрейма (триггеры / барьеры / Consideration Set / Точка А / Точка Б / Big Job) и записать кластеры в начало файла как навигацию.

**Правило сильного сигнала:** цитата — кластер если её вариация встречается у **≥ 3 разных персон**. Меньше — это шум, в кластер не идёт (помечается в backlog внутри файла).

**Каждая цитата в кластере остаётся 🔴** — даже если 10 персон сказали похожее. Повторение в синтетике не делает её фактом.

---

## Алгоритм (молча)

### 1. Прогон по 6 категориям

Открой `synthetic-factbase-<main>.md`. По каждой персоне найди ключевые цитаты (помечены `🎯 →` в Step 03). Распредели по 6 кластерам:

```
1. Триггеры покупки → из Блока 2 (триггер) и Блока 8 (большая задача)
2. Барьеры переключения → из Блока 5 (барьеры) и Блока 6 (критерии — что нужно для решения)
3. Consideration Set → из Блока 4 (альтернативы)
4. Точки А (что было до) → из Блока 1 (контекст) и Блока 3 (опыт-попытки)
5. Точки Б (что хочется получить) → из Блока 7 (эмоции-идеал)
6. Big Job (формулировки от первого лица) → из Блока 8 (зачем в большой картинке)
```

### 2. Группировка похожих цитат

Внутри каждого кластера группируй по **смысловому совпадению**, не по словам.

Пример:
- «Боюсь что снова сольёт деньги» (Персона #3)
- «Уже один раз обжёгся, вторая попытка стрёмная» (Персона #7)
- «Сложно поверить что в этот раз другое» (Персона #11)

Это всё **один кластер** «страх повторного провала после неудачного опыта». Группа: 3 персоны → сильный сигнал. В кластер.

### 3. Сильный сигнал vs шум

```
Сильный сигнал (попадает в кластер): вариация цитаты у ≥ 3 разных персон
Слабый сигнал (попадает в backlog внутри файла): 1-2 персоны
```

Слабые сигналы записываешь в самом конце файла как `## ⚠️ Слабые сигналы / для возможного возврата` — могут пригодиться когда клиент соберёт реальные интервью и появятся подтверждения.

### 4. Anti-slop фильтр на цитатах

Перед записью в кластер каждую цитату проверь: звучит ли это как **живая речь живого человека**?

- ❌ «Я хочу масштабироваться эффективнее» → не оставлять, переписать в живую речь персоны или отбросить
- ❌ «Мне нужна синергия процессов» → отбросить
- ✅ «Не понимаю как из 3 клиентов сделать 30, а сам уже не успеваю» → оставить
- ✅ «Бля, четвёртый раз курс, и опять чувствую что воду продали» → оставить (если по типажу персоны мат уместен)

### 5. Запись кластеров в файл

Перейди в начало файла `synthetic-factbase-<main>.md`, секция `## Кластеры цитат` — заполни через `str_replace`:

```markdown
## Кластеры цитат (топ-паттерны)

### Триггеры покупки
1. **«[ключевая формулировка кластера]»**
   - Встречается у: [N]/15 персон ([X]%)
   - Цитаты:
     - "[цитата 1 дословно]" — Персона #N 🔴
     - "[цитата 2 дословно]" — Персона #M 🔴
     - "[цитата 3 дословно]" — Персона #K 🔴
2. **«[следующий кластер]»**
   - ...

[обычно 3-5 кластеров на категорию]

### Барьеры переключения
1. ...
2. ...

### Consideration Set (с чем конкурируют)
1. **[Альтернатива]**
   - Что про неё говорят: "[цитата]" — Персона #N 🔴
   - Кто рассматривает: [N]/15 персон
2. ...

### Точки А (что было до)
1. ...

### Точки Б (что хочется получить)
1. **«[как описывают желаемое будущее]»**
   - Цитаты:
     - "[цитата]" — Персона #N 🔴
   - Что в этом эмоционально важно: [одна строка]
2. ...

### Big Job — формулировки от первого лица
1. **«[как сами клиенты могли бы сформулировать зачем]»**
   - Подтверждается у [N]/15 персон 🔴
2. ...
```

### 6. Сводка для handoff

В конце файла дописать секцию для следующего шага:

```markdown
## Сводка для /jtbd

- **Всего персон:** N
- **Сильных кластеров (≥3 персон):** [X] триггеров + [Y] барьеров + [Z] альтернатив + [A] Точек А + [B] Точек Б + [C] Big Job
- **Слабых сигналов (1-2 персоны):** [N] (в секции ниже — backlog)
- **Возможные сегменты по типажам персон:** [предварительная группировка — 2-3 группы по контексту/триггеру/Точке Б]
- **STATUS:** synthetic-draft 🔴
- **Заменить на реальную фактуру:** ДА (минимум 3-5 интервью с живыми платящими по сегменту-фокусу перед закрытием Stage 1)

## ⚠️ Слабые сигналы / для возможного возврата

[цитаты от 1-2 персон которые не попали в кластеры]
```

## Что СКАЗАТЬ клиенту в конце шага

Одно сообщение, 5-7 строк, **без терминов**:

> «Готово. По 15 портретам собралась фактура — всё в файле, вся помечена что AI-сгенерирована.
>
> Главное что увидел в повторениях:
> — Триггер чаще всего — [короткая формулировка]
> — Главный барьер — [короткая формулировка]
> — С чем сравнивают — [топ 1-2 альтернативы]
> — Точка Б — [короткая формулировка]
>
> Это черновая картинка. Сейчас по этой фактуре пройдёмся в основном анализе и соберём сегменты с jobStory. Когда позже у тебя появятся 3-5 живых клиентов — заменим синтетику на реальное в первую очередь по тому сегменту куда хотим бить.
>
> Поехали к группировке сегментов?»

Жди «ок» / «поехали» / «продолжай» → **Step 05 (handoff)**.

Если клиент скажет «погоди, посмотрю файл» — без проблем, дай ему открыть `synthetic-factbase-<main>.md`. После просмотра он либо подтвердит, либо попросит правки. На правки реагируй точечно: можно перегруппировать кластер, добавить пропущенный, переформулировать. **НЕ переделывай интервью** — это означает откат на Step 03, что дорого по времени.

---

## ЗАПРЕТЫ

- НЕ снимай 🔴 ни с одной цитаты
- НЕ кластеризуй по словам (одинаковые слова) — кластеризуй по смыслу
- НЕ оставляй AI-слоп в кластерах — отбраковка обязательна
- НЕ показывай клиенту все цитаты простынёй — только топ-выжимку
- НЕ переписывай интервью на этом шаге (это работа Step 03) — только перегруппировка
- НЕ создавай отдельный кластер «боли» / «возражения» / «страхи» — это БПСВ, не JTBD. Барьеры/триггеры/Точки А-Б покрывают всё, что нужно
