# JTBD AI-CustDev — синтетические касдевы для первого черновика

## Назначение

Когда у клиента **нет** живой фактуры от платящих клиентов (минимум 5 источников от 3+ разных клиентов) — этот модуль собирает **синтетические касдевы**. Алекс генерирует 10-15 правдоподобных портретов клиентов из ниши проекта (OCEAN-профилирование), проводит с каждым глубинное интервью по 8 блокам, кластеризует ответы и записывает результат в `projects/<main>/audience/synthetic-factbase-<main>.md`.

Этот файл становится «источником цитат» для основного `/jtbd`. **Каждая цитата помечается 🔴 AI-генерация** и не повышается на дальнейших шагах — даже если 10 синтетических персон сказали одно и то же.

**Ключевой инвариант:** синтетика — это временная заглушка для построения первой версии анализа. Перед закрытием Stage 1 — обязательная замена 🔴 на 🟢 минимум по сегменту-фокусу через 3-5 реальных интервью с живыми платящими клиентами.

## Когда вызывается

**Главный вход** — из `/jtbd Step 01a` при выборе варианта 2 (синтетика) или гибрида (реальные + синтетика).

**Ручные триггеры:** «нет данных от клиентов», «синтетический касдев», «сгенерируй интервью», «AI касдев», «синтетика».

## Чего модуль НЕ делает

- НЕ заменяет реальные JTBD-интервью с живыми клиентами
- НЕ снимает 🔴 пометку с цитат ни на каком шаге
- НЕ генерирует БПСВ-структуру (боли/потребности/страхи/возражения как отдельные категории) — это конкурирует с JTBD-фреймом, ломает его. Структура интервью сразу под jobStory / Точки А-Б / Consideration Set / барьеры / Big Job
- НЕ работает для дофаминовых продуктов (игры, соцсети, гемблинг) и неосознаваемых (еда, одежда). Если продукт клиента из этих категорий — STOP, скажи клиенту что Advanced JTBD на нём не работает, синтетика тем более

## Что показывать пользователю — только результат

Пользователь НЕ слышит:
- «OCEAN», «Big Five», «8 блоков Кадырова», «Awareness Level», «персона #N», «фабрика», «уровни follow-up»
- «Шаг 1», «Phase A», «Pre-flight», «гейт», «trigger»
- Имена файлов (`synthetic-factbase-<main>.md`, `synthetic-personas.md`)

Пользователь слышит:
- «Соберу 10-15 портретов клиентов из твоей ниши»
- «Проинтервьюирую каждого по основным вопросам»
- «Все цитаты будут с пометкой что это AI-сгенерировано — это временная заглушка»
- «Когда соберёшь 3-5 реальных интервью — заменим»

## Output

Один файл: `projects/<main>/audience/synthetic-factbase-<main>.md` со структурой:

```markdown

ниши. Все цитаты помечаются 🔴 AI-генерация. Использовать как ВРЕМЕННУЮ

фактуру при первой возможности (минимум 3-5 интервью с живыми платящими
клиентами по сегменту-фокусу).

- Ниша / тип бизнеса: ...
- Главный продукт: ...
- Типы клиентов (если назвал клиент): ...
- Количество персон: N
- Дата генерации: YYYY-MM-DD

## Кластеры цитат (топ-паттерны)
```

После создания файла — handoff в `/jtbd Step 01b` (cluster-segments) на этой синтетической фактуре. В сводной таблице достоверности основного JTBD-документа все блоки сегментов автоматически получают пометку 🔴 (только синтетика) или 🟡 (если был гибрид).

## Гейт закрытия — критичный инвариант

После прохождения через `/jtbd-aicustdev`:

1. В `agent-state.md` дописывается флаг `factbase_source: synthetic` (или `hybrid` если был гибрид)
2. В `JTBD_анализ_<main>.md` в начале добавляется блок **🔴 STATUS: synthetic-draft** с предупреждением что это черновик на синтетике
3. **Stage 1 категорически не закрывается** до сбора минимум 3-5 интервью с живыми платящими клиентами и пересборки сегмента-фокуса на реальной фактуре
4. `/jtbd-critic` автоматически выдаёт ⚠️ АУДИТ-блок с требованием замены

## Последовательность шагов

```
1. Прочитай steps/00-entry-and-decision.md → подтверди вариант 2, собери 3 параметра
2. Прочитай steps/01-clarify-niche.md → если нужно (1-2 уточняющих вопроса)
3. Прочитай steps/02-persona-factory.md → сгенерируй 10-15 OCEAN-персон, покажи 3-4 примера, дождись «ок»

```

## Тайминг

Полное прохождение для клиента: **30-40 минут**. Из них активного диалога с клиентом — 5-7 минут (3 вопроса в Step 00 + ок на 3-4 примера персон в Step 02 + ок на финальную фактуру). Остальное Алекс делает молча в файлах.

## Guardrails (железно)

- **Все цитаты — 🔴 AI-генерация. Без исключений.** Повторение в синтетических данных не делает цитату фактом.
- **10-15 персон — не больше.** Project-x фабрика делает 20, но для JTBD это избыточно. 10-15 даёт достаточный спектр без раздувания времени.
- **Структура ответов — JTBD-совместимая.** jobStory-материал (когда / хочу / чтобы), Точка А, Точка Б, Consideration Set, барьеры, потенциальная Big Job. **НЕ собираем БПСВ-категории отдельно.**
- **Diversity check обязателен** — 1-2 фаната + 2-3 интересующихся + 1-2 скептика + 1 случайный. Без скептиков синтетика звучит как маркетинговая брошюра.
- **Anti-slop:** ни одна персона не говорит «я хочу инновационное решение», «синергия», «трансформация». Только живая речь ниши.
- **Если продукт дофаминовый/неосознаваемый — STOP.** Не запускай интервью.
- **Не показывай клиенту 10-15 портретов простынёй.** Покажи 3-4 примера, спроси «ок продолжаю?» — потом интервью молча.
