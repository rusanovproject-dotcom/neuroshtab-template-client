# Алекс v2.0 — Internet-First Pipeline (Architecture v2)

> **Обновлено:** 2026-04-28 после прогона v1.6 у клиента
> **Старый план:** `_archive/v2.0-plan-v1-2026-04-27/` (архитектура «выкачиваем из клиента»)
> **Новая парадигма:** Internet-First — Алекс делает анализ САМ из реального интернета, клиент валидирует галочками за минуты.

---

## Видение

**Старая архитектура** (v1):
- 6 сессий по 1.5-2ч = 10-12 часов работы клиента в чате
- Алекс выкачивает из клиента БПСВ → потом валидирует
- 80% портрета = догадки клиента, 20% = живые цитаты
- Клиент устаёт к 4-му вопросу, отвечает галопом
- Нет реальных данных из интернета (YouTube/forums/queries)

**Новая архитектура** (v2 — Internet-First):
- 3 фазы (A/B/C) на каждый сегмент = **2-3 часа активной работы клиента** + 30-60 мин фоновой работы Алекса
- Алекс **сам идёт в интернет** (YouTube + форумы + поисковики + конкуренты) → возвращается с **жирной картой реальных данных**
- 80% портрета = валидированные интернет-источники, 20% = контекст клиента
- Клиент **валидирует галочками** через AskUserQuestion за 5-10 минут вместо часа писанины
- Финальный `voice-of-segment.md` = жирный артефакт уровня показанного Никитой VoC «Эксперт в потолке»

**JTBD клиента:** *«помоги найти точку приложения и распаковать ЦА жирно — но не заставляй меня писать кучу текста»*. Pull > Push + Anxiety + Habit достигается только через интернет-first подход.

## Эталонный финальный артефакт (что Алекс должен выдавать)

После прогона Phase A → B → C на каждый сегмент — `voice-of-segment.md` со структурой (валидировано на VoC «Эксперт в потолке»):

```
# VoC — {Имя сегмента} {краткое демо/ключ}

> Целевая ниша: 1-2 строки описания
> Источники сбора: WebSearch — vc.ru, форумы, YouTube, Telegram (N проверенных)
> Дата: YYYY-MM-DD
> Принцип: ВСЁ ДОСЛОВНО, без перефразировки

## Цитаты БОЛИ (30-50 штук, по подкатегориям)
### {Подкатегория 1: например "Потолок и застой"}
1. «{дословная цитата}» — [источник + URL]
2. ...

### {Подкатегория 2: например "Выгорание и потеря огонька"}
...

## Цитаты ЖЕЛАНИЯ (10-15 штук с источниками)
## Цитаты ВОЗРАЖЕНИЙ (15-25 штук, по типам: курсы / коуч / AI / ассистенты)
## Лингво — характерные слова и обороты (5+ категорий)
## Метафоры (10-15 с атрибуцией)
## Каналы и сообщества (Telegram + vc.ru + конференции)
## Конкуренты-учителя (5-10 имён + что предлагают + цена + источник)
## Что покупают / на что подписаны (топ-10)
## Сводка от исследователя (250 слов с 3 must-use цитатами для лендинга)
```

**Это и есть production-grade output Алекса v2.0.** Никита показал именно такой результат — мы воспроизводим его системно.

## Архитектура — 3 фазы (на каждый сегмент)

### **Phase A — Quick Context Capture** (10-15 мин, активный диалог)

**Цель:** понять физику бизнеса + получить материалы + **сформулировать 1-3 гипотезы сегментов** для дальнейшей разведки.

**Что Алекс делает:**

1. **6 тактов первого контакта** → выбор направления (как в v1.8, не меняется)

2. **Запрос материалов** (новое):
   > *«Скинь что есть в багаже про эту тему — кейсы, посты, отзывы, скрины переписок, голосовые после диагностик, отказы клиентов. Хоть в кучу. Я съем сам и пойду с этим в интернет — возвращусь с жирной картой через 30-60 минут.»*
   
   Материалы → `inbox/_raw/`

3. **4 ключевых вопроса** (вместо 5+уточняющих как в v1.8):
   - Физика бизнеса (5 типов клиента)
   - Кого ты зажигаешь сильнее всего? Кому уже дал больше всего пользы?
   - 1 конкретный кейс с цифрами (имя/результат/срок)
   - Какие у тебя ресурсы — связи, бэкграунд, кейсы — которых мало у кого?

   **Формат:** один вопрос за раз, не простыня (исправление бага v1.6).

4. **Формулирование 1-3 гипотез сегментов** на основе ответов + материалов:
   - Hot-гипотеза: где у клиента ЕСТЬ кейсы и связи + платят
   - Warm-гипотеза: рядом, ресурсы частично есть
   - Cold-гипотеза: рынок жирный, но ресурсов меньше — backlog

5. **Stop+wait:** клиент валидирует через AskUserQuestion («какая из 3 гипотез ближе?») или предлагает свою

**Output Phase A:**
- `inbox/_raw/` — материалы клиента
- `audience/segments/hypotheses.md` — 1-3 гипотезы сегментов с минимальным портретом (имя, профессия, доход, главный конфликт жизни в одной фразе, почему этот сегмент)
- `agent-state.md` обновлён: `active_phase: B, target_segments: [список slug]`

**Не делается в Phase A:**
- ❌ БПСВ ВЫКАЧИВАНИЕ из клиента (Алекс не пытает «дай 10 болей»)
- ❌ Глубокая сегментация через 8 этапов
- ❌ Ручной 7-осевой scoring (он будет автоматически после Phase B на основе данных интернета + ответов клиента)

### **Phase B — Internet Deep Research** (30-60 мин фоновая, **клиент свободен**)

**Цель:** для каждой гипотезы сегмента — собрать **жирный VoC с реального интернета** + конкурентную карту.

**5 параллельных субагентов** (запускаются через Task tool с `run_in_background: true`):

| # | Субагент | Что копает | Output |
|---|----------|-----------|--------|
| 1 | **YouTube Mining** | `site:youtube.com "{ниша}" "{боль}"` — 20-30 видео >10K просмотров за 24 мес. Заголовки + просмотры + топ-3 коммента | `intel/{slug}/youtube-cuts.md` |
| 2 | **Forum/Community VoC** | vc.ru / Reddit / Pikabu / b17.ru / holod.media / Telegram через t.me/s/ — 30-50 цитат с источниками | `intel/{slug}/community-voc.md` |
| 3 | **Search Queries** | Wordstat (если доступен) + Google PAA + suggest queries — топ-30 запросов с частотностью + 10-15 PAA-вопросов | `intel/{slug}/search-queries.md` |
| 4 | **Конкурентная карта** | Имена / тематики / лендинги / каналы / тарифы конкурентов в нише с источниками. 5-10 конкурентов | `intel/{slug}/competitors-map.md` |
| 5 | **БПСВ-синтезатор** | После завершения 1-4 — собирает паттерны (боли × подкатегории, желания, возражения по типам, лингво × категории, метафоры) с evidence_count и unique_sources | `intel/{slug}/bpsv-patterns.md` |

**Match-методика** (применяется в субагенте #5):
- Каждая боль/желание/возражение получает `evidence_count` + `unique_sources` из 5 типов источников (YouTube + Forums + Search + Competitors + материалы клиента из Phase A)
- `unique_sources ≥ 3 AND evidence ≥ 5` → **VALIDATED**
- `unique_sources = 2` OR `evidence = 3-4` → **SIGNAL**
- `unique_sources = 1` → **HYPOTHESIS** (только в backlog, не в финальный portrait)

**Финальный output Phase B на каждый сегмент: `voice-of-segment.md`** (как эталонный артефакт выше).

**Клиент во время Phase B:**
- Получает сообщение «Запустил 5 субагентов в фоне. Возвращаюсь через 30-60 минут с жирной картой по {сегмент}. Можешь отдохнуть, сделать другие дела, или скинуть ещё материалов в `inbox/_raw/` — я их съем по возвращении.»
- Алекс показывает прогресс через таблицу-dashboard (как в Director.md): | # | Субагент | Статус | (работает... / ✅ готов)

### **Phase C — Validation Through Choices** (15-25 мин на сегмент, через AskUserQuestion)

**Цель:** клиент **валидирует** жирный артефакт из Phase B через **серию мультиселектов**.

**Принцип:** Алекс приходит с готовой картой → клиент **отмечает галочками + дописывает свои дословно**. **Не «дай мне 2-3 формулировки»** — у клиента готовый список, он только подтверждает/корректирует.

**Серия мультиселектов через AskUserQuestion:**

**Чек 1 — Боли (1 минута):**
```
🎯 Какие из этих 15 болей ты реально слышишь у своей ЦА?
(отметь галочками + допиши 2-3 свои дословно если есть)

☐ "Я топчусь на 200К и не понимаю как сдвинуться" [vc.ru × 3 + YouTube × 2]
☐ "Кручусь как белка в колесе, толку ноль" [Forbes + 4 цитаты]
☐ "Достало, устал и хочу всё бросить" [holod.media + citydog.io]
... (15 вариантов с источниками)

+ Поле «Допиши свою боль дословно»
```

**Чек 2 — Желания (1 мин)**
**Чек 3 — Возражения (1 мин)**
**Чек 4 — Лингво-маркеры (1 мин)**
**Чек 5 — Метафоры (1 мин)**
**Чек 6 — Каналы где живёт ЦА (1 мин)**
**Чек 7 — Конкуренты (1 мин)** — «какие из этих 10 конкурентов реально в моей нише?»

Итого: **7 минут чистого времени клиента** на полную валидацию вместо часа писанины.

**Каждый отмеченный пункт:**
- VALIDATED по интернету (3+ источника) + клиент подтвердил → 🟢 GOLD
- VALIDATED по интернету, клиент не отметил → 🟡 SIGNAL (в файле остаётся, но помечается «клиент не подтвердил, требует custdev»)
- Дописанный клиентом, но нет в интернете → 🟡 SIGNAL (требует валидации в продажах)
- Не отмеченный + был только HYPOTHESIS → 🔴 в backlog

**Output Phase C:** финальный `segment-portrait.md` (бывший dossier.md) с цветовой схемой VALIDATED/SIGNAL/HYPOTHESIS на каждом пункте.

---

### **Phase D — Awareness и Lingvo Refinement** (30-45 мин на hot-сегмент, опционально)

**Цель:** для **hot-сегмента** (только одного, не всех) — добить распаковку через 4 уровня осознанности (Schwartz) + углублённый lingvo + Dream Outcome голосом клиента.

Это **сокращённая версия** старого `/segments-awareness` — большая часть работы уже сделана в Phase B (lingvo-маркеры собраны с интернета). Здесь только:
- Распределение цитат по 4 уровням (через subagent-классификатор за 5 мин, клиент только спорные валидирует)
- Anti-avatar профиль
- Dream Outcome от клиента в hero-блоке (1-2 цитаты владельца как «голос эксперта»)
- `segment-core.md` — сжатая база сегмента + 4 правила lingvo

**Output Phase D:** `segment-core.md` + дополнения в `segment-portrait.md` (секции 3, 4, 7, 8).

**Phase D НЕ делается** для warm/cold сегментов — они остаются на уровне Phase C (жирный VoC), это достаточно для Stage 2.

## Что меняется в скиллах

### Новые скиллы (создать)

1. **`/audience-quick-capture`** (Phase A) — 10-15 мин стартовая сессия. Заменяет первые шаги старого `/audience-stage`.
   - 6 тактов + запрос материалов + 4 вопроса + 1-3 гипотезы
   - Output: `hypotheses.md` + материалы в inbox/_raw/

2. **`/audience-internet-research`** (Phase B) — 30-60 мин фоновая разведка.
   - Запускает 5 параллельных субагентов
   - Финал: `voice-of-segment.md` × N сегментов
   - Заменяет старый `/competitors-research` + расширяет его кратно.

3. **`/audience-validation`** (Phase C) — серия 7 мультиселектов через AskUserQuestion.
   - 15-25 мин активного клиента
   - Output: финальный `segment-portrait.md`

4. **`/audience-awareness-lite`** (Phase D) — сокращённая версия старого `/segments-awareness`.
   - Только для hot-сегмента
   - Subagent-классификатор делает черновую раскладку, клиент валидирует спорное
   - Output: `segment-core.md`

## Roadmap по часам

### Этап 1 — Архитектура и new skills (5-6 часов)
1. Написать новый `/audience-quick-capture` (1 ч)
2. Написать `/audience-internet-research` с 5 субагентами (2 ч)
3. Написать `/audience-validation` через AskUserQuestion (1.5 ч)
4. Написать `/audience-awareness-lite` (1 ч)
5. Update `/audience-stage` → мета-оркестратор фаз (30 мин)

### Этап 2 — Migration старых скиллов (1.5 часа)
1. Move в `_legacy/`: `segments-discover`, `segments-unpack`, `segments-awareness`, `segment-money-map`, `segment-hypotheses`, `marketer-audience`, `marketer-ladder`, `marketer-funnel` (15 мин)

3. Удалить упоминания старых скиллов из core.md и Director.core.md (15 мин)
4. Routing rule в core.md под новую архитектуру (30 мин)

### Этап 3 — Wayfinding + UX (1.5 часа)
1. Wayfinding шаблоны (`📍 Phase X из 5` / `✅ Шаг N из M` / `🎯 Фаза закрыта`) во всех новых скиллах (45 мин)
2. AskUserQuestion корректное использование в Phase A + Phase C (30 мин)
3. Suppress «📚 0 failures» при N=0 (15 мин)

### Этап 4 — Финальная сборка (1.5 часа)
1. Update install.md под новую архитектуру (30 мин)
2. Renaming файлов на читаемый английский (`voc.md` → `voice-of-customer.md`, `dossier.md` → `segment-portrait.md`, etc) — везде в скиллах + шаблоне (45 мин)
3. Финальный синк pack → installed → workspace (15 мин)

### Этап 5 — Smoke test (1 час)
Симулировать 4 сценария:
- Established клиент (есть продажи) → Phase A → B → C должна пройти за 1-1.5ч активного времени
- Greenfield клиент (0 продаж) → Phase B опирается полностью на интернет
- Прерывание после Phase A → /audience-resume поднимает контекст
- HTML-артефакт собирается из жирного `voice-of-segment.md`

### Этап 6 — Phase E (HTML-артефакт) (2.5 часа)
Сохраняется как было в старом плане Фаза 8, но теперь имеет на входе **жирный** `voice-of-segment.md` — артефакт получается реально насыщенный.

**ИТОГО: ~13 часов реальной работы за 2-3 захода.**

## Risks & Mitigation

| Риск | Митигация |
|------|-----------|
| **AskUserQuestion не работает в чате клиента** (если клиент не в Claude Code) | Fallback на текстовый формат с цифрами «отметь номера 1, 5, 7». Авто-detection возможностей UI. |
| **5 субагентов в Phase B вернут мусор** | Match-методика: каждая боль требует ≥3 unique_sources до VALIDATED. Низкокачественные субагенты → SIGNAL/HYPOTHESIS, не идут в финал. |
| **Greenfield клиент не имеет материалов для Phase A** | Phase B опирается только на интернет (mode-aware). Mode detection в Phase A определяет глубину. |
| **Wordstat недоступен через WebSearch** | Fallback на Google PAA + suggest queries + ручное упоминание клиентом частотных запросов в Phase A. |
| **Telegram-каналы закрытые** | t.me/s/<channel> для публичных, для закрытых — `[мало данных, требует ручного парсинга]` пометка в файле. |
| **Старый pipeline сломается у текущих клиентов** | Старые скиллы остаются в `_legacy/`. `/audience-resume` для прерванных в v1.8 работ продолжает работать со старыми артефактами. Migration guide в RELEASE-NOTES. |

## Что НЕ входит в v2.0 (отложено в v2.1)

- Перепарсинг конкретных Telegram-каналов через telegram-channel-parser (требует MCP)
- Wordstat API (требует регистрации, fallback через WebSearch достаточен)
- Reviewer-субагент жирности (после Phase C автоматический чек «все ли пункты VALIDATED ≥3 источников»)
- Видео-демка для нового клиента
- Слив Phase 0+1+2 в одну сессию (актуально если UX покажет проблему)

## Migration старых артефактов v1.x → v2.0

При обновлении пака на установленных офисах:

1. Старые скиллы остаются в `_legacy/` — продолжают работать на текущих проектах
2. Если в проекте уже есть `dossier.md` — `/audience-status` распознаёт старый формат и предлагает либо продолжить со старым (использовать как input для v2.0 Phase B), либо начать заново
3. RELEASE-NOTES-v2.0.md содержит явный migration guide

## Финальное напутствие

**Главный сдвиг:** Алекс перестаёт быть «опросником» (выкачивает у клиента БПСВ через 50 вопросов) и становится **researcher-аналитиком** (сам идёт в интернет, приходит с готовым, клиент валидирует за минуты).

**JTBD клиента закрывается:** «помоги распаковать ЦА жирно — но не заставляй меня писать кучу текста». Pull (приходи с готовым) > Push (опросов слишком много) + Anxiety (ты наберёшь мусора) + Habit (AI всегда спрашивает много).

**Эталон финала:** VoC «Эксперт в потолке» Никиты — 45 цитат болей дословно, лингво по категориям, метафоры с источниками, конкуренты с ценами, сводка 250 слов. **Каждый сегмент клиента = такой же артефакт.**
