## 0. TL;DR

**Что строим:** Алекс — маркетолог-навигатор который находит точку приложения усилий + распаковывает жирную ЦА через Internet-First пайплайн (анализ интернета вместо опросов клиента).

**Финальный artifact на выходе:** `voice-of-segment.md` × N с 30-50 цитатами болей дословно с источниками + `segment-portrait.md` × N с цветовой схемой + `segment-core.md` для hot + `audience-report.html` для ментора.

**Эталон качества:** уровень VoC «Эксперт в потолке» который Никита получил вручную (45 цитат с vc.ru/Forbes/holod, лингво × 5 категорий, 15 метафор, 10 конкурентов с тарифами, сводка 250 слов).

**Архитектура:** 4 фазы (A → B → C → D) + опционально E (HTML).
- **Phase A:** 10-15 мин активного клиента, контекст + 1-3 гипотезы сегментов через AskUserQuestion
- **Phase B:** 30-90 мин Алекс работает в фоне, 5 параллельных субагентов копают интернет
- **Phase C:** 15-25 мин активного клиента на сегмент, валидация через 7 мультиселектов
- **Phase D:** 30-45 мин (только hot), углубление через subagent-классификатор по 4 уровням осознанности
- **Phase E (опц.):** 5-10 мин, интерактивный HTML-артефакт для ментора

**Setup для Никиты:** ~25 минут (3 API ключа + 2 MCP-сервера). Алекс проактивно через AskUserQuestion показывает что доступно/недоступно и предлагает установку.

**Стоимость:** $0/мес базовый (Free tier всех сервисов), $50/мес при росте до 5+ клиентов параллельно.

**Объём реализации:** ~10-12 часов работы → production-ready Алекс v2.0.

## 1. Архитектура — Internet-First Pipeline

### 1.1 Поток данных

```
Phase A (клиент)        Phase B (Алекс в интернете)         Phase C (клиент)
─────────────           ──────────────────────              ──────────────
Контекст                ┌─ YouTube Mining ──────┐           Валидация через
+ 4 вопроса      ───►   │ Forums VoC            │   ───►    7 мультиселектов
+ материалы             │ Search Queries (PAA)  │           AskUserQuestion
+ 1-3 гипотезы          │ Telegram Channels     │           + допиши свои
                        │ Конкурентная карта    │
                        │ → БПСВ-синтезатор     │
                        └────────────────────────┘
                              │
                              ▼
                    voice-of-segment.md × N
                    (30-50 цитат с источниками)
                              │
                              ▼
                  segment-portrait.md × N
                  (цветовая схема 🟢🟡🟠🔴)
                              │
                              ▼
Phase D (только hot)                      Phase E (опц.)
─────────────────                         ────────────────
Awareness 4 уровня      ───►              HTML-артефакт
+ Lingvo × 4 правила                      для ментора
+ Dream Outcome
+ Anti-avatar (5 типов)                   audience-report.html
                              │
                              ▼
                    segment-core.md (для копирайтера)
                              │
                              ▼
                    /accept Stage 1 → переход в Stage 2 Product
```

### 1.2 Принципы

1. **Internet-First:** реальные данные с интернета > догадки клиента. 80% портрета — VALIDATED цитаты с 3+ источниками.
2. **AskUserQuestion вместо открытых вопросов:** клиент валидирует галочками (8-15 опций на блок) + дописывает 2-3 свои дословно.
3. **Stage Lock:** запрет говорить про Stage 2-4 (продукт, цены, оффер, timeline) пока Stage 1 не закрыта на /accept.
4. **Wayfinding:** клиент видит «📍 Phase X из 4» в начале и «✅ Шаг N из M» после каждого блока.
5. **Match-методика:** двойной критерий VALIDATED — `(unique_sources ≥ 3 AND evidence ≥ 5) OR (unique_sources ≥ 2 AND evidence_weight ≥ 50)`.
6. **Mode Detection:** established/early-stage/greenfield автоматически из материалов клиента (0 → greenfield, 1-3 → early-stage, 4+ → established).
7. **Адаптивность по уровням возможностей:** Алекс детектирует доступные ключи/MCP, предлагает установку через AskUserQuestion, запускается на доступных уровнях.

### 2.3 Адаптивный Pre-flight в скиллах (переписано по UX-reviewer P0)

⚠️ **НЕ использовать мультиселект с 7 опциями для Setup** — нетехнический клиент отвалится в первые 5 минут. Вместо этого:

**Шаг 1 — Алекс сам делает env-grep + проверку MCP** (молча, без вопросов клиенту):

```bash
[ -n "$YOUTUBE_API_KEY" ] && have_youtube=1 || have_youtube=0
[ -n "$SERPAPI_KEY" ] && have_serpapi=1 || have_serpapi=0
[ -n "$TELEGRAM_API_ID" ] && have_telegram=1 || have_telegram=0
claude mcp list | grep -q youtube-transcript && have_yt_mcp=1 || have_yt_mcp=0
claude mcp list | grep -q telegram && have_tg_mcp=1 || have_tg_mcp=0
```

**Шаг 2 — выдаёт вердикт ОДНОЙ фразой:**

> *«Сейчас сам проверю что у тебя есть...*
> *✅ YouTube — на полной мощности (Data API + транскрипты)*
> *⚠️ SerpAPI — нет ключа (без него Google PAA недоступен)*
> *❌ Telegram — нет настройки (комменты TG-каналов недоступны)*
>
> *Без этих двух работаю на 60% от максимума. Сделать апгрейд (~25 минут) или поехали как есть?»*

**Шаг 3 — ОДНА развилка через AskUserQuestion с 3-4 опциями:**

```yaml
question: "Что выбираем?"
multiSelect: false
options:
  - id: full_setup
    label: "🛠 Сделаем апгрейд (25 мин) — дам пошаговые инструкции"
  - id: partial_setup
    label: "🔧 Только что-то одно (быстрее всего SerpAPI — 2 мин)"
  - id: go_as_is
    label: "⏭ Поехали как есть, апгрейдну потом"
  - id: explain
    label: "❓ Объясни что я теряю"
```

**Шаг 4 — если клиент выбрал «full_setup» — пошаговое сопровождение:**
- Алекс выдаёт ссылку + один шаг → ждёт «готово»
- Алекс **сам пишет в `.env`** через Write tool (клиент только присылает ключ)
- Алекс **сам проверяет** через env-grep что ключ работает
- Для Telegram (самая токсичная точка) — ссылка на скринкаст 3 мин + готов помочь по шагам

**Никаких 7 опций мультиселектом.** Один env-grep, один вердикт, одна развилка.

## 3. Скиллы — детальная спецификация

### 3.1 `/audience-quick-capture` — Phase A

**Объём:** ~250-300 строк SKILL.md.

**Шаги (обновлено — Mode Detection ДО вопросов, по UX-reviewer P0):**
1. **Pre-flight checks** (resume / state.md / Hard Pre-flight памяти)
2. **6 тактов первого контакта** (если не сделано) → выбор направления
3. **Запрос материалов** (5 категорий: cases / posts / transcripts / refusals / screenshots)
4. ⭐ **Mode Detection ПЕРЕД вопросами** — Алекс читает inbox/_raw/ + считает файлы → автоматически определяет mode + говорит клиенту живым языком (не термином):
   - 0 файлов → *«Слушай, у тебя сейчас на руках мало материалов — будем строить с гипотез, потом проверим в интернете.»* → mode=greenfield
   - 1-3 → *«Слышу, у тебя уже есть первые клиенты. Давай по ним пройдёмся — там самое богатое.»* → mode=early-stage
   - 4+ → *«Хорошо, у тебя уже база. Посмотрим паттерны и докрутим точку.»* → mode=established
5. **Бранчинг по mode** (НЕ единый набор вопросов):
   - **greenfield** (3 вопроса): физика бизнеса + желаемая ниша + что у тебя в багаже из ресурсов/связей. **БЕЗ вопросов про «1 кейс кто купил»** — у клиента их нет.
   - **early-stage** (4 вопроса): добавляется «1 первый клиент с минимальной цифрой» (но не пушим если не помнит)
   - **established** (5-6 вопросов): полный набор — кого зажигаешь / 1 купил + 1 НЕ купил / уникальные ресурсы / чек / куда уходят к конкурентам
6. **Формулирование 1-3 гипотез** через 3-circle-fit (ресурсы × боль × платёжность)
7. **AskUserQuestion для валидации гипотез** + опция «допишу свою»

**P0 правки от reviewer:**
- ⭐ Вопрос **#5 — Чек и платёжность** (новый): «Какой средний чек ты сейчас берёшь? Какой ты УЖЕ продавал хотя бы раз? От какого отказался / от тебя клиент?»
- ⭐ Вопрос **#3 расширить про отказников**: «1 кейс кто КУПИЛ + 1 кейс кто НЕ купил + причина отказа дословно»
- ⭐ Вопрос **#6 (опц.)**: «Куда твои потенциальные клиенты уходят к конкурентам и почему?» + «Где ты уже виден (Telegram, YouTube)?»
- ⭐ `confidence: 0.3-0.4` (не 0.7) на старте Phase A — соответствует Guardrail #5 core.md
- ⭐ Реалистичное время **20-40 минут** (не 10-15)
- ⭐ Жёсткое правило: если `state.md.physics_type` заполнено — пропускаем вопрос 1 (физика бизнеса)
- ⭐ Конкуренты НЕ в `inbox/_raw/` — отдельное поле `hypotheses.md.competitors_mentioned: [...]`
- ⭐ Mode Detection — явная таблица с числовыми порогами

### 3.2 `/audience-internet-research` — Phase B (главный скилл)

**Объём:** ~600-800 строк SKILL.md.

**Архитектура:** 5 параллельных субагентов через **5 Task tool calls в одном assistant turn** (НЕ `run_in_background: true` — этого параметра у Task нет, он только для Bash). Главный Алекс ждёт все 5 returns перед следующим turn.

⚠️ **UX impact:** клиент видит «Алекс молчит 10-15 мин, потом выдаёт всё разом». Нет live dashboard прогресса. Mitigations: (1) предупредить клиента в wayfinding-сообщении явно «уйду на 30-90 мин в фон, вернусь со всем готовым», (2) разбить Phase B на 2-3 ассистент-turn'а (например: «запускаю YouTube + Forums», ждёт returns, потом «запускаю Telegram + Search + Конкуренты», потом «синтезирую через БПСВ-агента») — даёт промежуточный прогресс.

**Полная переписка на real tech (P0 от reviewer):**

#### Субагент 1 — YouTube Mining

**Уровни глубины:**

```yaml
# Если есть YOUTUBE_API_KEY и MCP youtube-transcript:
level: 2
workflow:
  - search.list (100 units) → 25-30 видео
  - videos.list (1 unit × 25) → статистика, ER, длительность
  - Фильтр залётных: views/days_since_published > 1000
  - commentThreads.list (1 unit × 15) → топ-100 комментов под топ-15
  - MCP youtube-transcript → транскрипты топ-5 по залётам
  - AI-фильтр комментов (regex паттерны "Great video!", длина 80-150)
  - Output: youtube-cuts.md с 50-80 цитатами + 20 заголовками + 5 транскриптов

# Без YOUTUBE_API_KEY (Level 1):
level: 1
workflow:
  - WebSearch site:youtube.com "{ниша}" — 20 заголовков
  - WebFetch на каждое видео → метаданные (без комментов)
  - Output: youtube-cuts.md только с заголовками
  - Pre-flight предупреждение: "⚠️ Без API ключа собираю 30% от возможного. Установишь?"

# Level 3 (опционально с Python):
level: 3
extra:
  - youtube-comment-downloader без лимитов API
  - yt-dlp массовая выкачка транскриптов
  - Output: расширенный с 100+ комментов + 10+ транскриптов
```

⭐ **Маркетинговый фильтр (правила в промпте):**
- Engagement Rate = (likes + comments) / views × 100. Норма 4-5%, >7-9% — высокий
- Залёт = views/days. >100K за 7 дней в нише где обычно 5-10K — сигнал боли
- Комменты по `order=relevance`, не `time` — топ-100 согласного большинства
- AI-фильтр: одинаковая длина 80-150 без цифр/имён, общие фразы "Great video"
- Реальный VoC = специфика, мат, эмоции, цифры из жизни

#### Субагент 5 — Конкурентная карта

**Что копает:**
1. **WebSearch** «топ {ниша} 2024-2026» + конкуренты упомянутые клиентом
2. **WebFetch лендингов** топ-12 → матрица 7 атрибутов (имя/обещание/цена/Named Mechanism/Risk Reversal/каналы/источник)
3. **Реальные отзывы:** otzovik.com, holod.media, vc.ru/u/{author} комменты

- ✅ Только где есть деньги (продают, есть ценник публично или в отзывах)

**Output:** `competitors-map.md` — 8-12 конкурентов в матрице + whitespace + ценовые якоря + перенос инсайтов EN/EU → RU.

#### Субагент 6 — БПСВ-синтезатор (Match-методика)

**Запускается ПОСЛЕ возврата субагентов 1-5.**

**Алгоритм семантической кластеризации (явный):**

```
Шаг 1 — flat list всех цитат:
  Все цитаты из 5 файлов в одну таблицу:
  | quote | source | type (боль/желание/возражение/метафора) | likes/views |

Шаг 2 — pair-by-pair кластеризация:
  for each pair (cite_a, cite_b):
    if shared ≥ 2 ключевых смысловых слова AND triger одинаков:
      merge в один кластер
    else:
      разные кластеры

Шаг 3 — evidence_count и evidence_weight:
  Для каждого кластера:
    evidence_count = N цитат в кластере
    unique_sources = число РАЗНЫХ типов источников (out of 8 типов)
    evidence_weight = sum(likes × source_quality_multiplier)
      где multiplier:
        vc.ru = 3
        YouTube = 1
        Reddit = 2
        Wordstat = 5 (зеркало рынка)
        Telegram = 4
        Habr = 2
        Pikabu = 1
        Forums other = 1.5

Шаг 4 — двойной критерий статуса:
  🟢 GOLD     = (unique_sources ≥ 3 AND evidence ≥ 5) OR (unique_sources ≥ 2 AND evidence_weight ≥ 50)
  🌟 GOLD-VIP = single source но >200 лайков на vc.ru / >500 на YouTube / >50 на Reddit (для hero-блока)
  🟡 SIGNAL   = unique_sources = 2 OR evidence = 3-4 (требует custdev)
  🔴 BACKLOG  = unique_sources = 1 (только клиент или один источник интернета)

Шаг 5 — Алекс ВРУЧНУЮ читает топ-10 GOLD цитат для проверки на синонимы:
  Если 3 цитаты «упёрся в потолок» — это 1 уникальный кластер, не 3
```

**Дополнительный gate после Phase B (от маркетинг-reviewer):**

```
Если total_GOLD_clusters_after_BPSV < 8:
  STOP, эскалация владельцу:
  «Phase B вернула тонкий VoC ({N} GOLD кластеров вместо ≥8 ожидаемых).
   Это сигнал что: (а) субагенты копали не там / (б) ниша слишком узкая /
   (в) клиент дал недостаточно материалов в Phase A.
   3 варианта: переделать Phase B с другими ключевыми словами / расширить
   гипотезу сегмента / принять тонкий VoC и идти в Phase C с пометкой
   "ограниченный по источникам".»
```

Без этого gate тонкий VoC просочится в Phase C → клиент валидирует мусор → portrait худой.

**Финальный artifact: `voice-of-segment.md`** — структура ровно как VoC «Эксперт в потолке» Никиты, с **15-25 уникальными смысловыми кластерами** (после dedupe из 30-50 сырых цитат):
- Цитаты БОЛИ (по подкатегориям с цветами 🟢🌟🟡🔴) — 30-50
- Цитаты ЖЕЛАНИЯ — 10-15
- Цитаты ВОЗРАЖЕНИЙ (по типам: курсы / коуч / AI / ассистенты) — 15-25
- Лингво × 5+ категорий
- Метафоры с источниками — 10-15
- Каналы и сообщества (TG + vc.ru + конференции)
- Конкуренты-учителя 5-10 с тарифами
- Что покупают / на что подписаны топ-10

**Размер ограничение:** 600 строк max. Если больше — топ-30 болей по weight, остальное в `_archive/full-voc-{date}.md`.

### 3.3 `/audience-validation` — Phase C

**Объём:** ~400-500 строк SKILL.md.

**P0 правки от reviewer:**

- ⭐ **Размер блоков 8-10 опций** (по UX-reviewer — UI плывёт после 12-15, оптимум 8-10). НЕ 15-25.
- ⭐ **Длина каждой опции ≤35 символов** (для iPhone): цитата ≤25 + источник в скобках ≤10. Длиннее — обрезать многоточием.
- ⭐ **Новый порядок 7 блоков (правка от маркетинг-reviewer — lingvo блок 5, не 2):**
  1. Боли (12 опций) — узнавание сразу
  2. Желания (10 опций) — зеркало боли
  3. Возражения (12 опций по типам) — что мешает закрыть боль желанием
  4. Метафоры (8 опций) — как они описывают
  5. **Лингво-маркеры** (15 опций) — общий звук этих людей (синтезирующий блок после фактов)
  6. Каналы (8 опций) — где живут
  7. Конкуренты (8 опций) — кого знают
  - Итого: ~73 опции (вместо 113 в старой версии)
- ⭐ **Источники в опциях ≤25 символов** в parenthesis-tail. `«Я топчусь на 200К» (vc.ru ×3)` вместо `[vc.ru × 3 + YouTube × 2]`
- ⭐ **Опция «✏️ Допишу свою» — отдельная финальная фаза** после всех 7 блоков, не follow-up в каждом блоке
- ⭐ **Pre-flight gate:** если в блоке 1 (боли) клиент отметил <40% — STOP, *«мы целились не туда, возвращаемся в Phase B»*
- ⭐ **🟠 FROM CLIENT обязательная атрибуция**: «откуда цитата» — «слышал от Маши 12.04», «чат с Иваном». Без атрибуции = 🔴 BACKLOG автоматически.
- ⭐ **🟡 SIGNAL разделить:** 🟡 «надо проверить» (1-2 непопадания) vs 🔵 «не наше» (5+ непопаданий → возврат в Phase B)
- ⭐ **🟢 GOLD единый стандарт:** ≥3 источника интернета + клиент. Без двойного значения.

### Цветовая схема — упрощено до 4 цветов (по UX-reviewer P0)

❌ **Старая схема 6 цветов** (🟢 GOLD / 🌟 GOLD-VIP / 🟡 SIGNAL / 🔵 не наше / 🟠 FROM CLIENT / 🔴 BACKLOG) — нетехнический клиент не удержит 6 категорий.

✅ **Новая схема 4 цвета (для клиента в `segment-portrait.md`):**

| 🟠 | от тебя лично (с атрибуцией откуда) | от тебя лично |
| 🔵 | не наше (5+ непопаданий клиента → возврат в Phase B) | не наше |

🌟 **GOLD-VIP уходит во внутренний скоринг Алекса** — клиенту не показывается. Используется Алексом для определения hero-цитат для лендинга.

🔴 **BACKLOG отдельный блок** в `segment-portrait.md` секция «Backlog», не в основной разметке.

## 10. Risks & Mitigation

| Риск | Митигация |
|------|-----------|
| **AskUserQuestion не работает в текущей версии Claude Code** | Fallback на текстовый формат «отметь номера 1, 5, 7» с явным форматом ответа |
| **YouTube transcripts блокируются** (datacenter IP block с конца 2024) | Fallback: без транскриптов работаем по топ-100 комментов через `commentThreads.list` Data API. Транскрипты — bonus, не критичный путь |
| **Bukvarix free key устарел** | Регистрация (новый шаг 0.5 в Setup Guide) → личный free key до 1000 ключей/день |
| **Telegram аккаунт забанили за parsing** | Fallback: новый SIM → перерегистрация Application → восстановление session_string. На время бана — Level 1 (только публичные t.me/s/ посты без комментов). Setup Guide должен предупредить про **отдельный SIM** явно |
| **Forum unofficial APIs мёртвые** (vcru-api, pikabu, dtf — community libs) | **Основной путь = WebSearch + WebFetch** с фильтрами. Unofficial APIs — Level 3 опционально, не критично |
| **YouTube Data API quota превышена (10K units/день)** | Caching результатов в `intel/{slug}/_cache/` + fallback на youtube-comment-downloader (без API) |
| **Telegram аккаунт забанили за parsing** | Использовать отдельный SIM, бережные delays ≥2 сек, ловить FLOOD_WAIT |
| **SerpAPI free 100/мес исчерпан** | Fallback на Google Suggest напрямую + Bukvarix (без PAA, но 70% покрытия) |
| **Никита не хочет setup ключей** | Level 1 fallback (только WebSearch + WebFetch) — 5/10 качество, но работает |
| **Старые проекты ломаются после переименования файлов** | Migration через `/audience-status` + детектор старых имён в `/audience-resume` |
| **Markitologические правила «10 вариантов вместо 2-3» потеряются при сжатии** | Закреплено в core.md + повторено в каждом SKILL.md в anti-patterns |
| **Phase B возвращает мусор без качественных источников** | Match-методика с unique_sources ≥ 3 + жёсткие фильтры в каждом субагенте |
| **Stage Lock пробивается при настойчивости клиента** | Escalation pattern после 3 настаиваний — выносим решение клиенту с объяснением риска преждевременного запуска |

## 12. Финальное напутствие

**Главный сдвиг архитектуры:** Алекс перестаёт быть «опросником» (50 вопросов клиенту) и становится **researcher-аналитиком** (сам идёт в интернет через 5 субагентов с реальными API/MCP, приходит с готовым жирным VoC, клиент валидирует за 7 минут через мультиселекты).

**JTBD клиента закрывается:** «помоги распаковать ЦА жирно — но не заставляй меня писать кучу текста». Pull (приходи с готовым) > Push (опросов слишком много) + Anxiety (наберёшь мусора) + Habit (AI всегда спрашивает много).

**Эталон финала:** VoC «Эксперт в потолке» Никиты — каждый сегмент клиента = такой же артефакт.

**Что Никита получает:**
- Алекс который реально готов к production
- Setup guide на 25 минут (5 шагов)
- $0/мес на старте
- 2-3 часа активного времени клиента вместо 10-12
