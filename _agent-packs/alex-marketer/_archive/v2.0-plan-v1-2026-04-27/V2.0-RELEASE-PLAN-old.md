### 4.1 Hormozi Value Equation — trade-off + Perceived (30 мин)

В `knowledge/hormozi-offer-market.md` секция Value Equation:

```
Value = (Perceived Dream × Perceived Likelihood) / (Perceived Time × Perceived Effort)

Все четыре переменные — perceived, не реальные. У Хормози это центр методики.

TRADE-OFF приёмы (как трейдить переменные):
1. Не можешь увеличить Dream — снижай Effort через done-for-you
2. Не можешь снизить Time — увеличивай Likelihood через гарантию
3. Не можешь снизить Effort — повышай Dream через бонусы и истории
4. Низкий Likelihood — risk reversal (двойная гарантия / pay-only-if-results)
```

### 4.2 4 силы Christensen — собрать в одну схему (30 мин)

В `skills/segments-unpack/SKILL.md` Шаг 4 (7 блоков интервью):

Сейчас Anxiety/Habit в блоке 2.1, Push/Pull в блоке 2.6 — разнесены.

Добавить новый под-блок 2.6.1 «Equation 4 сил Christensen»:
```
Уравнение покупки: Push + Pull > Anxiety + Habit

Push — что выталкивает из текущей ситуации
Pull — что притягивает к новому решению  
Anxiety — что страшит в новом решении (страхи "Я" и "Ты")
Habit — что держит в старой привычке

Если Push + Pull < Anxiety + Habit — клиент не купит, даже если оффер хороший.
```

### 4.3 Schwartz: явное признание русификации (15 мин)

В `knowledge/schwartz-awareness.md`:

```
## Каноничный 5-уровневый Schwartz vs наша 4-уровневая сборка

Schwartz (Breakthrough Advertising):
1. Unaware — не знают что есть проблема
2. Problem-Aware — знают проблему, не знают решений
3. Solution-Aware — знают решения, не знают тебя
4. Product-Aware — знают тебя, выбирают
5. Most Aware — знают, готовы купить, ждут оффера

Наша русифицированная сборка (4 уровня):
1. Indifference (≈ Unaware)
2. Awareness (≈ Problem-Aware)
3. Consideration (≈ Solution + Product Aware слитые)
4. Choice (≈ Most Aware)

Почему слили 3+4 Schwartz: на русском рынке граница между Solution-Aware и Product-Aware размыта (одни и те же эксперты работают на оба уровня), плюс под Product-Aware у нас отдельная фаза работы Producer'а (оффер).
```

## ФАЗА 9 — Real Market Research + Wayfinding (~3.5 часа) ⭐⭐ КРИТИЧНО

### 9.1 YouTube research встройка (45 мин)

Новый под-скилл `youtube-research` (или встройка в `/competitors-research` Шаг 6):

- WebSearch: `youtube.com + "{ниша} + {боль}"` — топ-20 видео
- Извлечь: заголовок, просмотры, дата, топ-3 коммента
- Метрика залетания: views/days × engagement
- Output: `intel/competitors-{slug}/youtube-cuts.md` — топ-15 залетающих заголовков дословно + цитаты комментов
- **Зачем:** заголовки YouTube — это то на что **алгоритм бьёт у этой ЦА**. Это сырьё для копирайтинга и для понимания болей.

### 9.2 Search query research встройка (30 мин)

Новый под-этап в `/competitors-research`:
- WebSearch с операторами `"people also ask" + {ниша} + {боль}`
- Если доступно — Wordstat API (RU) и Google Trends
- Извлечь: топ-20 запросов с частотностью (если есть)
- Output: `intel/competitors-{slug}/search-queries.md` — реальные запросы дословно
- **Зачем:** запросы = то что человек НАБИРАЕТ когда ищет решение. Это подтверждённый сигнал боли.

### 9.3 Forum / community research встройка (30 мин)

В `/competitors-research`:
- WebSearch + конкретные платформы (vc.ru, Pikabu, Habr, профильные форумы)
- Telegram-каналы через `t.me/s/<channel>` — посты с комментами
- Извлечь топ-10 постов с реальными комментами под ними
- Output: добавить в `voice-of-customer.md` секцию «Реальные цитаты из интернета» с источниками

### 9.4 Match-методика — алгоритм сведения (30 мин)

В `/segments-unpack` Шаг 7 (веса болей) — обновить:

**Каждая боль в финальном artifact имеет:**
```
evidence_count: N
sources:
  - youtube-comment-1 (дословная цитата + ссылка)
  - search-query (точная фраза + частотность если есть)
  - forum-post (цитата + ссылка)
  - client-interview (если был кастдев)
  - client-mention (если только клиент сказал)
```

**Алгоритм:**
- N ≥ 3 разных типа источников → 🟢 валидированная боль
- N = 2 → 🟡 вероятная, требует валидации в первых продажах
- N = 1 (только клиент) → 🔴 [ГИПОТЕЗА], требует подтверждения

### 9.5 Финальный жирный artifact (45 мин)

Расширить структуру `segment-portrait.md` (бывший dossier.md) — теперь обязательно:

- **15 болей дословно** (не 5-7) — реальные слова ЦА с evidence_count
- **10+ хотелок** дословно
- **5+5 страхов** (Ты + Я)
- **5+5+5 возражений** (ценовые + ценностные + доверительные)
- **20 поисковых запросов** (топ из search query research)
- **15 залетающих заголовков YouTube** (топ из youtube research)
- **10+ цитат из реальных комментов / постов** с источниками
- **«Без чего не может спать»** — 3-5 самых глубоких болей с расширенным контекстом

Это **жирный artifact**, который клиент скидывает ментору, и ментор видит реальную картину, не воздушную.
