# Audience Framework — 3W + БПСВ + уровни осознанности + продуктовая матрица

Ядро методологии распаковки ЦА. Русскоязычная школа упаковки экспертов. Служебный файл — клиент не читает.

**Философия одной фразой:** *Маркетинг — это гипотезы и вероятности. Нет дважды-два-четыре. Не делай портрет ЦА пальцем в небо — выброси и сделай заново.*

---

## Один фрейм — три проекции (резолверная карта)

Жирная ЦА описывается через ОДИН фрейм с тремя проекциями. Это не три разных подхода — это **один концептуальный объект, рассмотренный с трёх углов**:

| Проекция | Назначение | Где применяется |
|----------|-----------|-----------------|
| **1. Философская — 3 круга жирной ЦА** | *Зачем* мы сужаем — пересечение 3 условий критично | Объяснение клиенту *«почему 1 жирный сегмент лучше 8 средних»* |
| **2. Проверочная — 4 угла multi-perspective** | *Что не пропустили* — каждая гипотеза прогоняется через 4 проверки | Pre-scoring gate в `/segments-discover` Этапы 5-7 |
| **3. Операционная — 7 осей scoring** | *Как количественно ранжировать* — 7 баллов 1-5 каждый | Финальный scoring на `/segments-discover` Этап 8 |

### Mapping (как одно соответствует другому)

```
ФИЛОСОФСКАЯ (3 круга)         ПРОВЕРОЧНАЯ (4 угла)         ОПЕРАЦИОННАЯ (7 осей)
─────────────────────         ────────────────────          ─────────────────────
Круг 1: Ресурсы клиента  ─→  Угол 1: Ресурсы          ─→  Ось 7: Соответствие ресурсам
Круг 2: Острая боль       ─→  Угол 3: Боль             ─→  Ось 1: Горячесть (awareness)
                                                              + Ось 4: Доказательная база
Круг 3: Платёжная         ─→  Угол 2: Рынок            ─→  Ось 2: Объём рынка
        способность                                          + Ось 3: Платёжеспособность
                                                              + Ось 5: Доступность канала
                              Угол 4: Конкуренция      ─→  Ось 6: Whitespace vs Red Ocean
```

**Логика проекций:**
- 3 круга — для **разговора с клиентом** (живая речь, философия)
- 4 угла — для **внутреннего фильтра** до scoring (если хоть один угол пуст — гипотеза 🔴, не идёт в scoring)
- 7 осей — для **финального количественного отбора ТОП-3** на Этапе 8

**Не путать с тремя разными методиками. Это одна сущность с разной разрешающей способностью.**

## 3 круга жирной ЦА (диаграмма пересечения)

Жирная ЦА = пересечение 3 кругов. Без всех трёх — это не ЦА, это аудитория. Аудитория ≠ деньги.

```
        ┌─────────────────────┐
        │ 1. РЕСУРСЫ КЛИЕНТА  │
        │ что он может дать   │
        │ (бэкграунд, кейсы,  │
        │ уникальность, связи)│
        └─────────┬───────────┘
                  │
       ┌──────────┴──────────┐
       │                     │
       │      🎯 ЖИРНАЯ ЦА   │
       │                     │
       └──────┬───────┬──────┘
              │       │
   ┌──────────┘       └──────────┐
   │                             │
┌──┴───────────────┐  ┌──────────┴────┐
│ 2. ОСТРАЯ БОЛЬ   │  │ 3. ПЛАТЁЖНАЯ  │
│ кому критично    │  │  СПОСОБНОСТЬ   │
│ (не "было бы      │  │ кто реально    │
│  неплохо")       │  │ платит деньги  │
└──────────────────┘  └────────────────┘
```

**Расширение до 4 углов scoring:**

| # | Угол | Вопрос | Источник данных |
|---|------|---------|-----------------|
| 1 | **Ресурсы** | Что у клиента в багаже из чего можно построить уникальный продукт? | Прямые вопросы клиенту, `client-profile.md`, кейсы |
| 2 | **Рынок** | Где в нише жирные сегменты — большой объём × платят высокие чеки? | `/segment-money-map` через WebSearch |
| 3 | **Боль** | Где острейшая боль которую клиент может решить? | Voc клиента, кастдев, форумы ЦА |
| 4 | **Конкуренция** | Где конкуренты слабые или отсутствуют? | `/segment-money-map` (голубой океан) |

Пересечение всех 4 = жирная ЦА. Если хоть один угол отсутствует — пересмотри.

⚠️ **Точка ≠ тёплая база.** Тёплые покупатели (по сарафану, прежним связям) — это **ошибка выжившего**. Используй их как **сигнал паттерна**, но размер рынка / чек / конкуренция считаются по большой нише, не по 8 знакомым.

## Главный принцип

> Если анализ ЦА сделан поверхностно — всё остальное уже не имеет смысла. Ни техничка, ни контент, ни триггеры продаж. Анализ — фундамент. Если он плохой, всё на нём падает.

ЦА — **первое что делается и единственное от чего зависит всё остальное**. Остальное — оптимизация.

## Фреймворк 3W (упрощённый Sherrington 5W)

Режет классический 5W до трёх ключевых вопросов, шаг за шагом:

```
Боль / Потребность
    ↓
What  — что покупает/делает потребитель для решения проблемы
    ↓
Why   — зачем он это делает (мотивация)
    ↓
Who   — кто тот самый потребитель (это и есть сегмент)
```

### Процедура

**Шаг 1. Выписать 10-15 гипотетических БОЛЕЙ ЦА.**
Много, не только очевидные. Первые 3-4 всегда будут порожняк — это ассоциативный ряд поверхности. Копай глубже.

Формат боли: *«у меня нет X / мне плохо / мне больно / не хватает».*

**Шаг 2. К каждой боли — ПОТРЕБНОСТЬ.**
Формат: *«хочу X / мне нужно / мне требуется».*

Правило: **боль ≠ потребность**. Боль — что болит, потребность — что хочется. Путают постоянно.

Как переводить: «если у тебя [боль] — то что ты хочешь?» → ответ = потребность.

**Шаг 3. К каждой паре (боль, потребность) — WHAT.**
*«Что покупает или делает потребитель для решения этой проблемы / реализации хотелки?»*

Важно: не только **покупает**, но и **делает**. Может не покупать — может откладывать под подушку, может ничего не делать, может ходить на бесплатные семинары.

**Шаг 4. К каждому WHAT — WHY.**
*«Почему / зачем потребитель это покупает или делает?»*

Драйверы покупки. Истинные мотивы.

**Шаг 5. К каждому WHY — WHO.**
*«Кто тот самый потребитель, который покупает это по этой причине?»*

Это **сегменты**. Вырастают именно здесь, после Why.

На выходе — **10-20 потенциальных сегментов**.

### Отбор 3 сегментов

Из всего списка выбирается **3 сегмента** разной теплоты:

| Тип | Критерий | Вера |
|-----|----------|------|
| **Горячий** | Точно знает проблему, ищет решение, готов платить | ≥ 85% |
| **Тёплый** | Знает проблему, интересуется, но не решился | ≥ 60% |
| **Неочевидный/холодный** | Проблема есть, но не осознаёт или не приоритизирует | ≥ 40% |

**Правило:** постарайся выбрать одну аудиторию, на которую **точно** зайдёт. Прямо интуитивно чувствуешь — она наша.

## 4 уровня осознанности

Пятый уровень Loyalty не считаем — там уже оплата. Работаем с 4:

| Уровень | Что на нём | Как тянуть |
|---------|-----------|-----------|
| **1. Indifference** (безразличие) | «Мне не надо / я не слышал / мне норм» | Образовательный контент о теме, не продукте |
| **2. Awareness** (осведомлённость) | «А что это? Интересно, расскажи» | Мотивационный контент, кейсы, разборы |
| **3. Consideration** (рассмотрение) | «А мне это подходит? Надо / не надо?» | Доказательства, сравнение, примерка на себя |
| **4. Choice** (выбор) | «Да или нет? Эти или те?» | Оффер, цена, гарантия, CTA |

**Критическое правило:**
- Уровни 1-2 = разговор о **теме** (большой области)
- Уровни 3-4 = разговор о **продукте** (конкретном решении)

**Нельзя давать цену человеку на уровне 1.** Он ещё не знает зачем это надо — ему про цену бессмысленно.

### Как распределять тезисы респондентов

В интервью клиент говорит разное. Каждый тезис относится к какому-то уровню:

- «Мне насрать, не думал об этом» → Indifference
- «Слышал что-то, интересно было бы разобраться» → Awareness
- «Подойдёт ли это мне, смогу ли я?» → Consideration
- «А точно у вас, а не у них? А за эту цену?» → Choice

Агент раскладывает тезисы по уровням, видит **откуда тянуть и куда тянуть** сегмент.

## БПСВ — Боли / Потребности / Страхи / Возражения

Полная структура описания сегмента.

### Соцдем
- Пол (м/ж/оба)
- Возраст (вилка, не точка)
- Доход (вилка, приблизительно)
- Профессия / роль
- География

### Боли (Б)
Формула: *«у меня нет, мне плохо, болит, не хватает»*
Минимум 5-7 на сегмент. Первые 2-3 очевидные, потом копай глубже.

### Потребности (П)
Формула: *«хочу, мне нужно, требуется»*
Зеркало болей — по одной на каждую боль.

Проверка: потребность = **желаемая трансформация**. Если не трансформация — переформулируй.

### Страхи (С) — делятся на 2 категории

**1. «Ты как продукт»** — страхи про тебя и твой продукт:
- *«Ты некомпетентен»*
- *«У тебя нет кейсов»*
- *«Я не верю в твою методологию»*
- *«Твой продукт новый, необкатанный»*
- *«Упаковка твоего продукта не внушает доверия»*

Минимум 3.

**2. «Я как потребитель»** — страхи про себя:
- *«У меня не получится»*
- *«Я не смогу применить это»*
- *«Это не для меня»*
- *«Я слишком ленив/занят/стар»*
- *«Боюсь взять на себя ответственность»*

Минимум 3.

### Возражения (В) — 3 типа

**1. Ценовые** (минимум 2):
- *«Дорого»*
- *«За эти деньги я могу...»*
- *«Не в рынке»*

**2. Ценностные** (минимум 2):
- *«Не нужно мне это»*
- *«Уже проходил подобное, не зашло»*
- *«Ничего нового не узнаю»*
- *«Не решит мою реальную проблему»*

**3. Доверительные** (минимум 2):
- *«Не доверяю тебе как человеку»*
- *«Нет кейсов которые бы я увидел»*
- *«Кто ты такой?»*
- *«Твоя упаковка не внушает»*

Формула возражения: *«Да, но...»*

### Главный принцип БПСВ

**Каждое возражение должно быть перевёрнуто в плюс контентом воронки.**

Воронка = это **восполнение недостающей информации по БПСВ**. Что клиент недоспросил — то ему и даёшь последовательно.

## Продуктовая матрица — переехала в Stage 2

Lead Magnet → Tripwire → Core Offer → Profit Maximizers → Return Path + два типа мотивации (избежать боли / получить удовольствие) + Логика vs Эмоция (гендерный паттерн) — это Stage 2 (Product) контент.

См. `knowledge/hormozi-offer-market.md` секция «Lead Magnet по Хормози» + раздел «Продуктовая матрица».

⚠️ В Stage 1 (Audience) продуктовая матрица **не разрабатывается** — это нарушение Stage Lock. Если клиент сам тянет в продукт во время audience-фазы — фиксируем в `inbox/_product-hints.md`, не углубляемся (см. `knowledge/stage-lock.md`).

## Главное правило в одной строке

> *Маркетинг работает исключительно на гипотезах в которые мы верим. Задача — двигаться по протоптанному 100 лет назад пути, а не вслепую.*

**Следствие для агента:** любое утверждение о ЦА/оффере/рынке — **гипотеза**, пока не проверено 5+ интервью и 20+ дословных цитат.
