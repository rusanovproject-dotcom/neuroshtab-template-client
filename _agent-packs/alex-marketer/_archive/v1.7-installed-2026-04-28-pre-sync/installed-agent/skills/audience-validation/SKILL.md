# `/audience-validation` — Phase C пайплайна

📍 **Где это в пайплайне:** Phase C из 4 (Quick Capture → Internet Research → **Validation** → Awareness-lite).

> **Цель:** клиент за 15-25 минут активной работы валидирует **жирный VoC из Phase B** через 7 мультиселектов. Каждая отметка → переводит цитату из 🟡 SIGNAL в 🟢 GOLD (если из интернета подтверждено + клиент узнал из своей практики).

> **Маркетинговая логика:** клиент не **создаёт** портрет — он **подтверждает узнавание** того что Алекс уже собрал. Это снимает с клиента когнитивную нагрузку («сформулируй 10 болей с нуля» — невозможно за разумное время) и одновременно даёт Алексу **двойную валидацию**: данные есть в интернете И клиент узнал в своей реальной практике.

## 1. Pre-flight checks (БЛОКИРУЮЩИЕ)

```bash
# 1. Phase B закрыта для этого сегмента?
[ -f "intel/{slug}/voice-of-segment.md" ] || STOP "Phase B не пройдена → /audience-internet-research"

# 2. voice-of-segment.md содержит минимум валидируемого материала?
grep -c "🟢\|🟡" intel/{slug}/voice-of-segment.md  # ожидаем ≥ 50 строк-цитат
# Если меньше 30 цитат болей — пометить [partial] и предложить re-run Phase B

# 3. agent-state.md показывает active_phase: ready_for_C?
yq '.active_phase' agent-state.md || STOP "Phase C запускается только после Phase B"

# 4. Какой сегмент валидируем (если несколько)?
# Если в state.md один target_segment — запускаем для него
# Если несколько — ASK клиента «начнём с hot или с warm?»
```

**Wayfinding (обязательно в начале):**

> *«📍 Phase C из 4 — валидация сегмента {имя}. Что будет: я уже собрал жирный портрет ({N} болей, {M} желаний, {K} возражений с источниками). Дальше — 7 коротких блоков, в каждом ты галочками отмечаешь что попадает + допишешь 2-3 свои если есть. На каждый блок ~1-2 минуты. Время: 15-25 минут.*
>
> *Один сегмент за раз — не валидируем три сразу, ты устанешь. После — переходим к следующему.*
>
> *Поехали с {имя сегмента}?»*

**Stop+wait:** клиент подтверждает.

## 2. Архитектура — 7 мультиселектов

⭐ **Маркетинговая логика порядка блоков:** идём от **сильных эмоций → к деталям**. Боли первые (узнавание сразу включает чувство), возражения позже (требует рефлексии), конкуренты в конце (ratio, не emotion).

| # | Блок | Что валидируем | Из какой секции voice-of-segment.md |
|---|------|----------------|-------------------------------------|
| 1 | **Боли** | 15 топ-болей по подкатегориям | «Цитаты БОЛИ» |
| 2 | **Желания** | 10-12 желаний дословно | «Цитаты ЖЕЛАНИЯ» |
| 3 | **Возражения** | 12-15 возражений по типам | «Цитаты ВОЗРАЖЕНИЙ» |
| 4 | **Лингво** | 20-25 слов/оборотов | «Лингво — характерные слова» |
| 5 | **Метафоры** | 10-12 метафор | «Метафоры» |
| 6 | **Каналы** | 8-10 каналов/сообществ | «Каналы и сообщества» |
| 7 | **Конкуренты** | 8-10 конкурентов | «Конкуренты-учителя» |

## 3. Шаблон AskUserQuestion для каждого блока

### Блок 1 — Боли (1-2 мин)

```yaml
question: |
  🎯 Какие из этих 15 болей реально слышишь у своей ЦА?
  (отметь галочками + допиши 2-3 свои дословно если есть)
multiSelect: true
header: "Боли — выбери что попадает"
options:
  - id: pain-01
    label: "«Я топчусь на 200К и не понимаю как сдвинуться» [vc.ru × 3 + YouTube × 2]"
  - id: pain-02
    label: "«Кручусь как белка в колесе, толку ноль» [Forbes + 4 цитаты]"
  - id: pain-03
    label: "«Достало, устал и хочу всё бросить» [holod.media + citydog.io]"
  ... (15 вариантов)
  - id: pain-other
    label: "✏️ Допишу свою боль — открыть текстовое поле"
```

⭐ **Маркетинговая логика выбора 15:**
- 5 топ-VALIDATED (🟢, ≥3 источника) — главные кандидаты
- 5 SIGNAL (🟡, 2 источника) — те которые нужно добить клиентом
- 5 неожиданных / контр-интуитивных (если есть) — проверка не упустили ли что-то важное

**После ответа клиента:**
- Отмеченные 🟢 → подтверждены (интернет ≥3 источника + клиент узнал) → **GOLD статус**
- 🌟 GOLD-VIP → single-source но >200 лайков на vc.ru / >500 на YouTube — для hero-блока лендинга
- Не отмеченные 🟡 → 1-2 непопадания → **«требует custdev»** | 5+ непопаданий 🔵 → **«не та аудитория, возврат в Phase B»**
- Дописанные клиентом 🟠 FROM CLIENT → **обязательная атрибуция** «откуда цитата» («слышал на созвоне с Машей 12.04», «чат с Иваном в TG»). Без атрибуции — автоматически 🔴 BACKLOG, не идёт в финальный portrait.

⚠️ **«✏️ Допишу свою» — НЕ follow-up в каждом блоке** (поток ломается). Единый финальный turn после всех 7 блоков:

> *«Окей, прошли 7 блоков. Теперь — что я пропустил? 5 минут на дописать дословные цитаты которые ты слышал и которых тут не было. По одной за раз через follow-up.»*

Каждая дописанная цитата проходит через follow-up «откуда?» — без атрибуции отбрасывается.

```yaml
question: |

options:
```

```yaml
question: |
  🛡 Какие возражения встречал у своих клиентов / в нише?
  (галочки + допиши)
header: "По типам: про курсы / коуча / AI / ассистентов"
options:
  - id: obj-courses-01
    label: "«А вдруг это очередной курс, и я опять брошу?» [otzovik]"

  - id: obj-time-03
    label: "«Сейчас нет времени всё это делать»"
  ... (12-15 по типам)
```

⭐ Маркетинговая логика: возражения — самый чувствительный блок. Группируем по 4 типам Хормози (trust / capability / time / ROI) ИЛИ по предметным группам (курсы / AI / ассистенты / продюсеры) — что лучше отзовётся у клиента.

### Блок 4 — Лингво (1 мин)

```yaml
question: |
  🗣 Какие слова и обороты ТЫ ТОЧНО слышал у своей ЦА?
  (галочки + допиши свои)
multiSelect: true
options:
  - id: ling-01
    label: "«упёрся в потолок» / «упёрлась»"
  - id: ling-02
    label: "«белка в колесе»"
  ... (20-25)
```

⭐ Маркетинговая логика: лингво — это **прямой материал для копирайтера**. Каждое валидированное слово/оборот = можно использовать на лендинге как **узнаваемый язык**. Слова что клиент НЕ отметил — НЕ использовать (значит они не из его аудитории).

### Блок 5 — Метафоры (1 мин)

```yaml
question: |
  🎨 Какие метафоры точно работают у твоей ЦА (узнаёшь?)
options:
  - id: met-01
    label: "Белка в колесе — Forbes + 8 комментов"
  - id: met-02
    label: "Театр одного актёра — vc.ru"
  ... (10-12)
```

### Блок 6 — Каналы (1 мин)

```yaml
question: |
  📡 Где твоя ЦА реально живёт в инфо-пространстве?
```

⭐ Маркетинговая логика: канал валидированный клиентом = **подтверждённый канал дистрибуции** для Stage 3 (воронка). Каналы которые клиент НЕ отметил = лидируют в шумовой канал, не использовать в первой воронке.

### Блок 7 — Конкуренты (1 мин)

```yaml
question: |
  🥊 Кто из этих конкурентов реально в твоей нише?
  (отметь тех с кем точно сравниваются твои клиенты + допиши пропущенных)

  - id: comp-01
    label: "Аяз Шабутдинов / Like Центр — 500₽-5М, бизнес-курсы"
  - id: comp-02
    label: "Радислав Гандапас — 200К, лидерство"
  - id: comp-03
    label: "Visotsky Consulting — от 100К, системный бизнес"
  ... (8-10)
```

⭐ Маркетинговая логика: валидированные конкуренты = **реальное конкурентное поле**. Не отмеченные = просто упоминания которые субагент 4 нашёл, но они не реальный compete для этого клиента — убираем из дальнейших анализов.

```
Шаг 1 — Pre-flight (см. секцию 1)

Шаг 4 — Запустить блок за блоком ПО ОДНОМУ через AskUserQuestion:
  for block in [pains, wants, objections, lingvo, metaphors, channels, competitors]:
    1. Подготовить options (15-25 пунктов)
    2. AskUserQuestion с multiSelect=true + опция «допишу свою»
    3. Получить ответ → сохранить в memory
    4. Если есть «допишу свою» → задать follow-up «опиши дословно»
    5. Wayfinding: «✅ Блок {N} из 7 закрыт. Дальше → блок {N+1}»

Шаг 5 — Собрать финальный segment-portrait.md (см. секцию 5 ниже)

Шаг 6 — Update agent-state.md → segment {slug} validated

Шаг 7 — Wayfinding: «✅ Phase C закрыта для {slug}. Готов перейти к следующему сегменту?»
  Stop+wait: или следующий сегмент → /audience-validation для него, или Phase D для hot
```

⚠️ **Между блоками — короткое подтверждение, не длинная пауза.** Клиент в потоке валидации, не отвлекать длинными комментариями.

## 7. Anti-patterns

❌ **Запустить все 7 блоков одной простынёй** — клиент перегрузится, отметит наугад. Только по одному за раз.

❌ **Не использовать AskUserQuestion** (написать «отметь номера 1, 5, 7» текстом) — клиент пишет неаккуратно, парсинг ломается. Используем мультиселект Claude Code.

❌ **Включить в options 50+ пунктов на блок** — выбор парализует. Максимум 25, лучше 15-20 топ-кандидатов.

❌ **Не показывать атрибуцию источника в options** — клиент не понимает откуда взято. Каждая опция должна иметь `[источник × N]`.

❌ **Сохранить в финал только 🟢 GOLD** — теряем SIGNAL цитаты которые могут быть подтверждены в продажах. Все 🟢/🟡/🟠 идут в portrait, BACKLOG (🔴) — нет.

❌ **Валидировать 3 сегмента подряд без перерыва** — клиент устаёт после первого. Один сегмент за раз, между — пауза.

❌ **Произносить в чате «AskUserQuestion / multiSelect / VALIDATED / SIGNAL»** — это технический жаргон. Клиенту: «выбираем галочками», «подтверждённое × проверим в продажах × из интернета».
