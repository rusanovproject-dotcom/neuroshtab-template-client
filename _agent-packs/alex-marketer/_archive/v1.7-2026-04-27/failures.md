## Failures

### 2026-04-27 — Срыв scope из Stage 1 в Stage 2 во время свободного диалога (без активного скилла)

**Что предположил:** «найти точку приложения усилий» = найти нишу + продукт + чек + бизнес-модель + позиционирование. То есть «лазерная фокусировка» — это полный маркетинговый разбор сразу.

**Что оказалось:** «найти точку» = только сегмент ЦА (NORTH-STAR + 3 dossier + voc + lingvo). Продукт / чек / тариф / бизнес-модель / timeline / позиционирование — это Stage 2-4, **другие скиллы**, **другая фаза**, между ними обязательный stop+wait.

**Как сорвался:** клиент попросил «давай прямо сейчас приступай» к распаковке ЦА. Алекс пошёл в WebSearch и 3 субагентов **без активации скилла** `/audience-stage`. После money-map по нишам — стал упаковывать продуктовую архитектуру (Tier Low/Mid/High/Premium с чеками 60К/180К/300К), бизнес-модели (one-time / subscription / hybrid revenue-share), позиционирование (Tiago Forte / Allie Miller / уникальное), timelines запуска (12 мая / июнь / сентябрь). Решал за клиента: «Atlas как самостоятельный продукт за 5К — отменяется». Числа без `[ГИПОТЕЗА N%]`.

**Когда клиент попросил «вернись к ЦА»** — Алекс выбрал lite-режим `/marketer-revision` (5 вопросов за 20 мин) вместо полного `/audience-stage`. Снова сам решил за клиента + использовал lite-режим там где Stage 1 не была закрыта.

**Правила на будущее (железные):**

1. **Stage 1 закрыта только** когда есть NORTH-STAR + 3 dossier + 3 base + voc + явный `/accept` владельца. До этого момента **запрещено** говорить про tier'ы / цены / бизнес-модели / timelines / позиционирование / оффер. См. core.md секцию «🔒 Stage Lock».

2. **На тему ЦА — только через скилл.** В свободном диалоге guardrail'ы из core.md превращаются в напоминалки. Триггеры «распакуй ЦА / поехали / разбери / вернись к ЦА» — немедленный запуск `/audience-stage` без свободных WebSearch и субагентов.

3. **Lite-режим (`/marketer-revision`) запрещён пока Stage 1 не закрыта.** После любого срыва / возврата к ЦА — только полный `/audience-stage`.

4. **Если клиент сам тянет в продукт** — записать в `inbox/_product-hints.md` как гипотезу `[PRODUCT/OFFER/PRICE]`, не углубляться, вернуть в текущий Шаг.

5. **Внутри `/audience-stage`** на Шаге 4a (вопросы про продукт/уникальность) — продуктовые ответы клиента **фиксировать в `_product-hints.md`**, не разворачивать в Tier-структуру.

6. **Срыв scope = обязательная запись в failures.md** до продолжения работы. Без этого следующий клиент наступит на те же грабли.
