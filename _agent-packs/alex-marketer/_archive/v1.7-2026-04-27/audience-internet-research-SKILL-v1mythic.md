# `/audience-internet-research` — Phase B пайплайна

📍 **Где это в пайплайне:** Phase B из 4 (Quick Capture → **Internet Research** → Validation → Awareness-lite).

> **Цель:** для каждой гипотезы сегмента из `hypotheses.md` собрать **жирный VoC с реального интернета** через 5 параллельных субагентов. На выходе — `voice-of-segment.md` × N сегментов готовый для валидации клиентом в Phase C.

> **Маркетинговая логика:** реальный голос ЦА живёт **в интернете**, не в голове клиента. Клиент дал контекст в Phase A — теперь Алекс **сам идёт за данными** и приходит с готовым. Это меняет роль Алекса: из «опросника» в **researcher-аналитика**.

## 2. Архитектура — 5 параллельных субагентов

Все 5 субагентов запускаются через **Task tool с `run_in_background: true`** в одном сообщении (параллельно). Алекс показывает таблицу-dashboard прогресса.

| # | Субагент | Что копает | Output | Время |
|---|----------|-----------|--------|-------|
| 1 | **YouTube Mining** | Залетающие заголовки + комменты | `intel/{slug}/youtube-cuts.md` | 10-15 мин |
| 2 | **Forum/Community VoC** | vc.ru / Reddit / Pikabu / Telegram через t.me/s/ | `intel/{slug}/community-voc.md` | 10-20 мин |
| 3 | **Search Queries** | Wordstat + Google PAA + suggest | `intel/{slug}/search-queries.md` | 5-10 мин |
| 4 | **Конкурентная карта** | Лендинги + цены + механики + источники | `intel/{slug}/competitors-map.md` | 10-15 мин |
| 5 | **БПСВ-синтезатор** | Запускается ПОСЛЕ 1-4 (когда все вернулись) — собирает паттерны через match-методику | `intel/{slug}/bpsv-patterns.md` + финальный `voice-of-segment.md` | 5-10 мин |

⭐ **Маркетинговая логика параллельности:** субагенты 1-4 не зависят друг от друга. Запускаем все одновременно, экономим 30-40 мин (вместо последовательных 60+).

```

## Match-методика — алгоритм сведения

### Шаг 1 — кластеризация цитат

Все цитаты болей / желаний / возражений из 4 файлов **семантически
кластеризуем** (не по ключевым словам, а по смыслу):

⭐ Пример: «топчусь на 200К» (vc.ru) + «упёрся в потолок» (YouTube коммент) +
«как пробить плато дохода» (Wordstat 12K/мес) + «застрял на одном уровне»
(Reddit) — это ОДИН кластер «потолок и застой».

### Шаг 2 — evidence_count + unique_sources

Для каждого кластера считаем:
- `evidence_count` — общее число цитат в кластере
- `unique_sources` — число РАЗНЫХ типов источников где встречается:
  1. Material from client (Phase A inbox/_raw)
  2. YouTube comments
  3. YouTube headlines
  4. Forum posts
  5. Forum comments
  6. Search queries
  7. Competitors landing pages
  8. Competitors negative reviews

### Шаг 3 — статусы по правилам

- **🟢 VALIDATED** — `unique_sources ≥ 3 AND evidence_count ≥ 5` — идёт в финальный portrait, на этом строим лендинг
- **🟡 SIGNAL** — `unique_sources = 2 OR evidence_count = 3-4` — идёт в portrait с пометкой «требует валидации в продажах»
- **🔴 HYPOTHESIS** — `unique_sources = 1` (только клиент / только YouTube без подтверждения) — в backlog, не в финал

### Шаг 4 — финальный voice-of-segment.md

⭐ Структура **ровно** как эталонный VoC «Эксперт в потолке» (см. ниже шаблон).

## 11. Anti-patterns

❌ **Запустить субагентов последовательно** — теряем 30+ минут. Только параллельно через Task tool с `run_in_background: true`.

❌ **Принять output субагента без match-методики** — сырьё × 4 ≠ финальный артефакт. Subagent 5 обязателен.

❌ **Включить в финал HYPOTHESIS-цитаты** (только 1 источник) — это шум, портит качество.

❌ **Перефразировать цитаты для красоты** — теряем самое ценное (живой язык ЦА). Дословно или ничего.

❌ **Не показывать прогресс клиенту** — клиент думает «Алекс пропал», паника. Dashboard обязателен.

❌ **Запустить Phase B без выбора сегментов клиентом в Phase A** — копаем не там, теряем 60 минут.

❌ **Произнести в чате «BПСВ» / «Match-методика» / «evidence_count»** — это технический язык. Клиенту: «жирные подтверждённые боли», «что повторяется в 5+ источниках».
