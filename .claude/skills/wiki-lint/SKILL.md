# wiki-lint — ревизия графа знаний (Karpathy LLM Wiki)

> Операция **lint** из паттерна Карпати. Канон: `knowledge/ai-offices/distilled/memory-architecture.md` (раздел 1, «3 операции»).
> Не путать с office-cleaner: тот проверяет агентов/роутинг/структуру офиса. Этот — только wiki-граф знаний.

## Что ищет (6 классов проблем)

1. **Дубликаты** — две заметки про одну сущность. Сигнал: похожие заголовки/summary, пересекающиеся теги. Кандидат на слияние.
2. **Сироты** — заметка без входящих И исходящих `[[ссылок]]`. Висит вне графа, агент её не найдёт по навигации.
3. **Висячие концепты** — `[[ссылка]]` на заметку, которой не существует. Либо создать страницу, либо убрать ссылку.
4. **Битые ссылки** — `[[ссылка]]` с опечаткой/неправильным slug.
5. **Противоречия** — две заметки утверждают несовместимое. Помечать `⚠️ CONFLICT: <описание>`, не удалять молча.
6. **Гигиена формата** — нет `Summary` / `Provenance` / frontmatter-полей (type, source, tags, importance); summary >100 символов; не-ASCII или не-kebab-case имя файла.

## Workflow

1. **Определи scope** — какой домен/папку линтим (по умолчанию весь `knowledge/domains/`, либо указанный, напр. `domains/product`).
2. **Собери карту** — Glob всех `*.md` в scope; для каждой вытащи frontmatter, заголовок, `[[ссылки]]` (Grep `\[\[.*?\]\]`).
3. **Прогон 6 классов:**
   - дубли — сравни заголовки/summary/теги (эвристика: сильное пересечение → кандидат);
   - сироты — заметки, на которые никто не ссылается и которые сами не ссылаются;
   - висячие/битые — каждая `[[ссылка]]` → есть ли файл с таким slug;
   - противоречия — заметки с одной темой, но конфликтующими утверждениями;
   - формат — проверь обязательные секции/поля.
4. **🚦 ОТЧЁТ + ГЕЙТ** — структурированный список находок по классам с `file:line`. Спроси:
   «фиксить всё / только важное / по номерам / ничего?».
5. **После approval — фикс:**
   - дубли → слить в одну, перенаправить ссылки, лишнее в `_archive/<дата>/` (не удалять молча);
   - сироты → проставить релевантные `[[ссылки]]` или поднять в `_MOC`;
   - висячие → создать заглушку-заметку (минимум frontmatter+Summary+Provenance) или убрать ссылку;
   - битые → исправить slug;
   - противоречия → пометить `⚠️ CONFLICT`, вынести в отчёт человеку (не решать за него факты);
   - формат → дописать недостающие секции/поля.
6. **Обнови `_MOC.md` + `INDEX.md` + `log.md`** если структура менялась (`## YYYY-MM-DD — lint — <scope>`).
7. **Финальный отчёт** — что исправлено, что осталось на человека (конфликты).

## Guardrails

1. **Ничего не удаляю без approval.** Старое → `_archive/<дата>/`, не в корзину. Первоисточники в `raw/` не трогаю вообще.
2. **Citation rule** — каждая находка с `file:line`, иначе не находка.
3. **Противоречия не решаю сам** — помечаю `⚠️ CONFLICT`, выношу человеку.
4. **Идемпотентность** — повторный прогон без изменений = тот же отчёт.
5. **Не лезу в зону Риты** — агенты/роутинг/skills/офис — это `/office-cleaner`. Я только граф знаний.
6. **Append-only log.**

## Output Contract

Ответ = одно из:
- **Отчёт о ревизии + гейт** — находки по 6 классам с `file:line`, вопрос что фиксить
- **Отчёт о фиксах** — что исправлено, что осталось (конфликты на человека)
- **Чисто** — «прогнал scope, граф в порядке, новых проблем нет»
