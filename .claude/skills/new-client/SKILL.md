# New Client — карточка нового пользователя

## Шаг 1. Собери данные

Живым диалогом, по одному вопросу. Если имя пользователя пришло в триггере — не переспрашивай.

1. **Имя / компания.** "Как зовут? (или название компании)"
2. **Ниша.** "Чем занимается — коротко?"
3. **Источник.** "Откуда пришёл? Реферал, вебинар, холодный контакт, инста — что было?"
4. **Статус.** "На каком этапе? Выбор: лид / в работе / пользователь / бывший."
5. **Контакты.** "Контакты есть — Telegram, email, телефон? (опционально, можно пропустить)"

## Шаг 2. Сделай slug

Из имени — латиница, lowercase, через дефис. Примеры:
- "Иван Петров" → `ivan-petrov`
- "ООО Ромашка" → `romashka`
- "Maria K." → `maria-k`

Если пользователь с таким slug уже есть — добавь суффикс (`ivan-petrov-2`) или спроси пользователя.

## Шаг 3. Скопируй шаблон

```bash
cp -r clients/_template/ clients/<slug>/
```

Если `clients/_template/` пустой — создай карточку с нуля по содержимому шаблона `office/templates/client-profile.md`.

## Шаг 4. Заполни profile.md

Файл — `clients/<slug>/profile.md`. Плейсхолдеры из шаблона:
- `{{CLIENT_NAME}}` — имя
- `{{NICHE}}` — ниша
- `{{SOURCE}}` — источник
- `{{STATUS}}` — статус
- `{{CONTACTS}}` — контакты (или "—")
- `{{DATE_CREATED}}` — сегодняшняя дата YYYY-MM-DD

## Шаг 5. Обнови INDEX.md

Файл — `clients/INDEX.md`. Таблица формата:

```
| Имя | Ниша | Статус | Папка |
|-----|------|--------|-------|
```

Добавь строку:
```
| <Имя> | <Ниша> | <Статус> | [<slug>](./<slug>/) |
```

Если в таблице ещё строка-плейсхолдер (`— | —`) — удали её.

## Шаг 6. Отчитайся

> Готово. Пользователь `<имя>` добавлен → `clients/<slug>/profile.md`
>
> Статус: <статус>. Индекс обновлён.
>
> Если хочешь записать историю общения — добавь файлы в `clients/<slug>/` (звонки, переписка, КП).

## Правила

- em-dash `—`, не `--`.
- Не допрашивай — если пользователь сказал "контакты потом" или пропустил, просто ставь `—` и двигайся.
- Не придумывай данные, которых нет. Лучше `—` чем выдуманное значение.
- Тон — живой, без формальностей.
