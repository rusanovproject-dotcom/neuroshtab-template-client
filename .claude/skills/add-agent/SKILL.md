# /add-agent — подключение нового агента через Демиурга

Клиент просит добавить/создать агента. Вызываем **Демиурга** (мета-агент, архитектор). Он получает контекст из файлов стратегии клиента и собирает персонального агента в диалоге.

## Когда запускается

- Триггеры: "добавь агента", "создай агента", "нужен копирайтер / дизайнер / hermes", "подключи X", "/add-agent <name>", "собери мне команду"
- Автоматически — после `/strategist-roadmap` как финальный шаг онбординга: Стратег передаёт клиента Демиургу со словами "давай создадим твоего первого персонального агента под неделю 1"

## Пререквизиты

Для качественной сборки Демиургу нужны артефакты Стратега:
- `office/strategy/strategy.md`
- `projects/<main>/diagnostic-card.md`
- `projects/<main>/brand/voiceprint.md` (особенно важно для Copywriter)
- `projects/<main>/audience/` (особенно для Producer, Hermes)

**Если этих файлов нет** (клиент не проходил `/strategist-intake`) — скажи:
> "Для персонального агента нужен контекст. Давай сначала короткий онбординг со Стратегом: `/strategist-intake`. Займёт 20-30 минут, зато агент получится твой, не generic. Готов?"

Если клиент настаивает — запускай Демиурга с пометкой `[GENERIC_MODE]`, но предупреди что агент будет шаблонным.

## Шаги

### 1. Brief — выясни что именно создаём (1-3 вопроса)

Задай клиенту по одному:

1. **Кого создаём?** Стандартные роли: Copywriter, Designer, Producer, Hermes, Tech Lead, Intelligence, Brahma (ревьюер), ПАРА (организатор). Или своя — "расскажи что должен делать, в 1-2 предложениях".

2. **Почему сейчас?** Привязка к неделе программы или конкретной задаче ("нужен Copywriter чтобы написать welcome-цепочку из фокуса неделе 2").

3. **Если роль кастомная** — уточни зону ответственности, связки с существующими агентами, output.

### 2. Передай управление Демиургу

Прочитай файлы стратегии (data pack) и запусти Демиурга. В инструкции для Демиурга передай:

```
Входные данные:
- Стратегия: office/strategy/strategy.md
- Профиль клиента: projects/<main>/diagnostic-card.md
- Voiceprint: projects/<main>/brand/voiceprint.md (если есть)
- Audience: projects/<main>/audience/ (если есть)
- Products: projects/<main>/products/

Задача: создать агента "<name>" со специализацией "<role>".
Почему: <reason_from_client>

Следуй своему pipeline (Brief → Mine → Build → Validate). При необходимости задавай клиенту уточняющие вопросы через этот чат.
```

### 3. Демиург создаёт агента

Следует своему pipeline (подробности — в `office/agents/demiurg/core.md` и `skills/build/`):
1. **Knowledge Mining** — собирает факты из data pack
2. **Build** — создаёт 3 файла в `office/agents/<new-name>/`: `core.md`, `overrides.md`, `CLAUDE.md` (склейка)
3. **Validate** — проверяет по рубрике (score ≥ 80 = PASS)
4. **Refine** — до 2 раундов правок если score < 80

Если после 2 раундов нет PASS — Демиург эскалирует клиенту с честным описанием "где не уверен".

### 4. Регистрация в команде

После PASS:
- Демиург добавляет агента в `office/AGENTS.md` в секцию **"Подключённые через Демиурга"** (таблица: Имя | Роль | Дата создания | Для какой задачи)
- Демиург дописывает запись в свой `overrides.md` в раздел "Мои агенты"
- Короткий анонс клиенту: "Готов! <name> в команде. Его роль — X. Director теперь знает про него и будет роутить задачи в его зону."

### 5. Передача Director'у

Director при следующей задаче клиента в зоне нового агента — **автоматически** его зовёт (он читает `office/AGENTS.md` при старте сессии).

## Fallback

- **Клиент не знает какого агента хочет** — Стратег может порекомендовать на основе текущего фокуса программы. Либо Демиург задаёт диагностические вопросы: "Что у тебя сейчас жрёт больше всего времени в этом проекте?"
- **Агент с таким именем уже существует** — предложить: пересобрать (через `/update-agent`) или дать новое имя (например, `copywriter-b2b` рядом с `copywriter`)
- **Создание упало (validate не проходит)** — НЕ отдаём плохого агента. Эскалация клиенту: "Вот что застопорилось — X. Нужен твой вклад: Y".
- **Клиент хочет сразу 3 агента** — Демиург говорит: "Строго по одному, чтобы каждый получился персональный. Начнём с первого?"

## Выход

- `office/agents/<new-name>/` — создан, 3 файла, PASS валидации
- `office/AGENTS.md` обновлён
- `office/agents/demiurg/overrides.md` — запись о созданном агенте
- Клиент видит анонс и может сразу ставить задачу новому агенту
