# wiki-ingest — сырьё → атомарная вики (Karpathy LLM Wiki)

> Канонический разбор подхода: `knowledge/ai-offices/distilled/memory-architecture.md`.
> Читай его раздел 1-2 при первом запуске в сессии — там суть паттерна и формат страниц.

## Зачем

RAG пересобирает знание из чанков на каждый вопрос — накопления нет. LLM Wiki —
это **компилятор**: знание один раз собирается в связанные markdown-страницы, дальше
просто читается по `[[ссылкам]]`. Этот скилл — операция **ingest** из паттерна Карпати.

## Три слоя (не путать!)

| Слой | Где | Правило |
|------|-----|---------|
| `raw` (сырьё) | `knowledge/ai-learning/<video>/`, `knowledge/inbox/`, `*/transcript*.txt`, исходные research-простыни | **Только читаю. Никогда не меняю.** |
| `wiki` (заметки) | `knowledge/domains/<домен>/` (concepts / tools / people / themes) | Создаю и линкую я. |
| schema (правила) | этот скилл + `knowledge/CLAUDE.md` | Читаю перед действием. |

## Формат атомарной заметки (строго)

Одна заметка = одна сущность (концепт / инструмент / человек / тема). Файл: kebab-case, ASCII.

```markdown
---
title: Человекочитаемый заголовок
type: concept | tool | person | theme
source: [raw/путь/к/файлу.md, ещё-источник.md]   # провенанс, откуда взято
tags: [memory, agents, context-engineering]
importance: 8        # 1-10, насколько ядерное знание (для ранжирования retrieval)
created: YYYY-MM-DD
last_used: YYYY-MM-DD
---

# Заголовок

## Summary
Одна строка для _MOC и INDEX (≤100 символов). Суть в одном предложении.

## Body
Суть простыми словами. Дословные цитаты — в кавычках с атрибуцией. 1-4 абзаца.
Длиннее — повод разбить на две заметки.

## Related
- [[другая-заметка]] — чем связано
- [[ещё-заметка]] — чем связано

## Provenance
- raw/<файл>:<секция/строка> — откуда конкретно взято
```

Правила содержания:
- **Атомарность.** Если в заметке две независимые мысли — две заметки.
- **Не выдумывать.** Нет в источнике → не пишем. Спорное → помечаем `[ГИПОТЕЗА]`.
- **Кросс-ссылки обязательны.** Заметка без `Related` — кандидат в сироты. Минимум 1-2 связи.
- **Provenance обязателен.** Каждая заметка знает, из какого сырья она.

## Workflow ingest (по шагам)

1. **Прочитай правила** — этот файл + `knowledge/CLAUDE.md` (если ещё не в контексте).
2. **Определи raw-источник(и)** — что ингестим (путь дал человек, либо новые файлы, которых нет в `log.md`).
3. **Определи целевой домен** — `knowledge/domains/<домен>/`. Для AI это `domains/ai/`.
   Таксономия домена — concepts / tools / people / themes (структура каждого домена — `knowledge/domains/<домен>/`).
4. **Прочитай источник целиком.** Извлеки концепты, сущности, людей, инструменты, связи.
5. **🚦 ГЕЙТ С ЧЕЛОВЕКОМ (обязательно).** Покажи списком: какие заметки создам/обновлю
   (имя + одна строка summary + тип), какие связи проставлю. Спроси: «всё ок / корректирую так-то?».
   **Не пиши в `wiki` до явного "ок".**
6. **После approval** — создай/обнови заметки в `domains/<домен>/` по формату выше.
   Один источник обычно трогает 5-15 страниц (Карпати: «LLM хорош в том, чтобы тронуть 15 файлов за проход»).
7. **Проставь `[[ссылки]]`** между новыми и существующими заметками. Если ссылаешься на ещё
   не созданный концепт — это нормально, пометь в отчёте для будущего ingest/lint.
8. **Обнови `_MOC.md`** домена — добавь новые страницы в нужную тему-карту.
9. **Обнови `knowledge/INDEX.md`** — строка на каждую новую заметку
   (`- [Title](domains/ai/concepts/.../slug.md) — summary ≤100 chars`) в нужной категории.
10. **Допиши `knowledge/log.md`** (append-only): `## YYYY-MM-DD — ingest — <источник>` +
    список созданных/обновлённых заметок.
11. **Отчёт человеку** — что создано, что обновлено, какие связи, какие «висячие» ссылки на будущее.

## Query (как это потом читается — справочно)

Ingest пишет так, чтобы query был дешёвым: агент читает `INDEX.md` → `_MOC.md` нужной темы →
идёт по `[[ссылкам]]` (глубина ≤3) → собирает контекст. Не grep по всему дереву.

## Guardrails

1. **Raw неприкосновенен** — читаю, не правлю, не удаляю.
2. **Гейт перед записью** — без явного «ок» в `wiki` не пишу.
3. **Не выдумываю факты** — только из источника, спорное → `[ГИПОТЕЗА]`.
4. **Provenance + хотя бы одна связь** на каждой заметке.
5. **kebab-case, ASCII** в именах файлов.
6. **INDEX + log + _MOC обновлены** — иначе заметка «потеряна» для агентов.
7. **Не дублирую** — перед созданием grep по существующим заголовкам; если есть похожее, обновляю, а не плодлю (это потом ловит `/wiki-lint`).
8. **Дёшево** — для тяжёлого ingest бери Haiku/Sonnet, Opus только на сложный синтез.

## Output Contract

Ответ = одно из:
- **План ingest + гейт** — «нашёл N топиков, создам такие заметки, ок?» (до записи)
- **Отчёт о выполнении** — что создано/обновлено/слинковано + висячие ссылки на будущее
- **Отказ** — «это не ingest, это <query/просто сохранить>, делаю иначе»
